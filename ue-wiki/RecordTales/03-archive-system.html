아카이브 시스템
기록 저장 & 캘린더

집중 세션 기록을 저장하고, 캘린더와 필터를 통해 조회하는 아카이브 시스템입니다. 기간·타입·카테고리별 필터링과 퀘스트 상태 연동까지 설계합니다.

기록 저장 캘린더 필터링
01 — concept

개념 이해

아카이브 시스템은 포커스 세션(집중 타이머 또는 스톱워치) 종료 후 사용자가 기록을 저장하고, 날짜·타입·카테고리별로 과거 기록을 조회하는 시스템입니다. "오늘 얼마나 집중했는지", "이번 주 공부에 얼마나 시간을 썼는지", "이번 달 운동 기록은 몇 번인지"를 한눈에 확인할 수 있게 합니다.

아카이브는 URecordSubsystem(GameInstanceSubsystem)이 담당하며, 기록 데이터는 URecordTalesSaveGame을 통해 디스크에 영속됩니다. 캘린더 위젯은 날짜별로 어떤 활동이 있었는지 도트(점)로 요약해 보여주며, RecordTales 전체에서 재사용되는 공통 UI 컴포넌트입니다.

핵심 용어

📖

FFocusRecord — 한 번의 집중 세션이 끝난 후 저장되는 기록 데이터. 카테고리, 집중 시간(초 단위), 모드(Timer/Stopwatch), 메모, 날짜, 시각, 타임스탬프를 포함합니다.

카테고리(Category) — 기록을 분류하는 사용자 정의 태그. "공부", "운동", "독서" 등을 자유롭게 추가할 수 있으며, 각 카테고리마다 고유 색상이 지정됩니다.

기간 필터(Period Filter) — 기록을 조회할 시간 범위를 결정하는 필터. Daily(하루), Weekly(월~일 한 주), Monthly(한 달) 3가지가 있습니다.

캘린더 도트(Calendar Dot) — 캘린더에서 날짜 아래에 표시되는 작은 점. F(주황, Timer 기록 있음), S(초록, Stopwatch 기록 있음), T(노랑, 퀘스트 있음) 3종류가 있습니다. 한 날짜에 여러 도트가 동시에 표시될 수 있습니다.

02 — data structure

데이터 구조

아카이브 시스템의 핵심 데이터는 FFocusRecord 구조체입니다. 집중 세션이 끝날 때마다 하나의 레코드가 생성되어 배열에 추가됩니다.

FFocusRecord — 집중 기록 구조체

// 포커스 모드 열거형
UENUM(BlueprintType)
enum class EFocusMode : uint8
{
    Timer      UMETA(DisplayName = "타이머"),
    Stopwatch  UMETA(DisplayName = "스톱워치")
};

USTRUCT(BlueprintType)
struct FFocusRecord
{
    GENERATED_BODY()

    // 카테고리 (예: "공부", "운동", "독서")
    UPROPERTY(BlueprintReadWrite, Category = "Record")
    FString Category;

    // 집중 시간 (초 단위)
    UPROPERTY(BlueprintReadWrite, Category = "Record")
    int32 TimeSeconds;

    // 포커스 모드: Timer(포모도로) 또는 Stopwatch
    UPROPERTY(BlueprintReadWrite, Category = "Record")
    EFocusMode Mode;

    // 사용자 메모 (선택 입력)
    UPROPERTY(BlueprintReadWrite, Category = "Record")
    FString Notes;

    // 기록 날짜 (YYYY.MM.DD 형식)
    UPROPERTY(BlueprintReadWrite, Category = "Record")
    FString Date;

    // 표시용 날짜 (예: "2026년 7월 2일 수요일")
    UPROPERTY(BlueprintReadWrite, Category = "Record")
    FString DateLong;

    // 기록 시각 (HH:MM 형식)
    UPROPERTY(BlueprintReadWrite, Category = "Record")
    FString Clock;

    // Unix 타임스탬프 (정렬·비교용)
    UPROPERTY(BlueprintReadWrite, Category = "Record")
    int64 Timestamp;
};

필드 설명 테이블

필드타입설명
CategoryFString기록의 분류. RecordSubsystem의 카테고리 목록에서 선택합니다. 필터링의 기준이 됩니다.
TimeSecondsint32실제 집중한 시간을 초 단위로 저장합니다. UI에서는 "1시간 23분" 형태로 변환하여 표시합니다.
ModeEFocusModeTimer(포모도로 타이머) 또는 Stopwatch(자유 스톱워치). 캘린더 도트 색상과 필터 분류에 사용됩니다.
NotesFString사용자가 선택적으로 남기는 메모. 빈 문자열 허용합니다.
DateFStringYYYY.MM.DD 형식의 날짜. 필터링과 그룹핑의 핵심 키입니다.
DateLongFStringUI 표시용 한국어 날짜 문자열. 기록 목록에서 사람이 읽기 쉽게 보여줍니다.
ClockFString기록 시점의 시:분. 같은 날 여러 기록이 있을 때 시간 순서를 보여줍니다.
Timestampint64Unix 타임스탬프. 날짜 문자열 비교보다 정확한 정렬이 필요할 때 사용합니다.

카테고리 시스템

카테고리는 두 가지 데이터로 관리됩니다. TArray<FString> Categories는 카테고리 이름 목록이고, TMap<FString, FLinearColor> CatColors는 각 카테고리의 고유 색상을 저장합니다. 사용자가 새 카테고리를 추가할 때 이름과 색상을 함께 지정합니다.

카테고리 추가 예시
// 카테고리 추가 로직
void AddCategory(FString Name, FLinearColor Color)
{
    // 1. 중복 검사
    if (Categories.Contains(Name)) return;

    // 2. 이름 목록에 추가
    Categories.Add(Name);

    // 3. 색상 맵에 등록
    CatColors.Add(Name, Color);

    // 4. SaveGame에 동기화
    SaveToSlot();
}

// 기본 카테고리 예시:
//   "공부"  → (0.35, 0.56, 0.98, 1.0)  파랑
//   "운동"  → (0.22, 0.85, 0.70, 1.0)  틸
//   "독서"  → (0.61, 0.43, 1.0,  1.0)  퍼플
03 — save flow

기록 저장 흐름

기록 저장은 포커스 세션이 끝난 직후에 발생합니다. UFocusSubsystem(포커스 타이머/스톱워치를 관리하는 서브시스템)이 세션 종료를 알리면, UI가 기록 모달을 띄우고, 사용자가 카테고리를 선택하고 메모를 입력한 뒤 "저장"을 누르면 URecordSubsystem에 기록이 추가됩니다.

1
세션 종료 → 델리게이트 브로드캐스트
UFocusSubsystem에서 타이머 완료 또는 스톱워치 중지 시 OnSessionCompleted 델리게이트(콜백 시스템)를 브로드캐스트합니다. 이 델리게이트에는 집중 시간(초)과 모드(Timer/Stopwatch) 정보가 포함됩니다.
2
Record Modal 표시
UI 레이어가 델리게이트를 수신하여 기록 모달(팝업 대화상자)을 화면에 띄웁니다. 모달에는 카테고리 선택 드롭다운과 메모 입력란이 있습니다.
3
사용자 입력
사용자가 카테고리를 선택하고, 선택적으로 메모를 입력합니다. 카테고리는 필수이며, 메모는 빈 문자열이 허용됩니다.
4
SaveRecord 호출
사용자가 "저장" 버튼을 누르면, UI가 URecordSubsystem::SaveRecord(FFocusRecord)를 호출합니다. FFocusRecord의 Date, DateLong, Clock, Timestamp 필드는 이 시점에 현재 시각으로 자동 채워집니다.
5
배열에 Unshift (최신순 삽입)
새 기록을 TArray<FFocusRecord> Records 배열의 맨 앞(인덱스 0)에 삽입합니다. 이렇게 하면 배열이 항상 최신순으로 정렬되어 있으므로, 조회 시 별도 정렬 없이 순서대로 읽으면 됩니다.
6
SaveGame에 동기화
Records 배열을 URecordTalesSaveGame에 저장하여 디스크에 영속합니다. 앱을 재시작해도 기록이 유지됩니다.
💡

기록 저장은 Optional. 사용자가 "건너뛰기"를 누르면 기록하지 않습니다. 집중 세션 자체는 완료되지만, 아카이브에 기록이 남지 않습니다. 이는 사용자가 기록을 원하지 않는 가벼운 세션(테스트, 짧은 타이머 등)을 위한 것입니다.

04 — calendar

캘린더

캘린더 위젯은 월별 탐색과 날짜 선택 기능을 제공합니다. 좌우 화살표로 이전/다음 달을 탐색하고, 특정 날짜를 탭하면 해당 날짜의 기록 목록이 표시됩니다. 각 날짜 셀 아래에는 도트(점)가 표시되어, 그 날짜에 어떤 종류의 활동이 있었는지를 색상으로 요약해 보여줍니다.

날짜별 도트 표시

도트색상데이터 소스판정 조건
F주황RecordSubsystem해당 날짜에 Mode = Timer인 기록이 1개 이상 존재
S초록RecordSubsystem해당 날짜에 Mode = Stopwatch인 기록이 1개 이상 존재
T노랑QuestSubsystem해당 날짜에 Active, Done, 또는 Overdue 상태의 퀘스트 인스턴스가 1개 이상 존재

하나의 날짜에 여러 도트가 동시에 표시될 수 있습니다. 예를 들어 2026년 7월 2일에 타이머 기록과 퀘스트가 모두 있다면, F와 T 도트가 나란히 표시됩니다.

📅

캘린더는 RecordTales 전체에서 재사용되는 공통 위젯입니다. 아카이브 화면에서는 기록 조회용으로, 퀘스트 생성/수정 화면에서는 시작일·마감일·반복 종료일(앞서 설명한 RecEndDate) 선택에도 동일한 캘린더 위젯을 사용합니다. 위젯을 하나만 만들고 용도별로 모드를 전환하는 구조입니다.

05 — filtering

필터링 시스템

필터링은 3축으로 구성됩니다: 기간(어느 범위의 날짜를 볼 것인가) × 타입(Timer/Stopwatch/전체) × 카테고리(특정 분류 또는 전체). 3축을 조합하면 "이번 주 공부 카테고리의 타이머 기록만"처럼 세밀한 조회가 가능합니다.

기간 필터 범위

기간 필터범위예시 (기준일: 2026.07.02 수요일)
Daily선택한 날짜 1일2026.07.02 하루
Weekly선택한 날짜가 속한 월요일~일요일2026.06.29(월) ~ 2026.07.05(일)
Monthly선택한 날짜가 속한 월의 1일~말일2026.07.01 ~ 2026.07.31

타입 필터

All(전체), Timer(포모도로 타이머만), Stopwatch(스톱워치만) 3가지 중 하나를 선택합니다. 기본값은 All입니다.

카테고리 필터

전체 카테고리를 보거나, 특정 카테고리 하나를 선택할 수 있습니다. 카테고리 목록은 RecordSubsystem의 Categories 배열에서 가져옵니다.

필터 적용 로직

1
기간으로 날짜 범위 계산
선택된 기간 필터(Daily/Weekly/Monthly)와 기준 날짜를 입력받아 시작일·종료일을 계산합니다. Weekly는 월요일 시작 기준입니다.
2
범위 내 기록 필터
Records 배열에서 Date 필드가 계산된 범위(시작일 ≤ Date ≤ 종료일)에 해당하는 기록만 추출합니다.
3
타입 필터 적용
All이 아닌 경우, Mode 필드가 선택된 타입(Timer 또는 Stopwatch)과 일치하는 기록만 남깁니다.
4
카테고리 필터 적용
전체가 아닌 특정 카테고리가 선택된 경우, Category 필드가 일치하는 기록만 남깁니다.
5
날짜별 그룹핑 + 누적 시간 계산
필터링된 기록을 Date 기준으로 그룹핑하고, 각 그룹(날짜)의 TimeSeconds를 합산하여 누적 시간을 계산합니다. 전체 기간의 총 누적 시간도 함께 계산합니다.

퀘스트 표시 연동

아카이브 화면에서는 집중 기록뿐 아니라, 같은 기간 내의 퀘스트 인스턴스도 함께 표시합니다. UQuestSubsystem::GetInstancesForDate()를 기간 내 각 날짜에 대해 호출하여 퀘스트를 가져옵니다.

✅ 왼쪽: 완료된 퀘스트

상태가 Done인 퀘스트 인스턴스를 표시합니다. "이 기간에 몇 개의 할 일을 완료했는지"를 보여줍니다.

Done

📋 오른쪽: 나머지 퀘스트

Active(진행 중), Overdue(기한 초과), Suspended(보류), Upcoming(예정) 상태의 퀘스트를 표시합니다.

Active Overdue Suspended Upcoming
06 — implementation skeleton

구현 스켈레톤

URecordSubsystem은 UGameInstanceSubsystem으로 구현합니다. GameInstance의 수명(앱 시작~종료)과 동일하게 유지되므로, 레벨 전환에도 기록 데이터가 사라지지 않습니다. 기록의 디스크 영속은 URecordTalesSaveGame이 담당합니다. 

#pragma once

#include "CoreMinimal.h"
#include "Subsystems/GameInstanceSubsystem.h"
#include "RecordTypes.h"
#include "RecordSubsystem.generated.h"

/** 기간 필터 열거형 */
UENUM(BlueprintType)
enum class EPeriodFilter : uint8
{
    Daily    UMETA(DisplayName = "일별"),
    Weekly   UMETA(DisplayName = "주별"),
    Monthly  UMETA(DisplayName = "월별")
};

UCLASS()
class RECORDTALES_API URecordSubsystem : public UGameInstanceSubsystem
{
    GENERATED_BODY()

public:
    virtual void Initialize(FSubsystemCollectionBase& Collection) override;
    virtual void Deinitialize() override;

    // ── 기록 저장 ──

    /** 새 기록을 저장한다 (배열 맨 앞에 삽입). */
    UFUNCTION(BlueprintCallable, Category = "Record")
    void SaveRecord(const FFocusRecord& InRecord);

    // ── 조회 ──

    /** 날짜 범위 내 기록을 반환한다. */
    UFUNCTION(BlueprintCallable, Category = "Record")
    TArray<FFocusRecord> GetRecordsForDateRange(
        const FString& StartDate,
        const FString& EndDate) const;

    /** 3축 필터(기간 × 타입 × 카테고리) 적용된 기록을 반환한다. */
    UFUNCTION(BlueprintCallable, Category = "Record")
    TArray<FFocusRecord> GetRecordsFiltered(
        const FString& BaseDate,
        EPeriodFilter Period,
        EFocusMode ModeFilter,
        const FString& CategoryFilter,
        bool bFilterAll) const;

    // ── 카테고리 ──

    /** 새 카테고리를 추가한다 (중복 시 무시). */
    UFUNCTION(BlueprintCallable, Category = "Record")
    void AddCategory(const FString& Name, FLinearColor Color);

    /** 카테고리 목록을 반환한다. */
    UFUNCTION(BlueprintCallable, Category = "Record")
    TArray<FString> GetCategories() const;

private:
    // 전체 기록 배열 (최신순)
    UPROPERTY()
    TArray<FFocusRecord> Records;

    // 카테고리 이름 목록
    TArray<FString> Categories;

    // 카테고리 → 색상 맵
    TMap<FString, FLinearColor> CatColors;

    // 기간 필터로 날짜 범위를 계산하는 헬퍼
    void CalcDateRange(const FString& BaseDate,
                       EPeriodFilter Period,
                       FString& OutStart,
                       FString& OutEnd) const;
};

TArray<FFocusRecord> URecordSubsystem::GetRecordsForDateRange(
    const FString& StartDate,
    const FString& EndDate) const
{
    TArray<FFocusRecord> Result;

    for (const FFocusRecord& Rec : Records)
    {
        // 날짜 문자열 비교 (YYYY.MM.DD 형식이므로 사전순 = 시간순)
        if (Rec.Date >= StartDate && Rec.Date <= EndDate)
        {
            Result.Add(Rec);
        }
    }

    return Result;
}

TArray<FFocusRecord> URecordSubsystem::GetRecordsFiltered(
    const FString& BaseDate,
    EPeriodFilter Period,
    EFocusMode ModeFilter,
    const FString& CategoryFilter,
    bool bFilterAll) const
{
    // 1단계: 기간으로 날짜 범위 계산
    FString RangeStart, RangeEnd;
    CalcDateRange(BaseDate, Period, RangeStart, RangeEnd);

    // 2단계: 범위 내 기록 가져오기
    TArray<FFocusRecord> Filtered = GetRecordsForDateRange(RangeStart, RangeEnd);

    // 3단계: 타입 필터 (bFilterAll이 false이면 ModeFilter 적용)
    if (!bFilterAll)
    {
        Filtered.RemoveAll([&](const FFocusRecord& R) {
            return R.Mode != ModeFilter;
        });
    }

    // 4단계: 카테고리 필터
    if (!CategoryFilter.IsEmpty())
    {
        Filtered.RemoveAll([&](const FFocusRecord& R) {
            return R.Category != CategoryFilter;
        });
    }

    return Filtered;
}

07 — extension & pitfalls

확장 & 주의

⚠️
기록 배열이 커지면 날짜별 TMap 인덱싱을 고려하세요. TArray<FFocusRecord>를 순차 탐색하는 GetRecordsForDateRange()는 기록이 수백~수천 개가 되면 느려질 수 있습니다. 이때 TMap<FString, TArray<FFocusRecord>>로 날짜별 인덱스를 별도로 유지하면 O(날짜 수)로 조회할 수 있습니다. 단, 삽입 시 두 곳(배열 + 맵)을 동시에 갱신해야 합니다.
⚠️
주간 범위 계산에서 월요일 시작 기준을 통일하세요. FDateTime::GetDayOfWeek()는 0=월요일, 6=일요일을 반환합니다. 주의 시작을 일요일로 바꾸면 캘린더 위젯과 필터링의 "주" 범위가 어긋나 다른 기록이 보입니다. RecordTales 전체에서 월요일 시작으로 통일합니다.

DO

  • 기록 삽입 시 항상 인덱스 0에 Insert하여 최신순 유지
  • 날짜 문자열은 YYYY.MM.DD 형식 함수를 통해 생성
  • 캘린더 위젯은 하나만 만들고 모드별로 재사용
  • 카테고리 색상은 FLinearColor로 저장하여 UI에서 직접 사용
  • 통계 대시보드 확장 시 이 필터링 로직을 기반으로 구축

DON'T

  • 기록을 정렬 없이 배열 끝에 Add하지 말 것 — 최신순 보장이 깨짐
  • 주간 범위 시작일을 일요일로 바꾸지 말 것 — 캘린더와 불일치
  • 카테고리를 하드코딩하지 말 것 — 사용자가 자유롭게 추가 가능해야 함
  • 날짜별 인덱싱 없이 수천 개 기록을 매번 전체 탐색하지 말 것
  • DeleteRecord 시 배열만 수정하고 SaveGame 동기화를 빼먹지 말 것
💡

통계 대시보드(일별/주별/월별 집중 시간 차트)는 이 데이터를 기반으로 확장 가능합니다. GetRecordsFiltered()의 결과를 날짜별로 그룹핑하고 TimeSeconds를 합산하면 차트 데이터가 됩니다. 카테고리별 비율 파이 차트, 주간 추이 라인 차트 등으로 확장할 수 있습니다.
