책임 배치 결정표

어떤 기능을 어느 UE 프레임워크 클래스에 둘지, 그리고 그 근거입니다.

기능 UE 클래스 근거
전역 데이터 (Gold, XP, 길드 레벨, 해금 상태) URecordTalesGameInstance 레벨 전환에도 유지. 프로그램 수명.
타이머/스톱워치 상태 UFocusSubsystem GameInstance 수명 + 독립 모듈.
퀘스트 CRUD + 반복 규칙 UQuestSubsystem GameInstance 수명 + 독립 모듈.
집중 기록 저장/조회 URecordSubsystem GameInstance 수명 + 아카이브 전담.
길드원 풀 + 편성 + 시너지 UGuildSubsystem GameInstance 수명 + 게임화 전담.
경제 (Gold/XP 적립, 레벨업) UEconomySubsystem GameInstance 수명 + 통화 전담.
오디오 재생 (앰비언트, 뮤직) UAudioSubsystem 레벨 전환에도 음악 유지.
디스크 영속 (세이브/로드) URecordTalesSaveGame 껐다 켜도 유지.
전체 UI 관리 (패널, 모달, 토스트) UUIManagerSubsystem 위젯 생성/전환/스택.
⚙

"왜 Subsystem인가" — 언리얼의 Subsystem은 수명 범위(GameInstance/World/Player 등)에 자동으로 붙는 서비스 객체입니다. 직접 생성/파괴를 관리할 필요 없고, 모듈별로 깔끔하게 분리됩니다. RecordTales의 모든 핵심 로직은 UGameInstanceSubsystem을 상속해 프로그램이 살아 있는 동안 유지되며, 레벨 전환에도 상태가 보존됩니다.

04 — 개발 용어집

개발 용어집

RecordTales 개발에 필요한 모든 핵심 용어를 정의 · 데이터 구조 · UE 프레임워크 · 설계 방향 · 상세 문서와 함께 정리합니다. 단순 정의가 아니라 실제 개발을 완성할 수 있을 만큼 구체적으로 기술합니다.

용어 정의 데이터 구조 UE 프레임워크 설계 방향 상세
시간 관리
포모도로 타이머 25분 집중 + 5분 휴식을 반복하는 시간 관리 기법 int32 FocusDuration, int32 RestDuration, int32 LastRestDuration, int32 RepeatCount UFocusSubsystem FTimerHandle로 카운트다운. 상태 머신(Idle→Focusing→Resting→LastResting) 01 포커스
스톱워치 0초부터 자유 측정하는 시간 추적기 float SwElapsed, bool bSwRunning UFocusSubsystem Tick 기반 누적. 1500초(25분) 이상 시 XP 적립 01 포커스
세션 하나의 집중 또는 스톱워치 실행 단위 FFocusSessionData {Mode, AccFocusTime, CycleIndex, bNormalEnd} UFocusSubsystem 종료 시 RecordSubsystem에 기록 전달 01 포커스
루틴 포모도로 반복 횟수(1~10) 설정 int32 RepeatCount, bool bIsRoutine UFocusSubsystem 1회=집중+단일휴식, 2+회=집중+휴식×(N-1)+마지막휴식 01 포커스
Gold 틱 집중 단계에서 초당 1 Gold 적립 int32 Gold, int32 GoldTickCounter UEconomySubsystem FocusSubsystem이 Tick마다 EconomySubsystem에 AddGold 호출 05 경제
퀘스트
퀘스트 마스터 할 일의 원본 레코드 (반복 규칙 포함) FQuestMasterData {Id, Title, Desc, Category, Recurrence, StartDate, Deadline, DeadlineOffset, RecEndDate, RecurData, bFavorite} UQuestSubsystem TArray<FQuestMasterData>로 관리. ID는 YYYYMMDD+5자리 시퀀스 02 퀘스트
퀘스트 인스턴스 특정 날짜에 발생하는 마스터의 복제본 FQuestInstanceData {MasterId, OccurrenceDate, bDone, DoneAt, bSuspended, SuspendReason} UQuestSubsystem 마스터의 RecurData TMap에서 날짜별 상태 관리 02 퀘스트
반복 규칙 None/Daily/Weekly/Monthly 주기 EQuestRecurrence UENUM UQuestSubsystem Weekly는 요일 배열, Monthly는 날짜 배열. 자동 종료 1년 02 퀘스트
자정 롤오버 매일 0시에 새 반복 인스턴스를 선점 FString LastRolloverDate UQuestSubsystem Tick에서 날짜 변경 감지 또는 타이머. 백그라운드 복귀 시 재확인 02 퀘스트
상태 머신(퀘스트) Overdue→Active→Upcoming→Done→Suspended EQuestStatus UENUM (계산 속성) UQuestSubsystem DB 저장 안 함. 현재 날짜 기준 실시간 계산 02 퀘스트
D-Day 마감일까지 남은 일수 표시 런타임 계산 UI 위젯 (오늘 - 마감일).Days로 계산. D-N/D-DAY/D+N 02 퀘스트
아카이브
집중 기록 세션 종료 후 저장되는 기록 FFocusRecord {Category, TimeSeconds, Mode, Notes, Date, Clock, Timestamp} URecordSubsystem TArray<FFocusRecord>. 최신순 Unshift 03 아카이브

05 — 사용법

이 허브 활용하기

RecordTales 개발 시 이 허브를 다음 3단계로 활용합니다.

1
문서를 선택해 시스템 이해
문서 목록에서 구현하려는 시스템의 설계 문서를 열어 전체 구조와 동작 흐름을 파악합니다.
2
책임 배치 결정표로 UE 클래스 확인
아키텍처 개요의 결정표에서 해당 기능이 어느 UE 프레임워크 클래스(GameInstance, Subsystem, SaveGame 등)에 배치되는지, 그리고 그 근거를 확인합니다.
3
용어집에서 데이터 구조 · 설계 방향 참조
개발 용어집에서 해당 기능의 데이터 구조(USTRUCT/UENUM)와 구체적인 설계 방향을 참조하여 구현을 시작합니다.
⚠️

이 허브의 링크는 같은 폴더(RecordTales/) 안에서만 동작합니다. 각 문서는 CSS inline으로 단독 열람 가능합니다.
