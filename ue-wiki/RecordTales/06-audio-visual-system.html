01 — 개념 이해

오디오 & 비주얼 시스템이란

오디오 & 비주얼 시스템은 사용자의 집중 환경을 조성하는 감각 시스템입니다. 게임플레이(타이머, 퀘스트 등)와 독립적으로 작동하며, 배경 소리와 음악, 시각적 분위기를 사용자 취향에 맞게 커스터마이징할 수 있도록 설계되었습니다. 핵심 클래스는 UAudioSubsystem(UGameInstanceSubsystem을 상속하여 프로그램 수명 동안 유지)이며, 배경 테마 적용은 UI 위젯이 담당합니다.

핵심 용어

📚

앰비언트 사운드(Ambient Sound) — 빗소리, 바람, 불, 숲, 바다 등 환경을 연출하는 배경 음향. 각 사운드는 독립 볼륨을 가지며 동시에 여러 종류를 재생(믹싱)할 수 있습니다.
뮤직 플레이어(Music Player) — 장르별 BGM 6곡을 Play/Pause, 이전/다음, 시크(시간 이동), 볼륨 조절로 재생하는 컴포넌트.
배경 테마(Background Theme) — UI 전체의 배경 이미지를 교체하는 시각 커스터마이징 기능. 8종 이상을 제공합니다.
해금 연동 — 배경과 일부 음악 트랙은 UEconomySubsystem의 해금 시스템(Gold로 구매)을 통해 점진적으로 개방됩니다.

📌

이 시스템은 게임플레이와 무관하며, 사용자 경험(분위기)을 위한 것입니다. 타이머가 돌아가는 동안 빗소리와 로파이 음악을 동시에 틀어 집중 환경을 만드는 것이 대표 사용 시나리오입니다.

02 — 앰비언트 사운드

환경 음향 5종 믹싱

앰비언트 사운드는 5가지 자연 환경음을 제공합니다. 각 사운드는 독립적인 볼륨(0~100%)을 가지며, 여러 사운드를 동시에 재생(믹싱)할 수 있습니다. 글로벌 토글로 전체 앰비언트를 한 번에 켜거나 끌 수 있습니다.

5종 앰비언트 목록

Rain (빗소리) Wind (바람) Fire (불) Forest (숲) Ocean (바다)

각 앰비언트는 언리얼의 UAudioComponent(오디오를 재생하는 컴포넌트)를 하나씩 생성하여 관리합니다. UAudioComponent는 액터에 부착 가능한 컴포넌트로, Play()/Stop()/SetVolumeMultiplier() 등의 함수를 제공합니다. Looping(반복 재생)을 true로 설정하면 끊김 없이 계속 재생됩니다.

데이터 구조

항목타입설명
EAmbientTypeUENUMRain, Wind, Fire, Forest, Ocean — 5종 식별자. 새 사운드를 추가하려면 이 열거형에 값을 추가합니다.
AmbientVolumesTMap<EAmbientType, float>각 사운드의 볼륨(0.0~1.0). 0.0이면 음소거, 1.0이면 최대.
AmbientComponentsTMap<EAmbientType, UAudioComponent*>런타임에 생성된 각 사운드의 재생 컴포넌트. Subsystem 초기화 시 5개 생성.
bAmbientEnabledbool글로벌 토글. false이면 모든 앰비언트를 일시 정지합니다.
💡

UAudioComponent를 사운드별로 하나씩 생성하면 독립 볼륨 제어가 쉽습니다. 각 컴포넌트의 Sound 속성에 해당 USoundWave 에셋을 연결하고, bIsLooping = true로 설정하세요. 볼륨 변경은 SetVolumeMultiplier(float) 한 줄이면 됩니다.

03 — 뮤직 플레이어

BGM 재생 시스템

뮤직 플레이어(앞서 설명한 BGM 재생 컴포넌트)는 장르별 6곡의 음악을 Play/Pause, 이전/다음 트랙 이동, 시크(Seek: 재생 위치를 특정 시간으로 이동), 볼륨 조절 기능으로 재생합니다. 미해금 곡은 30초 미리듣기 후 자동 정지됩니다.

마스터 데이터 (DataTable)

음악 트랙 정보는 UDataTable(언리얼의 데이터 테이블: 에디터에서 행 단위로 편집 가능한 구조체 배열)에 아래 구조체로 저장합니다.

FMusicTrackRow — USTRUCT 
USTRUCT(BlueprintType)
FMusicTrackRow : FTableRowBase
{
    FString                    Id;          // 고유 식별자 (예: "track_medieval_01")
    FString                    Title;       // 표시 이름
    FString                    Genre;       // Medieval, Lo-fi Fantasy, Ambient ...
    float                      Duration;    // 총 재생 시간(초)
    TSoftObjectPtr<USoundWave> SoundWave;   // 실제 오디오 에셋 (Soft 참조 → 필요 시 로드)
    bool                       bFree;       // true = 기본 제공, false = 해금 필요
}

TSoftObjectPtr(소프트 오브젝트 포인터)는 에셋을 즉시 메모리에 올리지 않고, 실제로 재생할 때만 비동기 로드하는 참조 방식입니다. 이를 통해 6곡의 오디오 에셋이 동시에 메모리를 차지하지 않도록 합니다.

플레이어 상태

항목타입설명
CurrentTrackIndexint32현재 재생 중인 곡의 인덱스(0부터 시작). 트랙 배열에서의 위치.
bPlayingbool현재 재생 중이면 true, 일시정지면 false.
PlaybackPositionfloat현재 재생 위치(초 단위). 시크 바 UI에 바인딩됩니다.
MusicVolumefloat뮤직 전용 볼륨(0.0~1.0). 앰비언트 볼륨과 독립입니다.

곡 재생 흐름

1
트랙 선택
사용자가 목록에서 곡을 클릭하거나, 이전/다음 버튼을 눌러 CurrentTrackIndex를 변경합니다.
2
해금 여부 확인
UEconomySubsystem(경제 서브시스템)의 UnlockedIds 목록에 해당 트랙 Id가 포함되어 있는지 확인합니다. bFree == true인 곡은 항상 통과합니다.
3
미해금 → 30초 미리듣기
해금되지 않은 곡은 30초만 재생 후 자동 정지합니다. FTimerHandle로 30초 후 PauseTrack()을 호출합니다.
4
해금 → 전체 재생
해금된 곡은 시간 제한 없이 끝까지 재생됩니다. 곡이 끝나면 다음 트랙으로 자동 전환할 수 있습니다.
5
UAudioComponent로 재생
뮤직 전용 UAudioComponent에 SetSound(LoadedSoundWave) 후 Play()를 호출합니다. SetVolumeMultiplier(MusicVolume)로 볼륨을 적용합니다.
04 — 배경 테마

UI 배경 이미지 교체

배경 테마(앞서 설명한 UI 배경 이미지 커스터마이징 기능)는 8종 이상의 배경 이미지를 제공합니다. 사용자가 선택한 배경은 HUD의 전체 화면 UImage 위젯(UMG의 이미지 표시 위젯)에 적용됩니다.

마스터 데이터

배경 정보도 UDataTable에 저장합니다.

FBackgroundRow — USTRUCT 
USTRUCT(BlueprintType)
FBackgroundRow : FTableRowBase
{
    FString                       Id;          // 고유 식별자 (예: "bg_forest_01")
    FString                       DisplayName; // UI에 표시할 이름
    TSoftObjectPtr<UTexture2D>   Texture;     // 배경 텍스처 (Soft 참조)
    int32                         Cost;        // 해금 비용 (Gold). 0이면 기본 제공
}

적용 방식

항목설명
CurrentBgId현재 적용 중인 배경의 ID. UAudioSubsystem이 보관하며, 세이브 데이터에도 저장됩니다.
해금 확인UEconomySubsystem의 UnlockedIds에 해당 배경 Id가 포함되어 있는지 확인합니다.
텍스처 적용UImage::SetBrushFromTexture(LoadedTexture) 또는 SetBrushFromSoftTexture(SoftPtr)로 위젯의 브러시(Brush: UMG에서 이미지를 표시하는 설정 객체)를 교체합니다.
⚠️

배경 텍스처는 반드시 TSoftObjectPtr로 관리하여 메모리 효율을 높이세요. 8장 이상의 고해상도 텍스처를 동시에 메모리에 올리면 수백 MB를 차지할 수 있습니다. 현재 적용 중인 배경만 로드하고, 이전 배경의 참조를 해제하면 가비지 컬렉션(GC: 참조 없는 UObject를 자동 해제하는 메모리 관리)이 메모리를 회수합니다.

05 — 해금 연동

EconomySubsystem과의 연결

배경 테마와 일부 음악 트랙은 UEconomySubsystem(경제 서브시스템: Gold/XP 관리, 해금 트리 담당)의 해금 시스템과 연동됩니다. 사용자가 Gold를 소비하여 콘텐츠를 구매하면, 해당 콘텐츠가 활성화됩니다.

해금 → 콘텐츠 활성화 흐름

1
사용자가 해금 샵에서 구매
해금 샵 모달(UUnlockShopModalWidget)에서 배경 또는 음악을 선택하고 "구매" 버튼을 누릅니다.
2
EconomySubsystem::PurchaseUnlock(Id)
UEconomySubsystem이 Gold 잔액을 확인하고, 충분하면 Gold를 차감한 뒤 UnlockedIds 목록에 해당 Id를 추가합니다. 부족하면 실패를 반환합니다.
3
AudioSubsystem에 알림
UEconomySubsystem이 발행하는 OnUnlockChanged 델리게이트(콜백 시스템: 이벤트 발생 시 등록된 함수를 자동 호출)를 UAudioSubsystem이 구독하고 있으므로, 해금 상태가 변경되면 자동으로 알림을 받습니다. 또는 필요할 때 직접 EconomySubsystem->IsUnlocked(Id)로 질의합니다.
4
UI 갱신
잠금 아이콘이 제거되고, 재생/적용 버튼이 활성화됩니다. 뮤직 라이브러리와 배경 테마 선택 UI가 갱신됩니다.
💡

해금 상태는 UEconomySubsystem이 유일하게 소유합니다. UAudioSubsystem은 "이 곡/배경이 해금됐는지?"를 EconomySubsystem에 질의할 뿐, 자체적으로 해금 목록을 들고 있지 않습니다. 이렇게 하면 해금 로직이 한 곳에 집중되어 유지보수가 쉽습니다.

06 — 구현 스켈레톤

UAudioSubsystem 최소 코드

아래는 UAudioSubsystem의 최소 동작 뼈대입니다. 헤더(.h)에 인터페이스를 선언하고, 소스(.cpp)에 핵심 함수 하나(SetAmbientVolume)의 구현 골격을 보여줍니다.

#pragma once

#include "CoreMinimal.h"
#include "Subsystems/GameInstanceSubsystem.h"
#include "AudioSubsystem.generated.h"

// 앰비언트 사운드 5종 식별자
UENUM(BlueprintType)
enum class EAmbientType : uint8
{
    Rain,
    Wind,
    Fire,
    Forest,
    Ocean
};

UCLASS()
class RECORDTALES_API UAudioSubsystem : public UGameInstanceSubsystem
{
    GENERATED_BODY()

public:
    virtual void Initialize(FSubsystemCollectionBase& Collection) override;
    virtual void Deinitialize() override;

    // ── 앰비언트 ──
    UFUNCTION(BlueprintCallable, Category = "Audio|Ambient")
    void SetAmbientVolume(EAmbientType Type, float Volume);

    UFUNCTION(BlueprintCallable, Category = "Audio|Ambient")
    void ToggleAmbient(bool bEnable);

    // ── 뮤직 ──
    UFUNCTION(BlueprintCallable, Category = "Audio|Music")
    void PlayTrack(int32 TrackIndex);

    UFUNCTION(BlueprintCallable, Category = "Audio|Music")
    void PauseTrack();

    UFUNCTION(BlueprintCallable, Category = "Audio|Music")
    void NextTrack();

    UFUNCTION(BlueprintCallable, Category = "Audio|Music")
    void PrevTrack();

    UFUNCTION(BlueprintCallable, Category = "Audio|Music")
    void SetMusicVolume(float Volume);

    // ── 배경 테마 ──
    UFUNCTION(BlueprintCallable, Category = "Audio|Background")
    void ApplyBackground(const FString& BgId);

private:
    // 앰비언트
    TMap<EAmbientType, UAudioComponent*> AmbientComponents;
    TMap<EAmbientType, float> AmbientVolumes;
    bool bAmbientEnabled = true;

    // 뮤직
    UPROPERTY()
    UAudioComponent* MusicComponent = nullptr;
    int32 CurrentTrackIndex = 0;
    bool bPlaying = false;
    float MusicVolume = 1.0f;

    // 배경
    FString CurrentBgId;
};
#include "AudioSubsystem.h"
#include "Components/AudioComponent.h"
#include "Kismet/GameplayStatics.h"

void UAudioSubsystem::Initialize(FSubsystemCollectionBase& Collection)
{
    Super::Initialize(Collection);

    // 앰비언트 5종에 대해 UAudioComponent를 생성합니다.
    // 실제 구현에서는 SoundWave 에셋을 로드하여 각 컴포넌트에 연결합니다.
}

void UAudioSubsystem::SetAmbientVolume(EAmbientType Type, float Volume)
{
    // 볼륨을 0.0~1.0 범위로 클램프
    Volume = FMath::Clamp(Volume, 0.0f, 1.0f);
    AmbientVolumes.Add(Type, Volume);

    // 해당 타입의 AudioComponent를 찾아 볼륨 적용
    UAudioComponent** Found = AmbientComponents.Find(Type);
    if (Found && *Found)
    {
        (*Found)->SetVolumeMultiplier(Volume);

        // 볼륨이 0이면 정지, 0보다 크면 재생 (글로벌 토글이 켜진 경우)
        if (Volume > 0.0f && bAmbientEnabled && !(*Found)->IsPlaying())
        {
            (*Found)->Play();
        }
        else if (Volume <= 0.0f && (*Found)->IsPlaying())
        {
            (*Found)->Stop();
        }
    }
}

void UAudioSubsystem::ToggleAmbient(bool bEnable)
{
    bAmbientEnabled = bEnable;

    for (auto& Pair : AmbientComponents)
    {
        if (Pair.Value)
        {
            if (bEnable)
            {
                float* Vol = AmbientVolumes.Find(Pair.Key);
                if (Vol && *Vol > 0.0f)
                    Pair.Value->Play();
            }
            else
            {
                Pair.Value->Stop();
            }
        }
    }
}

주요 함수 설명

함수역할핵심 로직
SetAmbientVolume특정 앰비언트의 볼륨 변경볼륨 클램프 → TMap 갱신 → AudioComponent에 적용. 0이면 정지, 0 초과면 재생.
ToggleAmbient전체 앰비언트 On/OffbAmbientEnabled 플래그 변경 → 모든 컴포넌트 Play/Stop.
PlayTrack지정 인덱스의 곡 재생해금 확인 → SoundWave 로드 → MusicComponent에 설정 → Play.
PauseTrack현재 곡 일시정지MusicComponent가 재생 중이면 Stop. bPlaying = false.
NextTrack / PrevTrack다음/이전 트랙 이동인덱스 증감(순환) → PlayTrack 호출.
SetMusicVolume뮤직 볼륨 변경MusicComponent->SetVolumeMultiplier(Volume).
ApplyBackground배경 테마 적용해금 확인 → TSoftObjectPtr 로드 → 델리게이트로 UI에 알림.
07 — 확장 & 주의

Pitfall & 확장 가이드

⚠️
앰비언트 + 뮤직 동시 재생 시 오디오 버스 분리 권장 — 두 종류의 사운드가 같은 오디오 버스(Sound Bus: 사운드를 묶어 볼륨/이펙트를 일괄 조절하는 단위)에서 재생되면 볼륨 조절이 엉킬 수 있습니다. 언리얼의 Sound Class(앰비언트용, 뮤직용 각각 생성)와 Sound Mix(마스터 믹스)를 사용해 분리하세요. Sound Class 속성은 각 UAudioComponent의 SoundClassOverride에서 지정합니다.
⚠️
TSoftObjectPtr 로드 실패 시 nullptr 체크 필수 — 소프트 포인터로 참조한 에셋이 삭제되었거나 경로가 변경되면 로드가 실패하여 nullptr이 반환됩니다. 로드 후 반드시 if (LoadedAsset) 체크를 하고, 실패 시 기본 에셋(폴백)을 사용하거나 토스트 알림으로 사용자에게 알려주세요.
⚠️
레벨 전환 시 UAudioComponent가 파괴되지 않도록 주의 — UAudioSubsystem은 UGameInstanceSubsystem이므로 레벨이 바뀌어도 Subsystem 자체는 살아있지만, Subsystem이 소유하지 않는 UAudioComponent는 월드와 함께 파괴될 수 있습니다. UAudioComponent를 Subsystem의 UPROPERTY() 멤버로 선언하거나, NewObject로 생성하여 Subsystem이 Outer(소유자)가 되게 하면 월드 전환 시에도 유지됩니다.

Do & Don't

✓ 이렇게

  • 각 앰비언트를 독립 UAudioComponent로 관리 (볼륨 독립 제어)
  • TSoftObjectPtr로 사운드/텍스처 지연 로드 (메모리 절약)
  • 해금 상태는 EconomySubsystem에만 질의 (단일 책임)
  • Sound Class 분리로 앰비언트/뮤직 볼륨 독립 관리
  • 세이브 시 AmbientVolumes, CurrentBgId를 SaveGame에 저장

✗ 이건 피하기

  • 모든 사운드를 하나의 AudioComponent에서 재생 (볼륨 충돌)
  • 배경 텍스처 8장을 미리 전부 로드 (메모리 낭비)
  • AudioSubsystem이 자체 해금 목록을 복제 보관 (동기화 버그)
  • UAudioComponent를 로컬 변수로만 관리 (레벨 전환 시 파괴)
  • 미리듣기 타이머를 걸어놓고 해제하지 않기 (메모리 누수)
