Table

아이템 스펙·몬스터 능력치·상점 가격처럼 자주 바뀌는 수치 데이터를 코드 밖의 표로 빼서 관리하는 언리얼의 표준 방법입니다. 구조체 한 개를 정의하면 에디터에서 엑셀처럼 편집하고, CSV로 임포트하고, 코드 한 줄로 조회할 수 있습니다.

엔진 자체 기능 UE4 · UE5 C++ · Blueprint · CSV/JSON
01 — 개념 · 역할

왜 데이터를 코드 밖으로 빼는가

검의 공격력을 Damage = 25;처럼 코드에 직접 쓰면(하드코딩), 밸런스를 조금 고칠 때마다 프로그래머가 코드를 수정하고 다시 컴파일해야 합니다. 아이템이 수백 개가 되면 이 방식은 유지가 불가능하고, 기획자는 수치 하나 바꾸는 데도 프로그래머를 거쳐야 합니다.

DataTable은 이런 반복 데이터를 행(Row) 단위의 표로 만든 에셋입니다. 표의 열(컬럼) 구성은 C++ 구조체 하나로 정의하고, 내용은 에디터에서 직접 편집하거나 CSV/JSON 파일로 임포트합니다. 결과: 수치 변경에 컴파일이 필요 없고, 기획자가 스프레드시트로 밸런싱한 것을 그대로 게임에 반영할 수 있습니다.

🔑

핵심 용어 — DataTable: 같은 구조의 행이 모인 표 형태의 에셋. · FTableRowBase: DataTable의 행이 되려면 반드시 상속해야 하는 기반 구조체. · RowName: 각 행을 식별하는 고유 이름(FName) — 표의 "기본 키". · CSV 임포트: 쉼표로 구분된 텍스트 파일(엑셀/구글시트에서 내보내기 가능)을 DataTable로 읽어 들이는 것.

02 — 구성 요소 관계

구조체 · 에셋 · 코드가 연결되는 방식

DataTable은 세 조각의 협업입니다: 구조체(형태 정의) → DataTable 에셋(내용 보관) → 조회 코드(사용).

⚠️

DataTable은 런타임 읽기 전용으로 쓰는 것이 원칙입니다. 플레이 중 값이 변해야 하는 데이터(현재 체력 등)는 DataTable이 아니라 액터·Subsystem의 변수이며, DataTable에는 그 초기값·정의만 둡니다.

03 — 데이터 보관 방식 선택

하드코딩 · DataTable · DataAsset, 언제 무엇을

DataTable

같은 구조 × 많은 행CSV 밸런싱

아이템 300개, 몬스터 80종처럼 동일한 형태가 반복되는 데이터. 스프레드시트에서 일괄 편집·임포트하는 워크플로우가 강점.

DataAsset

구조가 다른 낱개에셋 참조 중심

캐릭터 클래스 정의, 무기 설정처럼 하나하나가 개별 에셋인 데이터. 행 단위 표가 아니라 상속·에셋 참조가 필요할 때.

데이터 성격보관 방식왜
수백 행의 동일 구조 스펙 (아이템·몬스터·드롭률)UDataTable행 반복 구조 + CSV 일괄 편집. 이 문서의 주제
레벨·시간에 따라 연속으로 변하는 수치 곡선 (성장 곡선)UCurveTable / UCurveFloat이산 행이 아닌 보간 곡선이 필요 — 레벨 17.5의 값도 계산 가능
구조가 제각각인 설정 덩어리 (게임 모드 구성, 무기 정의)UPrimaryDataAsset표에 안 맞는 개별 에셋. 에셋 매니저로 비동기 로드 가능
거의 안 바뀌는 상수 한두 개코드 상수 / Config표를 만들 관리 비용이 더 큼 — 다만 밸런싱 대상이 되는 순간 DataTable로 승격
04 — 제작 · 임포트 절차

구조체 정의부터 조회까지 4단계

전제: C++ 프로젝트(구조체 정의용), 에디터 사용 가능. 각 단계의 끝에 확인해야 할 결과를 함께 적었습니다.

01
행 구조체 정의 (.h)
FTableRowBase를 상속한 USTRUCT(BlueprintType) 구조체를 만들고 컴파일합니다(05 섹션 코드). 결과 확인: 에디터 재시작 후 Row Type 목록에 구조체 이름이 나타남.
02
DataTable 에셋 생성
Content Browser에서 우클릭 ▸ Miscellaneous ▸ Data Table → "Pick Row Structure" 창에서 ItemRow 선택 → 이름을 DT_Items처럼 지정. 결과 확인: 더블클릭하면 표 편집기가 열림.
03
내용 채우기 — 에디터 또는 CSV
(방법 A) 표 편집기 상단 + Add 버튼으로 행 추가, RowName과 값을 직접 입력. (방법 B) 첫 열이 RowName인 CSV를 만들어 Content Browser로 드래그&드롭 → 임포트 창에서 Row Type 선택. 결과 확인: 행들이 표에 표시됨. CSV 재임포트는 에셋 우클릭 ▸ Reimport.
04
코드/BP에서 조회
C++은 FindRow<FItemRow>()(05 섹션), BP는 Get Data Table Row 노드 사용. 결과 확인: 존재하는 RowName으로 조회 시 값이, 없는 이름으로는 null/실패 핀이 반환됨.
⚠️
에디터 수정과 CSV 재임포트는 충돌합니다 — 표 편집기에서 고친 내용은 CSV 원본에 반영되지 않으므로, Reimport하면 에디터 수정분이 덮어써져 사라집니다. 팀에서 "진실은 CSV" 또는 "진실은 에셋" 중 하나를 정하고 한쪽으로만 편집하세요.
05 — 구현 스켈레톤

행 구조체 정의와 조회 코드

모듈 이름 MYGAME_API는 자기 프로젝트로 바꾸세요. 조회 예시는 임의 액터 클래스 안의 함수로 작성했습니다.

#pragma once

#include "CoreMinimal.h"
#include "Engine/DataTable.h"       // FTableRowBase가 여기 있음
#include "ItemRow.generated.h"       // 반드시 마지막 #include

// DataTable의 행 = 이 구조체 하나. 필드가 곧 표의 열이 됩니다.
USTRUCT(BlueprintType)
struct MYGAME_API FItemRow : public FTableRowBase
{
    GENERATED_BODY()

    // 화면 표시용 이름 (FText = 현지화 대상 문자열)
    UPROPERTY(EditAnywhere, BlueprintReadOnly, Category = "Item")
    FText DisplayName;

    UPROPERTY(EditAnywhere, BlueprintReadOnly, Category = "Item")
    int32 Damage = 0;

    // 아이콘은 소프트 참조: 표를 로드해도 텍스처까지 전부 로드되지 않게
    UPROPERTY(EditAnywhere, BlueprintReadOnly, Category = "Item")
    TSoftObjectPtr<UTexture2D> Icon;
};
#include "ItemRow.h"
#include "Engine/DataTable.h"

// 헤더에 선언해 둔 DataTable 참조.
// UPROPERTY(EditDefaultsOnly, Category="Item") UDataTable* ItemTable;
// → 에디터에서 BP 디폴트로 DT_Items 에셋을 지정해 둡니다.

int32 AMyShop::GetItemDamage(FName RowName) const
{
    if (!ItemTable) { return 0; }   // 테이블 미지정 방어

    // 두 번째 인자(ContextString)는 실패 로그에 찍힐 위치 표시용
    const FItemRow* Row =
        ItemTable->FindRow<FItemRow>(RowName, TEXT("GetItemDamage"));

    // RowName 오타·행 삭제 시 nullptr — 반드시 체크
    if (!Row)
    {
        UE_LOG(LogTemp, Warning, TEXT("Item row not found: %s"), *RowName.ToString());
        return 0;
    }
    return Row->Damage;
}

// 전체 행 순회가 필요할 때 (도감·상점 목록 등)
void AMyShop::ListAllItems() const
{
    if (!ItemTable) { return; }

    TArray<FItemRow*> Rows;
    ItemTable->GetAllRows<FItemRow>(TEXT("ListAllItems"), Rows);

    for (const FItemRow* Row : Rows)
    {
        UE_LOG(LogTemp, Log, TEXT("%s (Damage %d)"), *Row->DisplayName.ToString(), Row->Damage);
    }
}
06 — 확장 & 주의

더 나아가기와 흔한 함정

📈
성장 곡선
UCurveTable
레벨별 경험치처럼 연속 수치는 CurveTable로 — 중간 값을 보간으로 얻습니다.
🧩
테이블 합성
CompositeDataTable
기본 표 + DLC/시즌 표를 겹쳐 하나처럼 조회. 원본을 건드리지 않는 확장.
🗂️
행 핸들 참조
FDataTableRowHandle
"어떤 표의 어떤 행"을 UPROPERTY로 지정 — 에디터에서 드롭다운으로 행을 고를 수 있습니다.
⚠️
RowName 오타는 컴파일러가 못 잡습니다 — FindRow는 이름이 틀리면 조용히 nullptr를 반환합니다. 반환값 체크를 생략하면 크래시가 "표를 고친 다음 날" 터집니다. 조회 함수를 한 곳에 모으고 실패 로그를 남기세요(05 섹션 패턴).
⚠️
구조체 필드를 바꾸면 기존 데이터 확인 — 필드 이름 변경·삭제 후에는 에셋을 열어 값이 올바로 남았는지 확인하고, CSV가 진실인 팀은 CSV 헤더도 함께 고쳐 Reimport하세요. 헤더와 필드 이름이 다르면 그 열은 기본값으로 채워집니다.
💡
큰 에셋은 소프트 참조로 — 행에 텍스처·메시를 하드 참조(UTexture2D*)로 넣으면 표를 로드할 때 전부 함께 로드됩니다. TSoftObjectPtr로 선언하고 필요할 때만 로드하세요.
