툰 셰이딩 & 아웃라인 렌더링
Toon Shading & Outline

캐릭터·배경을 만화(애니메이션)처럼 보이게 하는 그래픽 스타일의 원리를 다룹니다. 이 스타일은 사실 두 개의 독립된 기법의 조합입니다 — 음영을 계단처럼 끊는 셀 셰이딩과, 형태 가장자리에 선을 긋는 아웃라인 렌더링. 각 기법이 무엇이고, 언리얼 엔진 어디에 개입하며, 어떤 절차로 구현하는지를 정리합니다.

렌더링 기법 UE5 기준 Material · Post Process
01 — 개념 · 스타일과 기법 구분

"만화처럼 보인다"를 분해하면 두 가지 기법이 나온다

젤다의 전설: 야생의 숨결, 원신, 길티기어 시리즈의 화면을 떠올려 보세요. 공통점은 두 가지입니다. 첫째, 빛을 받는 면과 그늘진 면이 부드럽게 이어지지 않고 경계가 뚜렷한 2~4단계로 나뉩니다. 둘째, 캐릭터와 사물의 가장자리에 펜으로 그린 듯한 선이 있습니다. 이 둘은 별개의 기법이며, 하나만 쓸 수도, 둘 다 쓸 수도 있습니다.

이런 비사실적 표현 전체를 묶는 상위 개념이 NPR(Non-Photorealistic Rendering, 비사실적 렌더링)입니다 — 사진처럼 그리는 것(포토리얼)이 아니라 그림·만화·수채화처럼 그리는 렌더링의 총칭입니다. 그 아래에서, 음영을 애니메이션 셀화(셀룰로이드에 그리던 원화)처럼 단계로 끊어 칠하는 기법이 셀 셰이딩(Cel Shading)이고, 흔히 툰 셰이딩(Toon Shading)이라고도 부릅니다(같은 것의 다른 이름 — 이 문서에서는 셀 셰이딩으로 통일합니다). 그리고 물체의 가장자리를 검출해 선을 입히는 기법이 아웃라인 렌더링(Outline Rendering)입니다.

용어 하나만 더 준비합시다. 셰이더(Shader)는 "각 픽셀(또는 각 정점)을 무슨 색으로, 어느 위치에 그릴지"를 계산하는 GPU용 작은 프로그램입니다. 언리얼에서는 셰이더를 직접 파일로 쓰는 대신 머티리얼(Material) 노드 그래프로 만들면 엔진이 셰이더로 컴파일해 줍니다. 즉 이 문서의 구현은 대부분 "머티리얼을 어떻게 조립하는가"의 이야기입니다.

🔑

핵심 용어 — NPR: 비사실적 렌더링의 총칭. · 셀 셰이딩(=툰 셰이딩): 음영을 2~4단계 계단으로 끊는 기법. · 아웃라인 렌더링: 가장자리에 선을 긋는 기법. · 셰이더: 픽셀/정점의 색·위치를 계산하는 GPU 프로그램. · 머티리얼: 언리얼에서 셰이더를 노드로 조립하는 에셋. · 포스트 프로세스(Post Process): 3D 장면을 다 그린 뒤, 완성된 "화면 이미지"에 가하는 후처리 단계.

왜 구분이 중요한가 — "툰 셰이더 만들기"를 한 덩어리로 검색하면 자료마다 다른 절반을 설명해서 혼란스럽습니다. "지금 나는 음영을 만드는 중인가, 선을 만드는 중인가"를 먼저 정하면 필요한 자료와 노드가 명확해집니다.

02 — 기법 지도

스타일은 조합이다 — 음영 축과 외곽선 축

툰 스타일 화면은 아래처럼 음영 기법 하나 + 외곽선 기법 하나를 골라 조합한 결과입니다. 화살표를 따라가며 "내가 만들려는 그림은 어느 조합인가"를 먼저 정하세요.

인버티드 헐(Inverted Hull) — "뒤집힌 껍데기"라는 뜻입니다. 원본 모델을 정점 노멀(각 정점이 바라보는 바깥 방향) 쪽으로 조금 밀어 부풀린 복제 메시를 만들고, 면의 앞뒤를 뒤집어(컬링 반전) 안쪽 면만 보이게 하면 — 원본이 복제본 대부분을 가리고, 가장자리로 부풀린 만큼만 삐져나와 테두리 선처럼 보입니다. GPU 입장에선 그냥 메시 하나를 더 그리는 것이라 포스트 프로세스가 없는 환경(모바일 등)에서도 동작합니다.

03 — 기법 선택 기준

내 프로젝트엔 어느 쪽인가

인버티드 헐 (메시 기반)

부위별 두께 제어모바일 OK메시 2배 렌더

선 두께를 모델별·부위별로 다르게 줄 수 있어(정점 컬러로 두께 마스킹) 캐릭터가 주인공인 게임에서 강합니다. 길티기어 시리즈가 이 방식을 극한까지 다듬은 대표 사례. 대신 외곽선 대상마다 메시를 한 번 더 그리므로 대상이 많으면 드로우 비용이 늘고, 안쪽 디테일 선(옷 주름 등)은 만들지 못합니다(실루엣 바깥 테두리만).

포스트 프로세스 엣지 검출 (화면 기반)

화면 전체 일괄안쪽 경계도 검출두께 개별 제어 어려움

머티리얼 하나로 화면의 모든 대상(배경 포함)에 선이 생기고, 깊이·노멀이 급변하는 곳이면 실루엣 안쪽 경계(팔과 몸 사이 등)도 검출됩니다. 대신 선 두께가 화면 픽셀 기준이라 오브젝트별로 다르게 주기 어렵고, 포스트 프로세스 비용이 화면 해상도에 비례합니다.

상황선택왜
배경 포함 화면 전체에 균일한 만화 룩포스트 프로세스 엣지 검출머티리얼 1개 + 볼륨 1개로 전 화면 커버. 오브젝트를 하나하나 만질 필요 없음
캐릭터 몇 명만 두껍고 개성 있는 선 (격투·캐릭터 중심)인버티드 헐부위별 두께·색 제어가 되는 유일한 방식. 대상이 소수라 메시 2배 비용도 감당 가능
특정 오브젝트에만 선 (선택 강조·조준 대상 표시)Custom Stencil + 포스트 프로세스Custom Stencil 값으로 "이 값을 가진 픽셀에만" 선을 긋도록 필터링 가능 (04 참조)
모바일 등 포스트 프로세스 비용이 부담인 플랫폼인버티드 헐일반 메시 렌더링만 쓰므로 후처리 파이프라인 부담이 없음
음영만 만화처럼, 선은 없는 스타일 (젤다 야숨 계열)셀 셰이딩 단독두 축은 독립 — 외곽선 없이 음영 단계화만 적용해도 성립
💡

실무 조합 — 상용 툰 스타일 게임은 대부분 혼합합니다: 주요 캐릭터는 인버티드 헐(굵고 통제된 선), 화면 전체는 약한 포스트 프로세스 선(배경 디테일). 처음 시도라면 포스트 프로세스부터 — 결과가 즉시 화면 전체에 보여 학습이 빠릅니다.

04 — 렌더링 파이프라인 개입 지점

한 프레임이 그려지는 동안 어디에 끼어드는가

전제가 되는 개념 둘을 먼저 풀겠습니다. Custom Depth는 언리얼이 제공하는 별도의 깊이 기록장입니다 — "Render CustomDepth Pass"를 체크한 메시만 자기 깊이(카메라로부터의 거리)를 여기에 따로 한 번 더 기록합니다. Custom Stencil은 그 기록에 덧붙이는 0~255 정수 도장으로, "이 픽셀은 3번 그룹" 같은 구분을 남깁니다. 포스트 프로세스 머티리얼은 이 기록장을 읽어 "선을 그릴 대상"만 골라낼 수 있습니다.

01
씬 렌더링 — 기본 화면 완성
모든 메시가 각자의 머티리얼로 그려져 화면 색(Scene Color)과 깊이(Scene Depth)가 만들어집니다. 인버티드 헐은 이 단계에서 끝납니다 — 부풀린 복제 메시도 그냥 메시라서 여기서 함께 그려지기 때문입니다.
02
Custom Depth 패스 — 대상만 별도 기록
"Render CustomDepth Pass"가 체크된 메시만 Custom Depth(+Stencil 값)에 기록됩니다. 화면에는 아무 변화가 없습니다 — 다음 단계가 읽을 데이터를 준비할 뿐입니다.
03
포스트 프로세스 단계 진입
3D는 끝났고 이제 화면은 한 장의 이미지입니다. 톤매핑·블룸 같은 엔진 후처리 사이에, Material Domain = Post Process로 만든 우리의 머티리얼이 끼어듭니다(끼어드는 위치는 Blendable Location 설정).
04
엣지 검출 — 이웃 픽셀과 깊이 비교
각 픽셀에서 상하좌우 이웃의 깊이(또는 Custom Depth)를 읽어 차이를 계산합니다. 차이가 문턱값(Threshold)을 넘으면 "여기는 물체의 경계"라고 판정합니다. 이웃을 읽는 도구가 SceneTexture 노드입니다.
05
합성 — 경계 픽셀에 선 색 입히기
경계로 판정된 픽셀은 선 색(보통 검정)으로, 나머지는 원래 화면 색으로 출력합니다. 셀 셰이딩(밝기 양자화)도 같은 자리에서 함께 처리할 수 있습니다.
⚠️
반투명(Translucent) 머티리얼은 Custom Depth에 기록되지 않습니다. 유리·이펙트류에 선이 안 생기는 건 버그가 아니라 파이프라인 구조입니다. 반투명 대상에 선이 꼭 필요하면 인버티드 헐로 우회하세요.
05 — 구현 절차 (스켈레톤)

세 가지 레시피 — 따라 하는 순서

전제: UE5 기본 프로젝트(디퍼드 렌더링 기본값) 기준이며, 코드는 뼈대 수준입니다 — 수치(두께·문턱값·단계 수)는 자기 프로젝트에서 조정하는 것이 전제입니다.

레시피 A — 포스트 프로세스 아웃라인 (권장 시작점)

A1
프로젝트 설정에서 Custom Depth-Stencil 켜기
Edit ▸ Project Settings ▸ Engine ▸ Rendering ▸ Postprocessing ▸ Custom Depth-Stencil Pass 를 Enabled with Stencil로 설정합니다. (기본값 Enabled는 Stencil 없이 Depth만 — 그룹 구분이 필요 없으면 Enabled로도 충분합니다.)
A2
선을 두를 메시에 Custom Depth 체크
레벨에서 대상 액터 선택 → Details 패널 → Rendering ▸ Render CustomDepth Pass 체크. 그룹 구분이 필요하면 CustomDepth Stencil Value에 1~255 값을 지정합니다. (화면 전체에 선을 긋는 스타일이면 이 단계를 건너뛰고 A4에서 Scene Depth를 씁니다.)
A3
포스트 프로세스 머티리얼 생성
Content Browser 우클릭 → Material 생성(이름 예: M_PP_Outline) → 머티리얼을 열고 Details에서 Material Domain = Post Process로 변경. Blendable Location은 우선 Scene Color After Tonemapping(비용 저렴)으로 두고, 색이 이상하면 Before Tonemapping으로 바꿔 비교합니다.
A4
엣지 검출 노드 조립
머티리얼 그래프에 SceneTexture 노드를 놓고 Scene Texture Id를 CustomDepth(A2를 했다면) 또는 SceneDepth(화면 전체)로 지정 → 현재 UV와 상하좌우로 1픽셀씩 밀린 UV, 총 5번 샘플 → 중심과 이웃의 깊이 차의 최대값이 문턱값을 넘으면 1(경계), 아니면 0. 노드가 많아지므로 아래처럼 Custom 노드(HLSL 직접 입력)로 묶는 것이 편합니다.
A5
Post Process Volume에 등록
레벨에 Post Process Volume 배치 → Details ▸ Post Process Volume Settings ▸ Infinite Extent (Unbound) 체크(맵 전체 적용) → Rendering Features ▸ Post Process Materials 배열에 항목 추가 → Asset Reference로 M_PP_Outline 지정. 뷰포트에 즉시 선이 보이면 성공입니다.
// Custom 노드 (Output Type: CMOT Float1)
// 입력 핀: UV (float2) · Thickness (float, 선 두께 px) · Threshold (float, 깊이 차 문턱값)
// 중심 픽셀과 상하좌우 이웃의 CustomDepth를 비교해 경계(1) / 비경계(0)를 반환한다.

float2 TexelSize = View.ViewSizeAndInvSize.zw;      // 화면 1픽셀의 UV 크기
float  Center    = CalcSceneCustomDepth(UV);        // 중심 깊이 (SceneDepth를 쓰려면 CalcSceneDepth)

float2 Dir[4] = { float2( 1, 0), float2(-1, 0),   // 우 · 좌
                  float2( 0, 1), float2( 0,-1) };  // 하 · 상

float MaxDiff = 0;   // 이웃과의 깊이 차 최대값 누적용 — 0에서 시작해 아래 루프에서 갱신
for (int i = 0; i < 4; ++i)   // 4방향 이웃을 차례로 검사 (중심 1 + 이웃 4 = 총 5회 샘플)
{
    // 1픽셀 크기 × Thickness만큼 밀린 지점의 깊이 — Thickness가 클수록 더 먼 픽셀과 비교되어 선이 굵어짐
    float Neighbor = CalcSceneCustomDepth(UV + Dir[i] * TexelSize * Thickness);
    MaxDiff = max(MaxDiff, abs(Center - Neighbor));  // 깊이 차가 가장 큰 방향을 채택
}

// 깊이 차가 문턱값보다 크면 경계 → 1. 결과를 Lerp의 Alpha로 써서
// (원래 화면색 ↔ 선 색)을 섞으면 최종 출력이 된다.
return MaxDiff > Threshold ? 1 : 0;
🧭

마지막 연결 — Custom 노드의 출력(0/1)을 Lerp의 Alpha에, A핀에는 SceneTexture:PostProcessInput0(원래 화면색), B핀에는 선 색(Constant3Vector, 검정)을 연결하고 Lerp 결과를 Emissive Color에 꽂으면 완성입니다. 포스트 프로세스 머티리얼의 출력 핀은 Emissive Color 하나만 사용합니다.

레시피 B — 인버티드 헐 (캐릭터 정밀 선)

B1
외곽선 머티리얼 생성
Material 생성(예: M_Outline_Hull) → Shading Model = Unlit(조명 무시, 선은 늘 같은 색), Emissive에 선 색(검정) 연결.
B2
정점을 노멀 방향으로 부풀리기
같은 머티리얼에서 VertexNormalWS(월드 공간 정점 노멀) × 두께 스칼라(예: 1.5)를 World Position Offset 핀에 연결. 이 머티리얼을 쓰는 메시는 겉면 방향으로 그만큼 부풀어 그려집니다.
B3
복제 메시에 씌우고 컬링 뒤집기
캐릭터 블루프린트에 같은 메시의 컴포넌트를 하나 더 추가(Skeletal이면 Leader Pose 연결로 애니메이션 동기화) → 머티리얼을 M_Outline_Hull로 교체 → Details ▸ Rendering ▸ Reverse Culling 체크. 앞면이 숨고 뒷면(안쪽)만 보이게 되어, 원본 뒤로 삐져나온 부분만 테두리로 남습니다.
B4
두께 다듬기
두께 스칼라를 Material Parameter로 승격해 인스턴스에서 조절합니다. 부위별로 선 굵기를 다르게 하려면 메시의 정점 컬러를 두께에 곱하세요(머리카락 얇게, 실루엣 굵게 — 길티기어 방식의 핵심).

레시피 C — 셀 셰이딩 음영 단계화 (최소 체험판)

정석은 라이팅 결과(디퓨즈)를 분리해 단계화하는 것이지만 설정이 깁니다. 아래는 화면 밝기를 통째로 계단화하는 최소 스켈레톤으로, 레시피 A의 포스트 프로세스 머티리얼에 그대로 추가할 수 있습니다 — "계단 음영이 이런 원리구나"를 체험하는 용도입니다.

// Custom 노드 (Output Type: CMOT Float3)
// 입력 핀: UV (float2) · Steps (float, 음영 단계 수 — 3~4 권장)
// 화면색의 밝기(휘도)만 N단계로 양자화하고 색조는 유지한다.

float3 Color = SceneTextureLookup(UV, 14, false).rgb;      // 14 = PostProcessInput0 (완성된 화면색)
float  Lum   = dot(Color, float3(0.299, 0.587, 0.114)); // 휘도(사람 눈 기준 밝기)

// floor로 연속 밝기(0~1)를 정수 단계로 끊고, (Steps-1)로 나눠 다시 0~1 범위로 정규화
float Band = floor(Lum * Steps) / (Steps - 1);            // 부드러운 밝기 → 계단 밝기

// 원래 색에서 밝기만 계단 값으로 치환 (0 나눗셈 방지용 max)
return Color * (Band / max(Lum, 0.0001));
⚠️

이 스니펫의 한계 — 화면 전체 밝기를 계단화하므로 UI 밖 모든 것(하늘·이펙트 포함)이 영향을 받고, 빛 방향과 무관하게 동작합니다. 프로젝트 수준의 셀 셰이딩은 디퓨즈(빛 계산 결과)만 분리해 단계화하는 방식으로 발전시켜야 하며, 그 확장 방향은 06 섹션을 참고하세요.

06 — 확장 & 주의

더 좋은 선, 더 좋은 음영으로 갈 때

📐
노멀 기반 엣지 추가
Depth + Normal Edge
깊이 차만으로는 못 잡는 경계(같은 거리의 꺾인 면)를 SceneTexture:WorldNormal 비교로 보강. 깊이 검출과 OR로 합치면 선 품질이 한 단계 오릅니다.
🎨
디퓨즈 분리 셀 셰이딩
Ramp / Curve Atlas
화면 밝기 대신 라이팅 결과를 커브(램프 텍스처·Curve Atlas)로 매핑하는 정석 방식. 그림자 경계 위치·색까지 아트 주도로 제어할 수 있습니다.
📏
선 두께 거리 보정
Depth-scaled Thickness
픽셀 고정 두께는 멀리 있는 물체에서 상대적으로 두꺼워 보입니다. Thickness를 깊이에 반비례시키면 원근감 있는 일정한 선이 됩니다.
⚠️
TAA·업스케일러가 선을 뭉갭니다 — Temporal AA와 TSR/DLSS는 프레임 간 픽셀을 섞기 때문에 1px 선이 흐려지거나 깜빡일 수 있습니다. 선이 지저분하면 AA 방식을 바꿔 비교하거나(Project Settings ▸ Anti-Aliasing Method), 선 두께를 2px 이상으로 키우세요.
⚠️
인버티드 헐 + 하드 엣지 모델 — 정점 노멀이 갈라진 하드 엣지(각진 모서리)는 부풀릴 때 테두리가 찢어집니다. 아웃라인용으로는 노멀을 부드럽게 평균 낸 값(스무스 노멀)을 별도 UV 채널이나 정점 컬러에 구워 쓰는 것이 정석입니다.
⚠️
Custom Depth를 켜면 공짜가 아닙니다 — 체크된 메시는 깊이 패스를 한 번 더 그립니다. "일단 전부 체크"하지 말고 선이 필요한 대상만 켜세요. 화면 전체 선이 목적이면 애초에 Scene Depth 기반(A2 생략)이 더 쌉니다.
💡
디버깅 팁 — 엣지 검출이 안 보일 때는 Custom 노드 출력을 Lerp에 넣기 전에 그대로 Emissive에 연결해 보세요. 경계가 흰색으로 표시되는 마스크 화면이 나오므로 문턱값·두께 문제인지, 등록(A5) 문제인지 즉시 구분됩니다.
