언리얼 리플렉션 매크로

순수 C++에는 없는 "자기 자신을 들여다보는 능력"을 언리얼이 어떻게 구현하는지, 6개 매크로(UCLASS·USTRUCT·UENUM·UINTERFACE·UPROPERTY·UFUNCTION)의 역할과 UHT 빌드 파이프라인을 설명합니다.

02-A 아키텍처 패턴 6대 매크로 UHT 파이프라인
01 — 개념 이해

리플렉션 — 프로그램이 자기 자신을 들여다보는 능력

리플렉션(Reflection)이란, 프로그램이 실행 중에 "내가 어떤 클래스를 가지고 있고, 그 클래스에 어떤 변수와 함수가 있는지"를 스스로 조회할 수 있는 능력입니다. 비유하자면, 거울을 보고 자기 모습을 확인하는 것과 같습니다.

Java나 C#은 언어 차원에서 리플렉션을 지원하지만, 순수 C++에는 리플렉션이 없습니다. C++ 컴파일러는 코드를 기계어로 바꾸는 순간 변수 이름이나 클래스 구조 정보를 버립니다. 그래서 런타임에 "이 객체에 Health라는 변수가 있나?"라고 물을 수 없습니다.

언리얼 엔진은 이 문제를 매크로 + 코드 생성 도구(UHT)로 해결합니다. 개발자가 클래스·변수·함수 위에 특수 매크로를 붙이면, 빌드 전에 UHT(Unreal Header Tool)가 헤더를 스캔해서 구조 정보를 담은 코드(.generated.h)를 자동 생성합니다. 이 정보 덕분에 언리얼 엔진은 런타임에 클래스 구조를 조회할 수 있습니다.

📖

핵심 용어 블록
리플렉션(Reflection) — 프로그램이 자기 타입·변수·함수 정보를 런타임에 조회하는 능력
UHT(Unreal Header Tool) — 빌드 전에 헤더를 스캔해 리플렉션 코드를 자동 생성하는 도구
.generated.h — UHT가 생성하는 헤더 파일. 리플렉션·직렬화·블루프린트 바인딩 코드가 들어감
GENERATED_BODY() — 클래스/구조체 본문에 넣는 매크로. generated.h의 코드를 클래스 안에 삽입
지정자(Specifier) — 매크로 괄호 안에 쓰는 키워드. 에디터 노출·블루프린트 접근 등 동작을 제어
직렬화(Serialization) — 객체를 바이트 스트림으로 변환해 저장/전송하는 것
가비지 컬렉션(GC) — 참조 없는 UObject를 자동으로 메모리에서 해제하는 시스템

왜 리플렉션이 필요한가 — 4가지 핵심 기능

리플렉션이 없으면 언리얼 엔진의 핵심 기능 4가지가 동작하지 않습니다. 매크로를 붙이는 이유가 바로 이것입니다.

BP
블루프린트 노출
C++ 변수·함수를 BP에서 사용
ED

디테일 패널에서 값 수정
GC

참조 추적 → 자동 메모리 해제
NET
네트워크 복제

순수 C++ vs 언리얼 C++ — 무엇이 달라지나

순수 C++

컴파일 후 변수 이름·클래스 구조 정보가 사라집니다. 런타임에 "이 객체에 어떤 멤버가 있는지" 알 수 없습니다.

런타임 타입 조회 불가 에디터 노출 불가

언리얼 C++ (매크로 사용)

매크로를 붙인 클래스·변수·함수의 정보를 UHT가 코드로 남깁니다. 런타임에도 구조를 조회할 수 있습니다.

런타임 타입 조회 BP·에디터·GC·복제
💡
매크로를 붙이지 않은 C++ 코드는 순수 C++과 동일하게 동작합니다. 리플렉션이 필요 없는 내부 유틸리티 함수나 헬퍼 구조체에는 매크로를 붙이지 않아도 됩니다. 매크로는 "엔진에 알려야 하는 것"에만 붙입니다.

6대 매크로 — 타입 등록 4개 + 멤버 등록 2개

리플렉션 매크로는 총 6개입니다. 역할에 따라 두 계층으로 나뉩니다.

매크로가 코드에서 붙는 위치

타입 매크로는 클래스/구조체/열거형 선언 바로 위에, 멤버 매크로는 변수/함수 선언 바로 위에 붙입니다. 아래 트리로 시각적으로 확인하세요.

MyCharacter.h ├── UCLASS() ← 클래스 선언 위 │ class AMyCharacter : public ACharacter │ ├── UPROPERTY() ← 변수 선언 위 │ │ float Health; │ ├── UPROPERTY() ← 변수 선언 위 │ │ int32 Level; │ ├── UFUNCTION() ← 함수 선언 위 │ │ void TakeDamage(float Amount); │ └── void InternalHelper(); ← 매크로 없음 = 리플렉션 미등록 │ ├── USTRUCT() ← 구조체 선언 위 │ struct FCharacterStats │ └── UPROPERTY() ← 구조체 멤버도 동일 │ float MaxHealth; │ ├── UENUM() ← 열거형 선언 위 │ enum class ECharacterState : uint8 │ └── UINTERFACE() ← 인터페이스 선언 위 class UDamageable / IDamageable

타입 매크로와 멤버 매크로의 관계

멤버 매크로는 타입 매크로가 붙은 타입 안에서만 의미가 있습니다. UCLASS()가 붙지 않은 순수 C++ 클래스 안에서 UPROPERTY()를 쓰면 컴파일 에러가 발생합니다.

UCLASS() / USTRUCT()
타입 자체를 엔진에 등록 → GC·CDO·BP 사용 가능
안에 포함
─────→
UPROPERTY() / UFUNCTION()
멤버를 엔진에 등록 → 에디터 노출·BP 호출·직렬화·복제
UENUM()
열거형을 엔진에 등록 → BP에서 드롭다운으로 사용
독립 사용
─────→
UPROPERTY() 타입으로 사용
UENUM으로 등록된 열거형을 UPROPERTY의 타입으로 지정
⚠️
UENUM과 UINTERFACE는 멤버 매크로를 포함하지 않습니다. UENUM은 값 나열만 하고, UINTERFACE는 순수 가상 함수만 선언합니다(구현은 UCLASS가 담당).
03 — 책임

매크로별 역할과 주요 지정자

지정자(Specifier)(앞서 핵심 용어 블록에서 설명)는 매크로 괄호 안에 쓰는 키워드입니다. 지정자를 통해 "이 변수를 에디터에서 편집 가능하게 할지", "이 함수를 블루프린트에서 호출 가능하게 할지" 등을 세밀하게 제어합니다. 형식은 UPROPERTY(지정자1, 지정자2)입니다.

각 매크로의 표 아래에는 표의 지정자들이 실제 헤더 파일에서 쓰이는 모습을 예시 코드로 붙였습니다. 예시는 지정자가 붙는 위치와 모양에 집중한 조각 코드라서 공통 준비물(#pragma once, #include, .generated.h)은 생략했습니다 — 빌드까지 되는 완전한 .h/.cpp 파일 형태는 아래 05 — 스켈레톤 섹션에 있습니다.

UCLASS() — 클래스를 엔진에 등록

UCLASS()는 C++ 클래스를 언리얼 리플렉션 시스템에 등록합니다. 등록된 클래스는 GC(가비지 컬렉션) 대상이 되고, CDO(Class Default Object)가 자동 생성되며, 블루프린트에서 사용할 수 있게 됩니다.

지정자효과쓰는 상황
Blueprintable이 클래스를 부모로 BP 생성 가능BP에서 상속받아 확장해야 할 때
BlueprintType이 클래스를 BP 변수 타입으로 사용 가능BP에서 이 클래스의 포인터를 변수로 쓸 때
Abstract인스턴스 생성 금지 (상속 전용)반드시 자식 클래스로만 사용할 때
NotBlueprintableBP에서 상속 금지C++ 전용 내부 클래스
config=Gameini 파일에서 설정값 로드게임 설정을 외부 파일로 관리할 때
// ── Blueprintable + BlueprintType ─────────────────────────
// BP에서 상속(자식 BP 생성)도 되고, BP 변수 타입으로도 쓸 수 있다
UCLASS(Blueprintable, BlueprintType)
class AWeaponBase : public AActor
{
    GENERATED_BODY()
};

// ── Abstract ──────────────────────────────────────────────
// 이 클래스 자체는 레벨 배치·스폰 불가 — 자식 클래스 전용 부모
UCLASS(Abstract, Blueprintable)
class AEnemyBase : public ACharacter
{
    GENERATED_BODY()
};

// ── NotBlueprintable ──────────────────────────────────────
// BP 상속 금지 — C++ 내부 전용 클래스
UCLASS(NotBlueprintable)
class UScoreManager : public UObject
{
    GENERATED_BODY()
};

// ── config=Game ───────────────────────────────────────────
// Config/DefaultGame.ini 에서 값을 읽어온다
// (읽어올 변수 쪽에도 Config 지정자를 함께 붙여야 로드됨)
UCLASS(config=Game)
class UMySettings : public UObject
{
    GENERATED_BODY()

    UPROPERTY(Config)
    float MasterVolume;
};

UPROPERTY() — 멤버 변수를 엔진에 등록

UPROPERTY()는 멤버 변수를 리플렉션에 등록합니다. 등록하면 에디터 디테일 패널에 표시되거나, 블루프린트에서 읽고 쓸 수 있거나, 네트워크 복제 대상이 됩니다. 가장 자주 쓰는 매크로입니다.

지정자효과쓰는 상황
EditAnywhere에디터 디테일 패널에서 편집 가능레벨 디자이너가 인스턴스별 값을 조정
EditDefaultsOnlyBP 기본값(CDO)에서만 편집, 인스턴스에서 잠금초기 수치만 설정하고 런타임에 변경 불가
VisibleAnywhere에디터에서 보이지만 편집 불가컴포넌트 포인터 등 읽기 전용 확인용
BlueprintReadOnlyBP에서 읽기만 가능BP가 값을 확인만 해야 할 때
BlueprintReadWriteBP에서 읽기+쓰기 가능BP에서 값을 변경해야 할 때
Replicated서버 → 클라이언트 네트워크 복제멀티플레이에서 모든 클라이언트에 값 동기화
Transient저장(직렬화)에서 제외런타임에만 쓰는 임시 데이터
// (UCLASS가 붙은 클래스 본문 안이라고 가정)
public:
    // EditAnywhere — 레벨에 배치된 인스턴스마다 다른 값 입력 가능
    UPROPERTY(EditAnywhere, Category = "Stats")
    float MaxHealth = 100.0f;

    // EditDefaultsOnly — BP 에디터(기본값)에서만 편집,
    // 레벨에 배치된 인스턴스에서는 잠김
    UPROPERTY(EditDefaultsOnly, Category = "Stats")
    float BaseDamage = 10.0f;

    // VisibleAnywhere — 디테일 패널에 보이지만 편집 불가
    // (컴포넌트 포인터를 확인용으로 노출할 때 단골)
    UPROPERTY(VisibleAnywhere, Category = "Components")
    UStaticMeshComponent* MeshComp;

    // BlueprintReadOnly — BP 그래프에서 Get 노드만 생성됨
    UPROPERTY(VisibleAnywhere, BlueprintReadOnly, Category = "Stats")
    float CurrentHealth;

    // BlueprintReadWrite — BP 그래프에서 Get + Set 노드 모두 생성됨
    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Stats")
    float MoveSpeed = 600.0f;

    // Replicated — 서버에서 바뀐 값이 클라이언트로 자동 동기화
    UPROPERTY(Replicated)
    int32 Score;

    // Transient — 저장(직렬화) 대상에서 제외되는 런타임 임시값
    UPROPERTY(Transient)
    float TimeSinceSpawn;
⚠️
EditAnywhere와 BlueprintReadWrite는 별개입니다. EditAnywhere는 "에디터 디테일 패널에서 편집", BlueprintReadWrite는 "BP 노드에서 읽기+쓰기"입니다. 둘 다 필요하면 UPROPERTY(EditAnywhere, BlueprintReadWrite)처럼 같이 씁니다.

UFUNCTION() — 멤버 함수를 엔진에 등록

UFUNCTION()은 멤버 함수를 리플렉션에 등록합니다. 블루프린트에서 호출 가능하게 하거나, 네트워크 RPC(원격 프로시저 호출)로 만들거나, BP에서 오버라이드하게 할 수 있습니다.

지정자효과쓰는 상황
BlueprintCallableBP에서 이 함수를 노드로 호출 가능C++ 로직을 BP에서 사용할 때 (가장 흔함)
BlueprintPure실행 핀 없이 값만 반환하는 BP 노드계산 함수, Getter 등 부작용 없는 함수
BlueprintImplementableEventC++에서 선언만, 구현은 BP에서BP 디자이너가 로직을 채워야 할 때
BlueprintNativeEventC++ 기본 구현 + BP 오버라이드 가능기본 동작은 C++로 두되 BP에서 변경 허용
Server서버에서만 실행되는 RPC클라이언트가 서버에 요청을 보낼 때
Client소유 클라이언트에서만 실행되는 RPC서버가 특정 클라이언트에 결과를 보낼 때
// (UCLASS가 붙은 클래스 본문 안이라고 가정)
public:
    // BlueprintCallable — BP에서 실행 핀이 있는 노드로 호출
    UFUNCTION(BlueprintCallable, Category = "Combat")
    void Attack(AActor* Target);

    // BlueprintPure — 실행 핀 없이 값만 반환 (Getter·계산 함수)
    UFUNCTION(BlueprintPure, Category = "Stats")
    bool IsAlive() const;

    // BlueprintImplementableEvent — 선언만 C++, 본문은 BP에서 작성
    // .cpp에 구현을 쓰지 않는다 (쓰면 중복 정의 에러)
    UFUNCTION(BlueprintImplementableEvent, Category = "FX")
    void OnHitEffect();

    // BlueprintNativeEvent — C++ 기본 구현 + BP 오버라이드 허용
    // C++ 구현부는 함수명 뒤에 _Implementation 을 붙여 작성한다
    UFUNCTION(BlueprintNativeEvent, Category = "Combat")
    void OnDeath();
    virtual void OnDeath_Implementation();  // ← .cpp에 구현할 실제 함수

    // Server — 클라이언트가 호출하면 서버에서 실행되는 RPC
    // Reliable = 전달 보장. 구현부 이름은 ServerFire_Implementation
    UFUNCTION(Server, Reliable)
    void ServerFire();

    // Client — 서버가 호출하면 소유 클라이언트에서 실행되는 RPC
    UFUNCTION(Client, Reliable)
    void ClientShowHitMarker();

USTRUCT() — 구조체를 엔진에 등록

USTRUCT()는 구조체(struct)를 리플렉션에 등록합니다. UCLASS와 달리 GC 대상이 아니고, 값 타입(복사·이동)으로 동작합니다. 데이터 묶음(HP·레벨·경험치를 한 덩어리로)을 만들 때 주로 씁니다.

지정자효과
BlueprintTypeBP에서 이 구조체를 변수 타입으로 사용 가능
Atomic직렬화 시 멤버 단위가 아닌 구조체 전체를 한 번에 처리
// BlueprintType — BP 변수 타입으로 사용 가능,
// BP 그래프에 Make/Break 노드가 자동 생성된다
USTRUCT(BlueprintType)
struct FItemData
{
    GENERATED_BODY()

    // 구조체 멤버도 UPROPERTY를 붙여야 BP·에디터에 노출된다
    UPROPERTY(EditAnywhere, BlueprintReadWrite)
    FName ItemID;

    UPROPERTY(EditAnywhere, BlueprintReadWrite)
    int32 Count = 1;
};

// Atomic — 직렬화할 때 멤버 하나씩이 아니라 구조체 전체를 한 덩어리로 처리
USTRUCT(Atomic, BlueprintType)
struct FDamageInfo
{
    GENERATED_BODY()

    UPROPERTY(EditAnywhere)
    float Amount = 0.0f;

    UPROPERTY(EditAnywhere)
    bool bIsCritical = false;
};
💡

USTRUCT의 접두사는 F입니다. 예: FCharacterStats, FTimerHandle. UCLASS의 접두사(A/U)와 다르니 주의하세요.

UENUM() — 열거형을 엔진에 등록

UENUM()은 열거형(enum)을 리플렉션에 등록합니다. 등록하면 BP에서 드롭다운 메뉴로 값을 선택할 수 있고, 에디터 디테일 패널에서 열거값을 이름으로 표시합니다.

지정자효과
BlueprintTypeBP에서 이 열거형을 변수 타입·스위치 노드로 사용 가능
// BlueprintType — BP에서 드롭다운 변수·Switch 노드로 사용 가능
UENUM(BlueprintType)
enum class EWeaponType : uint8   // enum class + uint8 이 표준 조합
{
    Sword,
    Bow,
    Staff
};

// 사용 예 — UPROPERTY 변수 타입으로 쓰면 에디터에 드롭다운으로 표시된다
UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Weapon")
EWeaponType WeaponType = EWeaponType::Sword;
💡

UENUM은 enum class(범위 지정 열거형)과 함께 쓰는 것이 표준입니다. 기본 타입은 uint8로 지정합니다(enum class EState : uint8). 이렇게 하면 BP에서 깔끔하게 사용할 수 있습니다.

UINTERFACE() — 인터페이스를 엔진에 등록

UINTERFACE()는 언리얼 인터페이스를 리플렉션에 등록합니다. 언리얼 인터페이스는 두 개의 클래스가 한 쌍입니다: UHT가 인식하는 UMyInterface(껍데기)와 실제 순수 가상 함수를 담는 IMyInterface(본체).

지정자효과
BlueprintTypeBP에서 이 인터페이스 타입 변수를 사용 가능
MinimalAPI모듈 외부에서 타입만 참조 가능 (함수 호출은 불가)
// BlueprintType — BP 변수 타입 허용 / MinimalAPI — 외부 모듈은 타입 참조만
// 지정자는 U 접두사 껍데기 클래스 쪽에 붙인다
UINTERFACE(MinimalAPI, BlueprintType)
class UInteractable : public UInterface
{
    GENERATED_BODY()
};

// I 접두사 본체 — 실제 함수는 여기에 선언한다
class IInteractable
{
    GENERATED_BODY()

public:
    // BlueprintNativeEvent와 조합하면 BP로 만든 클래스도 이 함수를 구현 가능
    UFUNCTION(BlueprintCallable, BlueprintNativeEvent, Category = "Interact")
    void Interact(AActor* Caller);
};
04 — 흐름

UHT 빌드 파이프라인 — 매크로가 코드가 되기까지

매크로를 헤더에 쓰면 그것이 어떻게 작동하는 코드로 바뀌는지, 4단계 파이프라인을 따라갑니다.

1
개발자가 .h 파일에 매크로를 작성
.h 파일에 UCLASS(), UPROPERTY() 등을 붙이고, 클래스 본문에 GENERATED_BODY()를 넣습니다. 마지막 #include로 MyClass.generated.h를 추가합니다.
2
UHT가 헤더 파일을 스캔
UHT(Unreal Header Tool)(앞서 핵심 용어 블록에서 설명)가 빌드 전에 모든 .h 파일을 읽고, UCLASS·UPROPERTY·UFUNCTION 등의 매크로를 찾아 클래스 구조 정보(이름·타입·지정자)를 수집합니다.
3
.generated.h 와 .gen.cpp 자동 생성
UHT가 수집한 정보를 바탕으로 두 가지 파일을 자동 생성합니다.
MyClass.generated.h — 리플렉션 메타데이터, GENERATED_BODY()가 확장될 코드(생성자 헬퍼, 타입 정보 등)
MyClass.gen.cpp — 리플렉션 등록 코드(엔진 초기화 시 실행되어 타입 정보를 전역 테이블에 등록)
4
C++ 컴파일러가 최종 빌드
일반 C++ 컴파일러(MSVC/Clang)가 원본 .h + .generated.h + .cpp + .gen.cpp를 모두 컴파일합니다. 결과물에는 리플렉션 정보가 포함되어, 런타임에 엔진이 클래스 구조를 조회할 수 있게 됩니다.

generated.h가 반드시 마지막 #include여야 하는 이유

.generated.h는 UHT가 자동 생성하는 파일입니다. 이 파일은 그 위에 있는 모든 #include가 이미 처리된 상태를 전제로 코드를 생성합니다. 따라서 .generated.h 아래에 다른 헤더를 넣으면 UHT가 생성한 매크로 확장 순서가 꼬여 컴파일 에러가 발생합니다.

✓ 올바른 순서

  • #include "CoreMinimal.h"
  • #include "GameFramework/Actor.h"
  • #include "MyClass.generated.h" — 맨 마지막

✗ 컴파일 에러

  • #include "CoreMinimal.h"
  • #include "MyClass.generated.h" — 너무 일찍!
  • #include "GameFramework/Actor.h"
⚠️
GENERATED_BODY()를 빠뜨리면 컴파일 에러가 발생합니다. UCLASS()나 USTRUCT()가 붙은 타입은 반드시 본문 첫 줄에 GENERATED_BODY()를 넣어야 합니다. 이 매크로가 .generated.h의 코드를 클래스 안에 삽입하는 역할을 합니다.
⚠️
.generated.h가 필요한 헤더와 불필요한 헤더의 구분. UCLASS·USTRUCT·UENUM·UINTERFACE 중 하나라도 사용하는 .h만 .generated.h를 넣습니다. 리플렉션 매크로가 없는 순수 C++ 헤더에 넣으면 오히려 에러가 납니다(생성될 파일이 없으므로).

UCLASS + UPROPERTY + UFUNCTION — 완전한 .h / .cpp 쌍

가장 흔한 조합입니다. 캐릭터 클래스에 체력 변수와 데미지 함수를 만드는 예제입니다.

#pragma once

#include "CoreMinimal.h"
#include "GameFramework/Character.h"
#include "MyCharacter.generated.h"  // 반드시 마지막 #include

// UCLASS: 이 클래스를 엔진에 등록한다
// Blueprintable → BP에서 이 클래스를 상속해 자식 BP 생성 가능
UCLASS(Blueprintable)
class AMyCharacter : public ACharacter
{
    GENERATED_BODY()  // generated.h의 코드를 여기에 삽입 (필수)

public:
    // 생성자 선언
    AMyCharacter();

    // UPROPERTY: 변수를 엔진에 등록
    // EditAnywhere  → 에디터 디테일 패널에서 편집 가능
    // BlueprintReadWrite → BP에서 읽기+쓰기 가능
    // Category → 에디터에서 "Stats" 카테고리로 묶어 표시
    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Stats")
    float Health;

    // VisibleAnywhere → 에디터에서 보이지만 편집 불가
    // BlueprintReadOnly → BP에서 읽기만 가능
    UPROPERTY(VisibleAnywhere, BlueprintReadOnly, Category = "Stats")
    bool bIsDead;

    // UFUNCTION: 함수를 엔진에 등록
    // BlueprintCallable → BP에서 이 함수를 노드로 호출 가능
    UFUNCTION(BlueprintCallable, Category = "Combat")
    void ApplyDamage(float Amount);

    // BlueprintPure → 실행 핀 없이 값만 반환하는 BP 노드
    UFUNCTION(BlueprintPure, Category = "Stats")
    float GetHealthPercent() const;

protected:
    // 이 변수는 UPROPERTY가 없으므로
    // 리플렉션 미등록 → 에디터·BP·GC에서 보이지 않음
    float InternalTimer;
};
#include "MyCharacter.h"

// 생성자: 기본값 설정
AMyCharacter::AMyCharacter()
{
    Health = 100.0f;
    bIsDead = false;
    InternalTimer = 0.0f;
}

// BlueprintCallable 함수의 구현
void AMyCharacter::ApplyDamage(float Amount)
{
    Health -= Amount;

    if (Health <= 0.0f)
    {
        Health = 0.0f;
        bIsDead = true;
    }
}

// BlueprintPure 함수의 구현
float AMyCharacter::GetHealthPercent() const
{
    return Health / 100.0f;  // 0.0 ~ 1.0 비율로 반환
}

USTRUCT — 데이터 묶음 구조체

#pragma once

#include "CoreMinimal.h"
#include "CharacterStats.generated.h"

// USTRUCT: 구조체를 엔진에 등록
// BlueprintType → BP에서 이 구조체를 변수 타입으로 사용 가능
USTRUCT(BlueprintType)
struct FCharacterStats    // 접두사 F (구조체 규칙)
{
    GENERATED_BODY()       // 구조체에도 필수

    // 구조체 멤버도 UPROPERTY로 등록해야 BP에서 사용 가능
    UPROPERTY(EditAnywhere, BlueprintReadWrite)
    float MaxHealth = 100.0f;

    UPROPERTY(EditAnywhere, BlueprintReadWrite)
    float AttackPower = 10.0f;

    UPROPERTY(EditAnywhere, BlueprintReadWrite)
    float MoveSpeed = 600.0f;   // 언리얼 이동 속도 단위는 cm/s (600 = 초당 6m)
};

UENUM — 상태 열거형

#pragma once

#include "CoreMinimal.h"
#include "CharacterState.generated.h"

// UENUM: 열거형을 엔진에 등록
// BlueprintType → BP에서 드롭다운·스위치 노드로 사용
UENUM(BlueprintType)
enum class ECharacterState : uint8   // uint8 = BP 호환 타입
{
    Idle       UMETA(DisplayName = "대기"),       // 에디터에 "대기"로 표시
    Running    UMETA(DisplayName = "이동"),
    Attacking  UMETA(DisplayName = "공격"),
    Dead       UMETA(DisplayName = "사망")
};

UINTERFACE — 인터페이스 (U + I 한 쌍)

#pragma once

#include "CoreMinimal.h"
#include "UObject/Interface.h"
#include "Damageable.generated.h"

// UINTERFACE: 언리얼 인터페이스 등록
// MinimalAPI → 모듈 외부에서 타입만 참조 가능
UINTERFACE(MinimalAPI)
class UDamageable : public UInterface   // U 접두사 (UHT 인식용 껍데기)
{
    GENERATED_BODY()
};

// 실제 인터페이스 본체 — 여기에 순수 가상 함수를 선언
class IDamageable                        // I 접두사 (구현 클래스가 상속)
{
    GENERATED_BODY()

public:
    // 구현 클래스가 반드시 오버라이드해야 하는 함수
    virtual void ReceiveDamage(float Amount) = 0;

    // 기본 구현이 있는 함수 (오버라이드 선택적)
    virtual bool IsDead() const { return false; }
};
🧭

UINTERFACE는 항상 U + I 한 쌍입니다. UDamageable(UINTERFACE가 붙는 껍데기, UHT 인식용)과 IDamageable(실제 함수를 담는 본체, 구현 클래스가 상속)을 반드시 같은 헤더에 함께 선언합니다. 구현 클래스는 class AMyEnemy : public ACharacter, public IDamageable처럼 I 접두사 클래스를 상속합니다.

06 — 확장

메타데이터로 에디터를 커스터마이징

지정자 외에 meta=(...) 키워드를 사용하면 에디터에서 변수·함수가 보이는 방식을 더 세밀하게 제어할 수 있습니다. meta는 리플렉션 매크로 괄호 안에 함께 씁니다.

메타 키워드효과사용 예
DisplayName에디터/BP에 표시되는 이름을 변경meta=(DisplayName="체력")
ToolTip에디터에서 마우스 올리면 나오는 설명 텍스트meta=(ToolTip="캐릭터의 현재 HP")
ClampMin / ClampMax에디터에서 입력 가능한 값 범위 제한meta=(ClampMin="0", ClampMax="100")
AllowPrivateAccessprivate 변수를 에디터에서 편집 허용meta=(AllowPrivateAccess="true")
BindWidgetUMG 위젯 BP에서 같은 이름의 위젯을 자동 바인딩meta=(BindWidget)
ExposeOnSpawnSpawnActor 노드에서 핀으로 노출meta=(ExposeOnSpawn="true")

meta 사용 예제

// 에디터에 "체력"으로 표시, 0~200 범위 제한, 툴팁 추가
UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Stats",
    meta = (DisplayName = "체력", ClampMin = "0", ClampMax = "200",
            ToolTip = "캐릭터의 현재 HP. 0이 되면 사망 처리."))
float Health;

// private 변수지만 에디터에서 편집 가능하게 허용
private:
UPROPERTY(EditDefaultsOnly, Category = "Internal",
    meta = (AllowPrivateAccess = "true"))
float RespawnDelay = 3.0f;
💡

이 문서에서 다룬 것은 가장 자주 쓰는 지정자와 메타 키워드입니다. 전체 목록은 언리얼 공식 문서의 "Property Specifiers", "Function Specifiers" 페이지에서 확인할 수 있습니다. 하지만 실무에서는 위 표의 것들만으로도 대부분의 상황을 커버할 수 있습니다.
