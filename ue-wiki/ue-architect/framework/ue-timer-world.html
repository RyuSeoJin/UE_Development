타이머와 World
시간 콜백은 누구 소속인가

"N초 뒤에 이 함수를 불러줘"를 담당하는 FTimerManager는 전역 서비스가 아니라 World의 부품입니다. 이 한 문장을 모르면 "Subsystem Initialize()에서 타이머를 걸었는데 GetWorld()가 null"이라는, 초보자가 가장 오래 헤매는 벽에 부딪힙니다. 이 문서는 그 벽의 원인(수명 타임라인)과 안전한 우회로를 다룹니다.

엔진 자체 기능 UE4 / UE5 공통 C++ 중심
01 — 개념 · 역할

타이머는 예약, 핸들은 예약 번호표

매 프레임 Tick에서 시간을 직접 빼는 대신, 엔진에 "예약"을 겁니다: "5초 뒤에(또는 5초마다) 이 함수를 호출해줘." 예약을 받는 곳이 FTimerManager, 예약증이 FTimerHandle입니다.

핵심 용어

📖

World — 레벨(맵)이 로드되어 실행 중인 무대. 액터·시간·물리가 전부 World 위에서 돕니다. 레벨을 이동하면 이전 World는 파괴됩니다.

FTimerManager — World가 소유한 예약 접수처. World의 시간(일시정지·시간 배속의 영향을 받는 시간)을 기준으로 만료를 판정합니다. GetWorld()->GetTimerManager()로 접근합니다.

FTimerHandle — 예약 번호표. 예약을 취소(ClearTimer)·조회(GetTimerRemaining)·일시정지(PauseTimer)하려면 이 핸들이 필요하므로 멤버 변수로 보관합니다.

수명(Lifetime) — 객체가 태어나서 죽기까지의 구간. GameInstance(앱 전체) ⊃ World(레벨 하나) 관계라서, GameInstance가 살아 있어도 World는 아직 없거나 이미 죽었을 수 있습니다. 이 어긋남이 이 문서의 주제입니다.

02 — 수명 관계

왜 Initialize()에서 GetWorld()가 null인가

앱이 켜질 때의 순서를 시간축으로 보면 답이 보입니다. GameInstance와 그 Subsystem들은 World가 만들어지기 전에 초기화됩니다.

flowchart LR
    GI[GameInstance 생성] --> INIT[Subsystem Initialize — World 없음!]
    INIT --> WA[World A 로드 + BeginPlay]
    WA --> WB[레벨 이동 → World A 파괴, World B 생성]
    WA -. World A의 타이머 전부 소멸 .-> WB
03 — 어디서 걸어야 안전한가

타이머 시작 위치 결정표

내가 있는 곳안전한 시작 시점이유
Actor / ComponentBeginPlay() 이후BeginPlay = "World에 배치 완료" 신호. GetWorld()가 항상 유효
UMG 위젯NativeConstruct() 이후화면에 붙은 위젯은 World를 가짐. 파괴 시 ClearTimer 필수
GameInstanceSubsystemInitialize() ✗ → 사용자 입력/게임 시작 등 첫 호출 시점에Initialize 시점엔 World가 없음. "타이머 시작" 함수(예: StartXxx())가 불릴 때는 World가 있으므로 그때 걸기
항상 돌아야 하는 전역 틱FTSTicker (World 무관)레벨 이동에도 살아남아야 하는 백그라운드 갱신은 World 타이머가 아니라 엔진 티커 사용(§06)
💡
판별 한 줄 — "이 타이머는 레벨이 바뀌어도 계속 가야 하나?" 아니오 → World 타이머(위 셋). 예 → 설계를 다시 보거나(대부분 만료 시각을 저장해 두고 재계산하면 충분) FTSTicker.
04 — 사용 흐름

걸기 · 멈추기 · 남은 시간 보존

1
걸기 — SetTimer
GetWorld()->GetTimerManager().SetTimer(Handle, this, &클래스::OnExpired, 5.f, false). 마지막 인자 true면 5초마다 반복. 전제: World 유효. 결과: Handle이 유효한 예약을 가리킴.
2
일시정지 / 재개
PauseTimer(Handle) / UnPauseTimer(Handle) — 남은 시간이 그대로 얼었다 녹습니다. 대부분의 "일시정지" 요구는 이 두 줄로 끝나며, 남은 시간을 직접 계산할 필요가 없습니다.
3
조회 — 남은 시간 표시
GetTimerRemaining(Handle)이 초 단위 float를 돌려줍니다. UI 카운트다운은 이 값을 읽어 그리기만 하면 됩니다(별도 카운트 변수를 두지 않기 — 이중 장부 방지).
4
끄기 — ClearTimer
ClearTimer(Handle) — 예약 취소 + 핸들 무효화. 소유 객체가 파괴되기 전(위젯 NativeDestruct 등)에 반드시 호출합니다.
05 — 구현 스켈레톤

Subsystem에서 안전하게 타이머 걸기

핵심은 Initialize가 아니라 Start 함수에서 건다는 것 — 그 시점엔 World가 반드시 있습니다.

#pragma once
#include "Subsystems/GameInstanceSubsystem.h"
#include "CountdownSubsystem.generated.h"

DECLARE_DYNAMIC_MULTICAST_DELEGATE(FOnCountdownFinished);

UCLASS()
class MYGAME_API UCountdownSubsystem : public UGameInstanceSubsystem
{
    GENERATED_BODY()
public:
    UFUNCTION(BlueprintCallable) void Start(float Seconds);
    UFUNCTION(BlueprintCallable) void Pause();
    UFUNCTION(BlueprintCallable) void Resume();
    UFUNCTION(BlueprintPure)     float GetRemaining() const;

    UPROPERTY(BlueprintAssignable)
    FOnCountdownFinished OnFinished;

private:
    void HandleExpired();
    FTimerHandle Handle;   // 예약 번호표 — 반드시 멤버로 보관
};
#include "CountdownSubsystem.h"
#include "TimerManager.h"

void UCountdownSubsystem::Start(float Seconds)
{
    // Initialize()가 아니라 여기서 건다 — 사용자가 부르는 시점엔 World가 있다.
    UWorld* World = GetGameInstance()->GetWorld();
    if (!World) return;   // 방어 — 로딩 경계에서 불리는 예외 케이스

    World->GetTimerManager().SetTimer(
        Handle, this, &UCountdownSubsystem::HandleExpired,
        Seconds, false);   // false = 1회성
}

void UCountdownSubsystem::Pause()
{
    if (UWorld* W = GetGameInstance()->GetWorld())
        W->GetTimerManager().PauseTimer(Handle);   // 남은 시간이 언 채로 보존
}

void UCountdownSubsystem::Resume()
{
    if (UWorld* W = GetGameInstance()->GetWorld())
        W->GetTimerManager().UnPauseTimer(Handle);
}

float UCountdownSubsystem::GetRemaining() const
{
    if (UWorld* W = GetGameInstance()->GetWorld())
        return W->GetTimerManager().GetTimerRemaining(Handle);
    return 0.f;
}

void UCountdownSubsystem::HandleExpired()
{
    OnFinished.Broadcast();   // 후속 처리는 구독자(델리게이트 문서 참조)
}
06 — 확장 & 주의

시간이 꼬이는 지점들

⚠️
① Initialize()에서 SetTimer — 이 문서의 주제. GetWorld()가 null이라 크래시하거나(체크 없이 역참조) 조용히 무시됩니다. 타이머는 "시작 요청 함수" 안에서 거세요.
⚠️
② 레벨 이동 = 타이머 전멸 — TimerManager가 World 소속이므로 World가 죽으면 예약도 함께 사라집니다. 레벨을 넘나드는 게임이라면 "만료 시각(FDateTime)을 저장해 두고 새 World에서 남은 시간을 재계산"하는 방식이 정석입니다.
⚠️
③ 람다 캡처 수명 — SetTimer(Handle, [this](){...}, ...) 식의 람다는 this가 먼저 죽으면 위험합니다. 멤버 함수 바인딩(자동 무효화됨)을 기본으로 하고, 람다가 꼭 필요하면 FTimerDelegate::CreateWeakLambda(this, ...)를 쓰세요.
⚠️
④ 게임 일시정지의 영향 — World 타이머는 게임 일시정지(SetGamePaused)와 시간 배속의 영향을 받습니다. "실제 벽시계 기준"이 필요하면 타이머가 아니라 FDateTime 비교로 판정하세요(위키 FDateTime 문서).
💡
FTSTicker — World와 무관하게 매 틱 불리는 엔진 레벨 티커. 레벨 이동에도 살아남아야 하는 얇은 배경 갱신(초당 1회 폴링 등)에만 제한적으로 사용하고, 게임 로직 타이머는 World 타이머를 유지하세요.
