Slate
Overview

UMG의 발밑에서 실제로 UI를 그리는 언리얼의 C++ UI 프레임워크입니다. 언리얼 에디터 화면 전체가 Slate로 만들어져 있습니다. 이 문서의 목표는 "Slate로 UI를 짜는 것"이 아니라 — 게임 UI는 UMG로 충분합니다(별도 문서: UMG 기초) — Slate 코드를 만났을 때 읽을 수 있고, 언제 필요한지 판단할 수 있는 상태가 되는 것입니다.

엔진 자체 기능 UE4 · UE5 C++ 전용
01 — 개념 · 역할

Slate란 — UObject 바깥에 사는 UI 프레임워크

Slate는 언리얼의 저수준 C++ UI 프레임워크입니다. 위젯 하나하나를 C++ 코드로 선언해 조립하며, 언리얼 에디터의 모든 창·버튼·패널이 Slate로 그려집니다 — 즉 여러분은 이미 매일 Slate를 쓰고 있는 셈입니다. UMG는 이 Slate를 UObject로 감싸 블루프린트에서 편집할 수 있게 만든 저작 레이어이고, 게임에서 UMG 위젯을 화면에 띄우는 순간 실제 그리기는 전부 Slate가 수행합니다.

구조적으로 중요한 특징이 하나 있습니다: Slate 위젯은 UObject가 아닙니다. 클래스 이름이 S 접두사(SButton, STextBlock)이고, UCLASS·UPROPERTY 같은 리플렉션 매크로가 없으며, GC(가비지 컬렉션) 대신 스마트 포인터(TSharedRef/TSharedPtr)의 참조 카운팅으로 수명이 관리됩니다. 리플렉션 매크로 문서에서 배운 규칙들이 적용되지 않는 별세계라는 점이, UMG와 Slate를 가르는 근본 차이입니다.

🔑

핵심 용어 — SWidget: 모든 Slate 위젯의 최상위 부모 클래스(S 접두사). · 선언적 문법(Declarative Syntax): 위젯 트리를 C++ 코드 안에 계층 모양 그대로 적는 Slate 전용 표기법 — 04 섹션의 주제. · SNew: Slate 위젯을 생성하는 매크로. · TSharedRef / TSharedPtr: 비-UObject의 수명을 관리하는 스마트 포인터(Ref = null 불가, Ptr = null 가능).

UMG

Blueprint 저작게임 UI

U 접두사 · UObject 세계. 디자이너 탭에서 시각 편집, 리플렉션·GC 적용. 게임 화면 UI의 기본 선택지.

Slate

C++ 전용에디터 · 저수준

S 접두사 · 비-UObject 세계. 코드로 선언·조립, 스마트 포인터 수명 관리. 에디터 확장과 UMG가 못 닿는 커스텀의 도구.

02 — 구성 요소 관계

SWidget 계층과 UWidget ↔ SWidget 매핑

모든 Slate 위젯은 SWidget의 자손이며, 자식을 몇 개 담을 수 있느냐로 세 계열이 나뉩니다.

// Slate 위젯의 3계열 — 자식 수용 능력 기준 SWidget // 모든 Slate 위젯의 뿌리 ├─ SLeafWidget // 자식 없음 — 스스로 그리는 말단 │ ├─ STextBlock // 텍스트 │ └─ SImage // 이미지 ├─ SPanel // 자식 여러 개 — 배치 담당 │ ├─ SVerticalBox / SHorizontalBox │ └─ SCanvas / SOverlay / SGridPanel └─ SCompoundWidget // 자식 슬롯 1개 — 내부에서 조합 ├─ SButton // 버튼도 "내용물 1개를 담는 조합 위젯" └─ (커스텀 위젯의 기본 부모 — 05 섹션)

UMG 팔레트의 위젯들은 대부분 같은 이름의 Slate 위젯을 1:1로 감싼 것입니다. UMG 위젯(UWidget)이 화면에 표시될 때 내부에서 짝이 되는 SWidget을 생성해 넘겨줍니다.

UTextBlockSTextBlock1:1 래핑
UButtonSButton1:1 래핑
UImageSImage1:1 래핑
UBorderSBorder1:1 래핑
UVerticalBoxSVerticalBox슬롯 개념도 동일하게 대응
UUserWidget (위젯 BP)SWidget 트리 전체디자이너의 계층이 통째로 Slate 트리가 됨
UMG 위젯 (UWidget)
에디터에서 편집 · UObject · 리플렉션/GC
표시 시점에 내부 SWidget 생성→
Slate 위젯 (SWidget)
실제 레이아웃 계산 · 렌더링 담당
💡

이 매핑이 주는 실용 지식: UMG 문서·포럼에서 "SButton의 스타일이…" 같은 말이 나와도 당황할 필요가 없습니다 — UMG Button의 밑바닥 이야기일 뿐입니다. 또 UMG 위젯의 어떤 속성이 그래프 노드로 노출되어 있지 않을 때, C++에서 내부 SWidget에 접근해 조작하는 우회로가 존재한다는 것도 이 구조 덕분입니다(고급 주제).

03 — 선택 기준

언제 UMG로 충분하고, 언제 Slate를 직접 쓰나

만들려는 것도구왜
게임 화면 UI — HUD·메뉴·인벤토리·대화창UMG시각 편집 + BP 로직 + 애니메이션까지 전부 제공. 게임 UI에서 Slate를 직접 쓸 이유는 거의 없음
에디터 확장 — 커스텀 에디터 창·툴바·디테일 패널 커스터마이즈Slate에디터 자체가 Slate 세계 — 에디터 확장 API가 SWidget을 요구
간단한 에디터 도구 (버튼 몇 개짜리 내부 툴)Editor Utility Widget (UMG)UMG 문법으로 에디터 도구를 만드는 지름길 — Slate 없이도 가능한 영역이 꽤 넓음
UMG에 없는 동작의 커스텀 위젯 — 특수 레이아웃·저수준 입력 처리Slate로 만들고 UMG에 노출SWidget으로 구현 후 UWidget으로 감싸면(06 섹션) 팔레트에서 재사용 가능
극한 성능의 대량 UI 요소Slate 직접 제어UObject 래핑 비용을 걷어낼 수 있는 최후의 층 — 프로파일링으로 병목을 확인한 뒤에만
🧭

초보 단계의 현실적인 로드맵: ① UI는 전부 UMG로 만든다 → ② Slate는 이 문서 수준(구조 + 문법 읽기)만 알아둔다 → ③ 에디터 확장이나 커스텀 위젯이 실제로 필요해지는 날, 04·05 섹션을 발판으로 파고든다. Slate를 미리 깊게 공부하는 것은 게임 제작 진도에 거의 도움이 되지 않습니다.

04 — 흐름 · 문법 읽기

선언적 문법 — 코드가 UI 계층 모양 그대로

Slate 코드가 처음에 외계어처럼 보이는 이유는 선언적 문법(Declarative Syntax) 때문입니다. 일반 C++처럼 "한 줄씩 실행"으로 읽지 말고, 들여쓰기가 곧 위젯 트리의 계층이라고 읽으면 풀립니다. UMG 디자이너의 계층 패널을 코드로 옮겨 적은 것뿐입니다. 

// "이름표 + 체력바" 세로 묶음을 Slate로 쓰면:
// (UMG 계층: Vertical Box ─ Text Block, Progress Bar 와 동일한 구조)

SNew(SVerticalBox)                                // ① SNew(타입): 위젯 생성

+ SVerticalBox::Slot()                            // ② +Slot(): 자식 자리를 하나 추가
  .AutoHeight()                                  // ③ .속성(): 슬롯 설정 (UMG의 Size: Auto)
  .Padding(4.0f)
  [                                               // ④ [ ]: 이 슬롯에 들어갈 콘텐츠
      SNew(STextBlock)                        // 슬롯의 실제 내용물: 텍스트 위젯 하나
      // FText는 UI용 문자열 타입(다국어 대응) · FromString/TEXT로 리터럴을 FText로 변환
      .Text(FText::FromString(TEXT("Player 1")))
  ]

+ SVerticalBox::Slot()                            // 두 번째 자식 슬롯
  .FillHeight(1.0f)                              // (UMG의 Size: Fill)
  [
      SNew(SProgressBar)                      // 두 번째 슬롯의 내용물: 진행 막대 위젯
      .Percent(0.75f)                          // 채움 비율 0.0~1.0 (여기선 75%) · 1.0f = 가득 참
  ];                                                // 세미콜론은 트리 전체 끝에 딱 하나(연산자 오버로딩이라 그렇다)
문법 요소뜻UMG로 치면
SNew(위젯타입)위젯 생성 매크로 — TSharedRef 반환팔레트에서 드래그
.속성(값)생성 직후 속성 지정(체이닝으로 연속 지정)디테일 패널 입력
+ 타입::Slot()패널에 자식 슬롯 추가계층 패널에 자식 드롭
[ 콘텐츠 ]그 슬롯(또는 위젯)에 들어갈 자식계층의 들여쓰기 한 단계
SAssignNew(변수, 타입)생성하며 동시에 멤버 변수에 저장 — 나중에 접근할 위젯용Is Variable 체크
⚠️
+와 [ ]는 마법이 아니라 연산자 오버로딩입니다. Slate가 operator+와 operator[]를 재정의해서 "코드가 트리 모양"이 되도록 만든 것입니다. 그래서 세미콜론은 트리 전체가 끝나는 지점에 딱 하나— 중간 슬롯 뒤에 세미콜론을 찍으면 트리가 끊겨 컴파일 에러가 납니다. 에러 메시지가 길고 난해한 것도 이 문법의 특성이니, 막히면 괄호·대괄호 짝부터 확인하세요.
05 — 구현 스켈레톤

커스텀 SCompoundWidget 최소 골격

커스텀 Slate 위젯의 기본 부모는 SCompoundWidget(자식 슬롯 1개를 가진 조합 위젯)입니다. 리플렉션 매크로 대신 Slate 전용 매크로(SLATE_BEGIN_ARGS)가 등장하는 점, 생성자 대신 Construct 함수가 초기화를 맡는 점이 UObject 세계와 다릅니다. MYGAME_API는 자기 모듈 이름으로 바꾸세요. 

#pragma once

#include "CoreMinimal.h"              // UE 공통 최소 타입(FText, TSharedRef 등)
#include "Widgets/SCompoundWidget.h" // 부모 클래스 SCompoundWidget 선언
// 주의: .generated.h 없음 — Slate 위젯은 리플렉션 대상이 아님

// S 접두사 + UCLASS 없음: UObject가 아닌 Slate 위젯
// MYGAME_API는 이 클래스를 다른 모듈에서 쓰기 위한 DLL export 매크로 — 자기 모듈 이름으로 교체
class MYGAME_API SStatusWidget : public SCompoundWidget
{
public:
    // SLATE_BEGIN_ARGS ~ END_ARGS: SNew에 넘길 인자 묶음(FArguments 구조체)을 매크로로 자동 생성
    // Slate 전용 인자 선언부 — 이 위젯을 SNew로 만들 때 넘길 값들
    SLATE_BEGIN_ARGS(SStatusWidget) {}
        // SLATE_ARGUMENT(타입, 이름): 인자 하나 정의 · 내부 저장 이름은 _접두사가 붙어 _PlayerName
        SLATE_ARGUMENT(FText, PlayerName)   // SNew(...).PlayerName(...) 으로 전달됨
    SLATE_END_ARGS()

    // 생성자 대신 Construct가 초기화 담당 — SNew가 자동 호출
    void Construct(const FArguments& InArgs);
};
#include "SStatusWidget.h"
#include "Widgets/Text/STextBlock.h"  // 내부에서 만들 STextBlock을 쓰려면 헤더를 직접 include

// Construct: SNew가 위젯 생성 직후 자동 호출 · InArgs에 SNew에서 넘긴 인자들이 담겨 옴
void SStatusWidget::Construct(const FArguments& InArgs)
{
    // ChildSlot: SCompoundWidget이 가진 단 하나의 자식 슬롯
    ChildSlot
    [
        SNew(STextBlock)
        .Text(InArgs._PlayerName)   // SLATE_ARGUMENT는 _이름 으로 접근
    ];
}
⚠️

모듈 의존성: <모듈이름>.Build.cs의 PrivateDependencyModuleNames에 "Slate", "SlateCore"를 추가해야 컴파일됩니다(새 프로젝트의 Build.cs에 주석 처리된 상태로 이미 들어 있는 경우가 많으니 먼저 확인). UMG 모듈과 별개입니다.

⚠️
SWidget에는 UPROPERTY를 쓸 수 없습니다. UObject가 아니므로 리플렉션·GC 세계 밖입니다. 멤버로 다른 위젯을 들고 있으려면 TSharedPtr<SWidget>으로 선언하고(SAssignNew로 채움), 수명은 참조 카운팅이 관리합니다. 반대로 SWidget이 UObject를 참조해야 하면 GC에 걸리지 않도록 TWeakObjectPtr를 쓰는 것이 안전합니다.
06 — 확장 & 주의

Slate가 실제로 필요해지는 순간들

🧩
UMG에 노출
UWidget + RebuildWidget
만든 SWidget을 UWidget 파생 클래스로 감싸고 RebuildWidget()에서 반환하면, UMG 팔레트에 등장해 디자이너에서 쓸 수 있습니다 — 커스텀 위젯의 완성형.
🛠️
에디터 확장
TabManager · ToolMenus
커스텀 에디터 탭·메뉴·디테일 패널은 Slate 위젯을 요구합니다. 04·05 섹션의 문법이 그대로 쓰이는 실전 무대.
⚡
Editor Utility Widget
UMG로 에디터 툴
"에디터 도구 = Slate"가 부담이라면 — 간단한 내부 툴은 Editor Utility Widget(UMG 문법)으로 먼저. Slate는 그것으로 안 될 때.
💡
학습 순서 제안 — 이 문서를 읽었다면 Slate에 대한 "지금 필요한 만큼"은 끝났습니다. 다음 걸음은 Slate 심화가 아니라 UMG로 실제 화면을 만들어 보는 것입니다(UMG 기초 04 절차 → 위젯 카탈로그에서 부품 찾기). Slate가 다시 등장하는 시점은 에디터 확장이 필요해질 때이고, 그때 이 문서의 04·05 섹션이 출발점이 됩니다.
