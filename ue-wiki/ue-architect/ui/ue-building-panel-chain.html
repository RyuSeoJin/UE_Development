건물 클릭 계층 UI
패널 체인 아키텍처

2D 월드의 건물을 클릭하면 요약 팝업이 뜨고, 그 버튼을 누르면 옆으로 다음 패널이, 또 그 항목을 누르면 상세 패널이 이어지는 계층형 팝업 UI를 만듭니다. 핵심 원칙은 하나 — "패널은 자식을 모른다." 체인의 소유·열기·닫기·배치는 전부 HUD 매니저 한 곳이 담당합니다.

UMG · 이벤트 디스패처 UE5 · 2D/3D 공통 Blueprint 전용 (C++ 불필요)
01 — 요구사항

스크린샷의 흐름을 깊이(Depth)로 정의하기

참고 화면(방치형 게임의 스킬 연구소)의 동작을 그대로 요구사항으로 옮기면, UI가 깊이 4단계의 사슬로 정리됩니다:

깊이예시 화면여는 조건내용
Depth 0월드의 건물(돌조각 조형물)—클릭 가능한 액터. UI가 아님
Depth 1건물 요약 팝업 ("스킬 연구소 Lv.1")건물 클릭이름·레벨 + 기능 버튼 줄(업그레이드 등)
Depth 2기능 패널 ("스킬 연구" 목록)Depth 1의 버튼 클릭항목(스킬 아이콘) 목록 — 부모의 우측, 공간 없으면 좌측
Depth 3상세 패널 ("신속 훈련 I")Depth 2의 항목 클릭설명·비용·실행 버튼

체인 규칙 — 이 두 줄이 전체 동작을 정의합니다

⛓️

규칙 ① Depth N을 닫으면 N+1 이후가 전부 닫힌다 (요약 팝업을 닫으면 목록·상세도 사라짐). 규칙 ② Depth N에서 다른 항목을 선택하면 기존 N+1 이후를 닫고 새 체인으로 교체한다 (다른 스킬을 누르면 이전 상세가 새 상세로 바뀜).

📌

핵심 용어 — 패널 체인(Panel Chain): 부모→자식으로 이어지는 팝업 패널의 사슬. 배열 하나(PanelChain)로 관리하며 인덱스가 곧 깊이(Depth). / 이벤트 디스패처(Event Dispatcher): 블루프린트의 방송 장치(멀티캐스트 델리게이트의 BP 이름) — 위젯이 "내 버튼이 눌렸다"를 외치면, 구독해 둔 쪽(매니저)만 반응. / HUD 매니저: 패널 체인을 소유한 최상위 위젯(WBP_HUD). 이 문서의 주인공.

초록 패널이 보라 패널을 직접 스폰하면: 상위 닫기 시 하위가 남고, 패널끼리 참조가 얽혀(순환) 교체 규칙 ②를 구현할 곳이 없어집니다.

💡
왜 여기선 캐스트를 써도 되나 — 건물이 자기 PlayerController를 Cast To BP_UIPlayerController하는 것은 "상대가 항상 정확히 그 클래스"인 경우라 정당합니다(블루프린트 인터페이스 문서의 판단 기준 그대로). 반대로 건물의 종류가 늘어나는 방향은 캐스트 대신 위젯 클래스 변수/인터페이스로 열어 둡니다.

04 — 준비 (위젯 · 변수 · 디스패처)

구현 전에 만들어 둘 것들

전제: 2D 사이드뷰 카메라 문서의 설정(카메라 Pawn, PlayerController의 Show Mouse Cursor·Enable Click Events)이 되어 있다고 가정합니다 — 없다면 06-2에서 재확인합니다.

1
위젯 4종 생성
Content Browser 우클릭 → User Interface → Widget Blueprint로 4개: WBP_HUD(루트는 Canvas Panel — 패널들을 좌표로 배치할 도화지), WBP_BuildingSummary(Depth 1), WBP_SkillList(Depth 2), WBP_SkillDetail(Depth 3). 각 패널의 루트 Canvas 자식들은 원하는 대로 꾸미되, 패널 최상위는 Size To Content가 아닌 고정 크기로 시작하면 배치 계산이 단순해집니다.
2
매니저 변수·함수 뼈대
WBP_HUD에 변수 PanelChain(User Widget 배열) 추가. 함수 3개를 미리 만들어 둡니다(내용은 05에서): Open Panel(입력: Depth Integer · Panel Class Class · Anchor Widget), Close Chain From(입력: Depth), Place Next To(입력: Panel · Anchor).
3
패널에 디스패처 추가
각 패널 위젯의 My Blueprint ▸ Event Dispatchers ＋ → 이름 OnItemSelected, Details ▸ Inputs에 ItemIndex(Integer) 추가 — "몇 번째 항목이 눌렸다"만 외부에 알립니다. 어떤 패널이 다음에 열릴지는 여기 없습니다(매니저의 결정).
4
건물 액터 준비
BP_Building(Actor)에 Static Mesh(돌조각) 추가 + 변수 SummaryPanelClass(User Widget의 Class Reference, Instance Editable) — 레벨에 놓인 건물마다 다른 요약 패널을 지정할 수 있게. 콜리전: 메시의 Collision Preset에서 Trace Response ▸ Visibility = Block — OnClicked는 Visibility 채널로 판정하므로 이게 없으면 클릭이 영원히 안 잡힙니다.
5
컨트롤러 준비
BP_UIPlayerController에 변수 HUDWidget(WBP_HUD 참조)과 함수 Open Building UI(입력: Building Actor) 추가 — 내용은 "HUDWidget의 Open Panel(Depth 0 정리 후 1 열기)을 호출"뿐인 전달자입니다.
05 — 구현 (블루프린트 노드 그래프)

그래프 4개로 완성

이벤트 함수 호출 순수 함수 흐름 제어·함수 입구 캐스트 Bool Integer Class Object

그래프 ① — 건물 클릭을 매니저에게 (BP_Building)

메시 컴포넌트 선택 → Details 하단 Events ▸ On Clicked ＋. 자기 컨트롤러로 캐스트해(02의 "정당한 캐스트") Open Building UI를 부릅니다.

🧭

컨트롤러의 Open Building UI 내부는 한 줄입니다: HUDWidget → Close Chain From(0) → Open Panel(Depth 1, Building의 SummaryPanelClass, Anchor 없음). 새 건물을 클릭하면 기존 체인 전체가 교체됩니다(규칙 ②의 Depth 0 버전).

그래프 ② — 매니저의 Open Panel (WBP_HUD)

체인 규칙이 전부 이 함수에 모입니다: 정리 → 생성 → 배치 → 기록. Close Chain From과 Place Next To는 04-2에서 만든 접은 함수입니다(Close Chain From 내부: PanelChain을 뒤에서부터 돌며 Depth 이상을 Remove From Parent + 배열 Remove).

그래프 ③ — Place Next To: 우측 우선, 공간 없으면 좌측

새 패널을 캔버스에 Add Child to Canvas한 뒤 위치를 정합니다. 판정식은 하나: Anchor 우측 끝 + 새 패널 폭 + 여백(16px)이 뷰포트 폭을 넘는가. Anchor가 없으면(Depth 1) 건물 위 화면 좌표나 고정 위치를 씁니다.

⚠️
PanelW가 0으로 나온다면 — Create Widget 직후에는 위젯이 아직 레이아웃 계산 전이라 Get Desired Size가 (0, 0)을 돌려줍니다. 판정식 앞에 Force Layout Prepass(대상: 새 패널)를 한 번 호출해 크기를 강제로 계산시키거나, 04-1처럼 패널을 고정 크기로 만들어 상수를 쓰세요.

그래프 ④ — 방송(패널)과 수신(매니저)

좌측은 패널 위젯 내부(버튼 → 방송), 우측은 매니저가 Create Widget 직후 그 패널의 디스패처에 바인딩하는 부분입니다. 수신 이벤트가 Open Panel(Depth+1)을 다시 부르면서 체인이 이어집니다 — 같은 항목이든 다른 항목이든 Close Chain From(Depth+1)이 먼저 돌므로 규칙 ②가 자동으로 지켜집니다.

🧭

바인딩 위치가 곧 아키텍처입니다 — Bind Event는 반드시 매니저의 Open Panel 안(Create Widget 직후)에서 합니다. 패널이 스스로 매니저를 찾아 바인딩하게 만들면 "패널은 자식(그리고 부모)을 모른다" 원칙이 무너집니다. 빨간 사각 핀(델리게이트)은 Bind 노드의 Event 핀에서 드래그해 Handle Item Selected 커스텀 이벤트를 만들면 자동 연결됩니다.
