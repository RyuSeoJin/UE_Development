UMG + C++
연동 프레임워크

언리얼 엔진의 UMG(Unreal Motion Graphics)를 C++로 제어하는 아키텍처입니다. Widget Blueprint를 C++ 기반 클래스에 연결해 로직은 코드로, 레이아웃은 Designer로 분리하는 표준 패턴을 다룹니다. UMG Designer 기초부터 BindWidget, 라이프사이클, 델리게이트 바인딩까지 초보자도 따라 할 수 있는 완전한 절차를 제공합니다.

UUserWidget BindWidget Designer + C++
01 — 개념

UMG란 무엇인가?

UMG(Unreal Motion Graphics)는 언리얼 엔진의 공식 UI 시스템입니다. 내부적으로는 Slate라는 저수준 UI 프레임워크 위에 구축된 고수준 위젯 프레임워크로, 게임 내 HUD, 메뉴, 인벤토리, 상점 등 모든 2D 인터페이스를 제작하는 데 사용합니다.

UMG는 Widget Blueprint(WBP)라는 시각적 에디터를 제공합니다. 마치 포토샵처럼 드래그 앤 드롭으로 버튼, 텍스트, 이미지를 배치하고, 실시간으로 미리보기할 수 있습니다. 그러나 UI 로직이 복잡해지면 Blueprint 노드가 스파게티처럼 엉키게 됩니다. 이 문제를 해결하는 것이 바로 C++ + UMG 연동 패턴입니다.

왜 C++ + UMG인가?

Blueprint만으로 UI를 만들 수 있지만, 로직이 복잡해질수록 한계가 드러납니다:

기준Blueprint OnlyC++ + Blueprint (권장)
로직 복잡도노드 수 증가 → 스파게티, 유지보수 곤란C++ 코드로 정리 → 가독성, 리팩토링 용이
성능BP VM 해석 실행 (느림)네이티브 C++ (10~20배 빠름, Tick/대량 데이터 처리)
분업프로그래머와 디자이너 작업 영역 충돌로직(C++) vs 레이아웃(Designer) 완전 분리
버전 관리바이너리(.uasset) 머지 불가C++ 텍스트 파일 → Git 머지 가능
디버깅브레이크포인트 제한적IDE 디버거 전체 기능 사용 가능

핵심 원칙: C++ 기반 클래스를 만들고, Widget Blueprint가 이를 상속(Reparent)합니다. 그러면 로직은 C++로, 레이아웃은 Designer로 깔끔하게 분리됩니다.

핵심 용어

i

Widget Blueprint (WBP) — UMG Designer에서 시각적으로 편집하는 UI 에셋. .uasset 파일로 저장되며, Content Browser에서 더블클릭하면 Designer가 열립니다.

UUserWidget — 모든 UMG 위젯의 C++ 베이스 클래스. Widget Blueprint는 내부적으로 이 클래스(또는 이를 상속한 클래스)의 인스턴스입니다.

BindWidget — C++ 멤버 변수를 Designer에 배치한 위젯에 자동 연결하는 메타 지정자. UPROPERTY(meta=(BindWidget))로 선언합니다.

NativeConstruct() — 위젯이 뷰포트에 추가될 때 호출되는 C++ 초기화 함수. Blueprint의 Event Construct와 동일 시점입니다.

Slate — UE의 저수준 UI 프레임워크. UMG는 Slate 위에 구축된 래퍼(wrapper)입니다. 직접 사용할 일은 드뭅니다.

Canvas Panel — 자유 배치 컨테이너. 자식 위젯을 절대 좌표 + 앵커로 배치합니다.

Vertical/Horizontal Box — 자동 정렬 컨테이너. 자식을 세로/가로로 순서대로 나열합니다.

앵커(Anchor) — 부모 위젯 내에서 자식의 위치 기준점. 해상도가 바뀌어도 일관된 배치를 유지하는 반응형의 핵심입니다.

03 — 책임

C++ / Designer / Blueprint 책임 분담

UMG + C++ 패턴에서 각 레이어가 담당하는 역할을 명확히 구분합니다. 이 분담을 지키면 코드 충돌 없이 분업이 가능합니다:

담당역할예시
C++ (.h/.cpp) 비즈니스 로직, 데이터 바인딩, 이벤트 핸들링, 서브시스템 연동 타이머 갱신, 골드 표시 업데이트, 버튼 클릭 핸들러, 서브시스템 델리게이트 바인딩
Designer (WBP) 시각 레이아웃, 위젯 배치, 앵커링, 색상/폰트, 애니메이션 버튼 위치/크기, 텍스트 스타일, 이미지 소스, 앵커 설정
Blueprint (최소) Designer에서만 가능한 것 (애니메이션 타임라인, 간단한 시각 피드백) Play Animation, Set Visibility (C++로 해도 되지만 디자이너 편의)

BindWidget 규칙

!

BindWidget 필수 규칙: C++에서 UPROPERTY(meta=(BindWidget))로 선언한 변수는 Designer에 같은 이름, 같은 타입의 위젯이 반드시 존재해야 합니다.

이름 불일치 시: C++ 컴파일은 성공하지만, Widget Blueprint를 에디터에서 열 때 크래시가 발생합니다.

Optional 변형: UPROPERTY(meta=(BindWidgetOptional))로 선언하면 Designer에 해당 위젯이 없어도 nullptr로 유지됩니다 (크래시 안 남).

이름 규칙 권장사항

위젯 타입접두어예시
UTextBlockTXT_TXT_Gold, TXT_Timer, TXT_PlayerName
UButtonBTN_BTN_Start, BTN_Close, BTN_Confirm
UImageIMG_IMG_Background, IMG_Icon, IMG_Portrait
UProgressBarPB_PB_Health, PB_XP, PB_Loading
UVerticalBoxVB_VB_ItemList, VB_Settings
UHorizontalBoxHB_HB_Buttons, HB_Stats
UScrollBoxSB_SB_ChatLog, SB_Inventory
*

팁: 접두어를 사용하면 C++ 코드에서 변수만 보고도 타입을 즉시 파악할 수 있고, Designer에서 검색도 쉬워집니다. 팀 내 규칙으로 통일하세요.

04 — 흐름

Widget Blueprint 생성부터 화면 표시까지

4-1. Widget Blueprint 생성 절차

C++ 기반 클래스와 Widget Blueprint를 연결하는 전체 과정입니다:

1
C++ 기반 클래스 생성
에디터 상단 메뉴: Tools > New C++ Class...
또는 Content Browser 빈 영역 우클릭 > New C++ Class...
팝업에서: Parent Class = User Widget 검색 후 선택 > Name: UFocusPanelWidget > Create Class 클릭
(Visual Studio/Rider가 자동으로 열리며 .h와 .cpp 파일이 생성됨)
2
C++ 코드 작성
.h 파일에 BindWidget 변수 선언, NativeConstruct() override, 이벤트 핸들러 함수를 작성합니다. (상세 코드는 섹션 5 스켈레톤 참조)
3
컴파일
IDE에서 Ctrl+B(Visual Studio) 또는 Ctrl+Shift+B(Rider)로 빌드.
또는 에디터 우측 하단 Compile 버튼 클릭. 성공하면 초록색 체크 표시가 나타납니다.
4
Widget Blueprint 생성
Content Browser > 원하는 폴더에서 우클릭 > User Interface > Widget Blueprint > 이름: WBP_FocusPanel > Enter
(이 시점에서는 기본 UUserWidget을 상속한 상태)
5
Parent Class 변경 (Reparent)
WBP_FocusPanel을 더블클릭하여 Widget Blueprint 에디터를 엽니다.
상단 메뉴: File > Reparent Blueprint > 검색창에 FocusPanelWidget 입력 > 선택 > Reparent 클릭
상단에 "Parent Class: FocusPanelWidget"으로 표시되면 성공입니다.
!
주의: Reparent를 하지 않으면 C++ 코드와 전혀 연결되지 않습니다. Widget Blueprint를 만든 뒤 반드시 Reparent를 수행하세요. 가장 흔한 실수입니다!
6
Designer에서 레이아웃 구성
Designer 탭에서 위젯을 배치합니다. 중요: BindWidget으로 선언한 C++ 변수와 정확히 같은 이름으로 위젯을 만들어야 합니다.
예: C++에서 UTextBlock* TXT_Timer를 선언했다면, Designer에서 TextBlock을 추가하고 이름을 TXT_Timer로 변경합니다.
7
컴파일 & 저장
Widget Blueprint 에디터 상단의 Compile 버튼 클릭 > Save 버튼 클릭.
BindWidget 이름이 일치하면 정상 컴파일됩니다. 불일치 시 에러 메시지가 나타납니다.

4-2. UMG Designer 기초 조작

Widget Blueprint를 더블클릭하면 열리는 Designer 에디터의 패널 구성을 이해해야 합니다:

패널 이름위치역할
Palette (팔레트)좌측 상단사용 가능한 위젯 목록. Text, Button, Image, Canvas Panel, Vertical Box 등을 여기서 드래그합니다.
Hierarchy (계층구조)좌측 하단현재 위젯 트리. 부모-자식 관계를 시각적으로 보여줍니다. 드래그로 재배치 가능.
Designer Canvas (캔버스)중앙시각적 편집 영역. 위젯을 직접 드래그하여 위치를 조정합니다.
Details (디테일)우측선택한 위젯의 속성 편집. 앵커, 크기, 색상, 폰트, Visibility 등 모든 설정.
Animations (애니메이션)하단UMG 애니메이션 타임라인. Fade In/Out, 슬라이드 등 시각 효과 제작.

레이아웃 패널 종류

위젯을 배치할 때 어떤 컨테이너를 사용할지 결정해야 합니다:

패널자식 배치 방식언제 사용
Canvas Panel앵커 + 절대 오프셋 (자유 배치)HUD 최상위 루트, 고정 위치 요소, 자유로운 배치가 필요할 때
Vertical Box위에서 아래로 순서대로리스트, 폼, 설정 항목, 세로로 쌓이는 요소
Horizontal Box왼쪽에서 오른쪽으로 순서대로버튼 그룹, 상태바, 가로로 나열하는 요소
Overlay모든 자식을 같은 위치에 겹침배경 위에 콘텐츠 올리기, 레이어 구조
Size Box자식 하나, 최소/최대 크기 강제고정 크기를 보장하고 싶은 요소 감싸기
Scale Box자식 하나, 종횡비 유지 스케일이미지/아이콘의 비율을 유지하면서 확대축소

Wrap Box가로 채우다 자동 줄바꿈태그 목록, 아이템 목록 (가변 개수)

앵커(Anchor) 시스템

앵커는 부모 위젯 내에서 "기준점"을 어디에 둘지 결정합니다. 화면 해상도가 바뀌어도 위젯이 올바른 위치를 유지하려면 앵커 설정이 필수입니다.

프리셋 9종 그리드:

Left (0.0)Center (0.5)Right (1.0)
Top (0.0)좌상중상우상
Middle (0.5)좌중정중앙우중
Bottom (1.0)좌하중하우하

스트레치 앵커: 앵커 Min과 Max를 다르게 설정하면 부모 크기에 맞춰 위젯이 늘어납니다. 반응형 UI의 핵심입니다.

설정 방법: 위젯 선택 > Details 패널 > Slot (Canvas Panel Slot) 섹션 > Anchors > 프리셋 아이콘 클릭 또는 Min/Max 값을 수동 입력

!
주의: 앵커를 설정하지 않으면 해상도가 바뀔 때 위젯이 화면 밖으로 벗어납니다. 모든 위젯에 앵커를 설정하세요.
*
팁: 전체 화면을 채우는 루트 위젯(예: 배경 패널)은 Anchors를 (0,0)~(1,1) 스트레치로 설정하면 어떤 해상도에서든 화면을 꽉 채웁니다.

위젯 추가 절차:

1. Palette에서 원하는 위젯(예: Text)을 찾아 드래그
2. Hierarchy에서 원하는 부모 위젯 위에 드롭
3. Hierarchy에서 추가된 위젯의 이름을 클릭 후 F2를 눌러 이름 변경
4. 이름을 C++ BindWidget 변수명과 정확히 일치시킴 (예: TXT_Timer)

4-3. 위젯 라이프사이클

위젯이 생성되어 화면에 표시되고, 제거되기까지의 생명주기입니다:

1
CreateWidget<UMyWidget>(OwningPlayer)
위젯 인스턴스를 메모리에 생성합니다. 아직 화면에는 표시되지 않습니다. BindWidget 변수가 이 시점에 바인딩됩니다.
2
AddToViewport(ZOrder)
뷰포트(화면)에 위젯을 추가합니다. 이 시점에 NativeConstruct()가 호출됩니다. 여기서 초기화 로직(델리게이트 바인딩, 초기값 표시)을 수행합니다.
3
NativeTick(FGeometry, float DeltaTime)
(선택) 매 프레임 호출. Tick이 필요한 위젯만 override합니다. 대부분의 UI는 이벤트 기반으로 갱신하므로 Tick 불필요.
4
RemoveFromParent()
뷰포트에서 위젯을 제거합니다. 이 시점에 NativeDestruct()가 호출됩니다. 여기서 델리게이트를 해제해야 합니다.
5
GC(가비지 컬렉션)에 의해 수거
더 이상 참조하는 곳이 없으면 엔진의 GC가 메모리에서 해제합니다.
!
주의: NativeConstruct()에서 BindWidget 변수에 접근하는 것은 안전합니다. 하지만 생성자(Constructor)에서는 아직 바인딩이 완료되지 않았으므로 접근하면 nullptr 크래시가 발생합니다!
i

ZOrder 가이드: AddToViewport(ZOrder)의 ZOrder 값이 높을수록 앞(위)에 표시됩니다. 권장 계층: HUD = 0, 일반 패널 = 5, 모달(팝업) = 10, 시스템 알림 = 20

4-4. 데이터 바인딩 & 갱신 흐름

서브시스템의 데이터가 변경될 때 UI가 자동으로 갱신되는 흐름입니다:

1
NativeConstruct()에서 서브시스템 캐싱
GetGameInstance()->GetSubsystem<UEconomySubsystem>()으로 서브시스템 포인터를 멤버 변수에 저장합니다. 매번 조회하지 않아도 됩니다.
2
델리게이트 바인딩
EconomySub->OnGoldChanged.AddUObject(this, &UMyWidget::HandleGoldChanged)로 "골드가 변경되면 이 함수를 호출해줘"라고 등록합니다.
3
핸들러에서 UI 갱신
데이터가 변경되면 등록한 핸들러가 자동 호출됩니다. 핸들러에서 TXT_Gold->SetText(...)로 UI를 갱신합니다.
4
NativeDestruct()에서 바인딩 해제
EconomySub->OnGoldChanged.RemoveAll(this)로 등록을 해제합니다. 위젯이 파괴된 후에도 콜백이 호출되면 크래시가 발생하기 때문입니다.
!
치명적 실수: NativeDestruct()에서 델리게이트를 해제하지 않으면, 위젯이 제거된 후 서브시스템이 브로드캐스트할 때 dangling pointer 크래시가 발생합니다. 반드시 해제하세요!

완전한 코드 예제

5-1. C++ 위젯 기반 클래스

아래는 서브시스템에서 골드와 XP를 받아 UI에 표시하는 완전한 예제입니다. 이 패턴을 기반으로 모든 UI 위젯을 작성합니다:

#pragma once   // 이 헤더가 한 번만 include 되도록 보장 (중복 정의 방지)
#include "CoreMinimal.h"   // int32·FString 등 UE 기본 타입 모음 — UE 헤더의 관례상 첫 줄
#include "Blueprint/UserWidget.h"   // 부모 클래스 UUserWidget 정의 — 상속하려면 실제 include 필요
#include "MyPanelWidget.generated.h"   // UHT(리플렉션 코드 생성기)가 만드는 헤더 — 반드시 include 목록의 맨 마지막에 있어야 컴파일됨

// 전방 선언 — 헤더에서는 포인터 타입만 쓰므로 #include 대신 이름만 알려줌 (컴파일 의존성·시간 절약)
class UTextBlock;
class UButton;
class UProgressBar;
class UEconomySubsystem;

/**
 * UMyPanelWidget — 골드/XP 표시 + 액션 버튼 패널.
 * Widget Blueprint가 이 클래스를 상속(Reparent)합니다.
 */
// Abstract: 이 C++ 클래스 자체는 인스턴스화 금지(반드시 WBP 자식으로만 사용) · Blueprintable: Widget Blueprint가 상속(Reparent) 가능
UCLASS(Abstract, Blueprintable)
// MYPROJECT_API: 이 클래스를 모듈 밖에서도 쓸 수 있게 내보내는 매크로 (프로젝트명_API 형태로 UE가 정의)
class MYPROJECT_API UMyPanelWidget : public UUserWidget
{
    GENERATED_BODY()   // UHT가 리플렉션 보일러플레이트를 채워 넣는 자리 — UCLASS 클래스 첫 줄에 필수

public:
    // --- BindWidget: Designer에서 같은 이름의 위젯을 배치해야 함 ---
    // UPROPERTY: 이 포인터를 GC(가비지 컬렉터)가 추적하게 함 → 위젯이 살아있는 동안 수거 안 됨.
    // meta=(BindWidget): 위젯 로드 시 엔진이 Designer의 동일 이름 위젯을 이 포인터에 자동 연결.
    UPROPERTY(meta=(BindWidget))
    UTextBlock* TXT_Title;   // Designer에 이름이 "TXT_Title"인 TextBlock이 반드시 있어야 함 (없으면 WBP 열 때 크래시)

    UPROPERTY(meta=(BindWidget))
    UTextBlock* TXT_Gold;

    UPROPERTY(meta=(BindWidget))
    UButton* BTN_Action;

    UPROPERTY(meta=(BindWidget))
    UProgressBar* PB_XP;

    // --- BindWidgetOptional: 없어도 크래시 안 남 ---
    UPROPERTY(meta=(BindWidgetOptional))
    UTextBlock* TXT_Optional;

protected:
    virtual void NativeConstruct() override;   // 엔진이 위젯을 뷰포트에 추가할 때 호출 → 여기서 초기화 (이 시점엔 BindWidget이 이미 연결됨)
    virtual void NativeDestruct() override;   // 엔진이 위젯을 뷰포트에서 제거할 때 호출 → 여기서 정리(델리게이트 해제)

private:
    // 서브시스템 캐시 (매번 조회하지 않기 위해)
    UPROPERTY()
    UEconomySubsystem* EconomySub;

    // 버튼 클릭 핸들러 (Dynamic Delegate이므로 UFUNCTION 필요)
    UFUNCTION()
    void HandleActionClicked();

    // 데이터 변경 핸들러 (일반 Multicast이므로 UFUNCTION 불필요)
    void HandleGoldChanged(int32 NewGold);
    void HandleXPChanged(int32 NewXP, int32 MaxXP);
};
#include "MyPanelWidget.h"
#include "Components/TextBlock.h"
#include "Components/Button.h"
#include "Components/ProgressBar.h"
#include "Subsystems/EconomySubsystem.h"

void UMyPanelWidget::NativeConstruct()
{
    Super::NativeConstruct();   // 부모(UUserWidget)의 초기화를 먼저 실행 — 생략하면 엔진 내부 상태가 누락됨

    // 1) 서브시스템 캐싱
    EconomySub = GetGameInstance()->GetSubsystem<UEconomySubsystem>();

    // 2) 버튼 클릭 바인딩 (Dynamic Multicast → AddDynamic)
    if (BTN_Action)
    {
        BTN_Action->OnClicked.AddDynamic(this, &UMyPanelWidget::HandleActionClicked);
    }

    // 3) 서브시스템 델리게이트 바인딩 (일반 Multicast → AddUObject)
    if (EconomySub)
    {
        EconomySub->OnGoldChanged.AddUObject(this, &UMyPanelWidget::HandleGoldChanged);
        EconomySub->OnXPChanged.AddUObject(this, &UMyPanelWidget::HandleXPChanged);

        // 초기값 표시 (위젯이 열릴 때 현재 값을 바로 보여줌)
        HandleGoldChanged(EconomySub->GetGold());
        HandleXPChanged(EconomySub->GetXP(), EconomySub->GetXPToNextLevel());
    }
}

void UMyPanelWidget::NativeDestruct()
{
    // 델리게이트 해제 — 크래시 방지 (반드시!)
    if (EconomySub)
    {
        EconomySub->OnGoldChanged.RemoveAll(this);
        EconomySub->OnXPChanged.RemoveAll(this);
    }

    if (BTN_Action)
    {
        BTN_Action->OnClicked.RemoveAll(this);
    }

    Super::NativeDestruct();
}

void UMyPanelWidget::HandleActionClicked()
{
    // 버튼 클릭 시 수행할 로직
    UE_LOG(LogTemp, Display, TEXT("Action button clicked!"));
}

void UMyPanelWidget::HandleGoldChanged(int32 NewGold)
{
    if (TXT_Gold)
    {
        TXT_Gold->SetText(FText::Format(
            NSLOCTEXT("UI", "GoldFmt", "{0} Gold"),
            FText::AsNumber(NewGold)));
    }
}

void UMyPanelWidget::HandleXPChanged(int32 NewXP, int32 MaxXP)
{
    // MaxXP > 0 확인: 0으로 나누기(0 나눗셈 크래시/NaN) 방지
    if (PB_XP && MaxXP > 0)
    {
        // SetPercent 인자는 0.0~1.0 범위. int/int는 소수점이 버려지므로 float로 캐스팅 후 나눔
        PB_XP->SetPercent(static_cast<float>(NewXP) / static_cast<float>(MaxXP));
    }
}

5-2. 델리게이트 선언 (서브시스템 측)

위의 위젯이 바인딩하는 델리게이트를 서브시스템에서 어떻게 선언하는지 보여줍니다:

#pragma once
#include "CoreMinimal.h"
#include "Subsystems/GameInstanceSubsystem.h"
#include "EconomySubsystem.generated.h"

// 델리게이트 선언 — 매크로로 시그니처 정의
// Multicast: 여러 위젯이 동시에 구독 가능(1:N). OneParam/TwoParams = 콜백에 넘길 인자 개수.
DECLARE_MULTICAST_DELEGATE_OneParam(FOnGoldChanged, int32 /*NewGold*/);
DECLARE_MULTICAST_DELEGATE_TwoParams(FOnXPChanged, int32 /*NewXP*/, int32 /*MaxXP*/);

// GameInstanceSubsystem: 게임 인스턴스 수명(앱 실행 내내) 동안 존재하는 싱글턴 성격의 매니저 → 레벨이 바뀌어도 데이터 유지
UCLASS()
class MYPROJECT_API UEconomySubsystem : public UGameInstanceSubsystem
{
    GENERATED_BODY()

public:
    // 외부에서 바인딩할 수 있도록 public 멤버
    FOnGoldChanged OnGoldChanged;
    FOnXPChanged OnXPChanged;

    int32 GetGold() const { return Gold; }
    int32 GetXP() const { return XP; }
    int32 GetXPToNextLevel() const { return XPToNextLevel; }

    // 골드 추가 — 브로드캐스트로 UI가 자동 갱신됨
    void AddGold(int32 Amount)
    {
        Gold += Amount;
        OnGoldChanged.Broadcast(Gold);  // 바인딩된 모든 위젯에 통지
    }

    // XP 추가
    void AddXP(int32 Amount)
    {
        XP += Amount;
        OnXPChanged.Broadcast(XP, XPToNextLevel);
    }

private:
    int32 Gold = 0;
    int32 XP = 0;
    int32 XPToNextLevel = 1000;
};

5-3. CreateWidget + AddToViewport (소유자 측)

PlayerController에서 위젯을 생성하고 화면에 표시하는 코드입니다:

#pragma once
#include "CoreMinimal.h"
#include "GameFramework/PlayerController.h"
#include "MyPlayerController.generated.h"

class UMyPanelWidget;

UCLASS()
class MYPROJECT_API AMyPlayerController : public APlayerController
{
    GENERATED_BODY()

protected:
    virtual void BeginPlay() override;

private:
    // BP에서 WBP_FocusPanel을 할당 (EditDefaultsOnly)
    // TSubclassOf<T>: T를 상속한 "클래스 자체"를 담는 타입(인스턴스 아님) → 어떤 WBP를 띄울지 지정.
    // EditDefaultsOnly: 클래스 기본값에서만 편집(인스턴스별 편집 불가) → BP에서 WBP를 한 번 지정하는 용도에 적합.
    UPROPERTY(EditDefaultsOnly, Category = "UI")
    TSubclassOf<UMyPanelWidget> FocusPanelClass;

    // 생성된 위젯 인스턴스 참조 (GC 방지)
    // UPROPERTY()만 붙여도 GC가 이 포인터를 추적 → 참조하는 동안 위젯이 수거되지 않음(안 붙이면 갑자기 null)
    UPROPERTY()
    UMyPanelWidget* FocusPanel;
};
#include "MyPlayerController.h"
#include "MyPanelWidget.h"

// BeginPlay: 컨트롤러가 게임 월드에서 활동을 시작할 때 엔진이 1회 호출 → UI 초기 생성에 적합한 시점
void AMyPlayerController::BeginPlay()
{
    Super::BeginPlay();   // 부모의 BeginPlay 먼저 실행 (엔진 초기화 순서 보장)

    // 로컬 플레이어만 UI 생성 (서버에서는 생성하지 않음)
    // IsLocalController(): 이 컨트롤러가 실제 화면을 가진 로컬 플레이어의 것인지 검사 → 데디케이트 서버에서 UI 낭비 방지
    if (IsLocalController() && FocusPanelClass)
    {
        // CreateWidget: this(PlayerController)가 소유자
        FocusPanel = CreateWidget<UMyPanelWidget>(this, FocusPanelClass);

        // AddToViewport: ZOrder 5로 화면에 추가
        FocusPanel->AddToViewport(5);
    }
}
i

FocusPanelClass 할당 방법: TSubclassOf<UMyPanelWidget> FocusPanelClass를 EditDefaultsOnly로 선언하면, PlayerController의 Blueprint(BP_MyPlayerController)를 열어 Details 패널에서 Focus Panel Class 드롭다운을 클릭 > WBP_FocusPanel을 선택하면 됩니다.

5-4. AHUD vs PlayerController 비교

위젯을 누가 소유/생성할지 결정하는 3가지 패턴입니다:

방식장점단점언제 사용
PlayerController에서 CreateWidget 간단, 초보자 친화, 코드 최소 HUD 전용 로직과 게임 로직이 한 클래스에 혼재 소규모 프로젝트, 위젯 수가 적을 때
AHUD 서브클래스 HUD 전용 로직 분리, DrawHUD() 사용 가능 한 단계(HUD 클래스) 더 추가해야 함 대규모 프로젝트, HUD 로직이 복잡할 때
GameInstanceSubsystem (UIManager) 레벨 전환에도 유지, 위젯 라이프사이클 일괄 관리 초기 설정이 복잡, 위젯 소유권 별도 관리 패널/모달을 여러 개 관리, Show/Hide 자주 전환

06 — 확장

고급 패턴과 디버깅

위젯 애니메이션 (UWidgetAnimation)

Designer의 Animations 탭에서 생성한 애니메이션을 C++에서 재생할 수 있습니다:

// .h — BindWidgetAnim으로 Designer 애니메이션을 C++에 연결
// Transient: 저장(직렬화)하지 않음 — 애니메이션은 런타임에만 존재하므로 디스크에 안 담음.
// meta=(BindWidgetAnim): Designer Animations 탭에서 같은 이름으로 만든 애니메이션을 이 포인터에 자동 연결.
UPROPERTY(Transient, meta=(BindWidgetAnim))
UWidgetAnimation* FadeInAnim;   // Designer에 "FadeInAnim" 애니메이션이 있어야 연결됨

// .cpp — NativeConstruct()에서 페이드인 재생
if (FadeInAnim)
{
    // 인자: (애니메이션, 시작시각 0초, 재생횟수 1, 방향 Forward, 재생속도 1.0배). 횟수 0 = 무한 반복.
    PlayAnimation(FadeInAnim, 0.f, 1, EUMGSequencePlayMode::Forward, 1.f);
}

// 역재생 (페이드아웃)
PlayAnimationReverse(FadeInAnim);

// 애니메이션 종료 시 콜백
FWidgetAnimationDynamicEvent EndDelegate;
EndDelegate.BindDynamic(this, &UMyWidget::OnFadeOutFinished);
BindToAnimationFinished(FadeInAnim, EndDelegate);

입력 처리

위젯이 화면에 표시될 때 게임 입력을 차단하거나, 마우스 커서를 표시하는 방법입니다:

// UI 모드로 전환 (게임 입력 차단, 마우스 커서 표시)
FInputModeUIOnly InputMode;
// TakeWidget(): UMG 위젯의 내부 Slate 위젯을 꺼내 포커스 대상으로 지정 (키 입력이 이 위젯으로 감)
InputMode.SetWidgetToFocus(MyWidget->TakeWidget());
InputMode.SetLockMouseToViewportBehavior(EMouseLockMode::DoNotLock);
PlayerController->SetInputMode(InputMode);
PlayerController->SetShowMouseCursor(true);

// 게임 + UI 동시 입력 (인게임 HUD, 클릭 가능 버튼)
FInputModeGameAndUI GameAndUI;
GameAndUI.SetWidgetToFocus(MyWidget->TakeWidget());
PlayerController->SetInputMode(GameAndUI);

// 게임 전용 모드로 복귀 (UI 닫을 때)
PlayerController->SetInputMode(FInputModeGameOnly());
PlayerController->SetShowMouseCursor(false);

위젯 풀링 (성능 최적화)

자주 생성/파괴되는 위젯(데미지 텍스트, 알림 팝업 등)은 매번 CreateWidget하면 GC 부하가 큽니다. SetVisibility로 재사용하는 풀링 패턴을 권장합니다:

// 위젯 숨기기 (파괴하지 않음)
MyWidget->SetVisibility(ESlateVisibility::Collapsed);  // 공간도 차지 안 함

// 위젯 다시 표시
MyWidget->SetVisibility(ESlateVisibility::SelfHitTestInvisible);  // 보이지만 클릭 투과
MyWidget->SetVisibility(ESlateVisibility::Visible);  // 보이고 클릭 가능

비동기 로딩 (TSoftClassPtr)

위젯 클래스를 즉시 로드하지 않고, 필요할 때 비동기로 로드하여 초기 메모리를 절약합니다:

// .h — 소프트 레퍼런스 (즉시 로드하지 않음)
UPROPERTY(EditDefaultsOnly, Category = "UI")
TSoftClassPtr<UUserWidget> InventoryWidgetClass;

// .cpp — 필요할 때 비동기 로드
void AMyController::OpenInventory()
{
    if (InventoryWidgetClass.IsNull()) return;

    // 이미 로드됐는지 확인
    if (UClass* Loaded = InventoryWidgetClass.Get())
    {
        // 즉시 사용
        UUserWidget* Widget = CreateWidget<UUserWidget>(this, Loaded);
        Widget->AddToViewport();
    }
    else
    {
        // 비동기 로드 요청
        FStreamableManager& Mgr = UAssetManager::GetStreamableManager();
        Mgr.RequestAsyncLoad(InventoryWidgetClass.ToSoftObjectPath(),
            FStreamableDelegate::CreateUObject(this, &AMyController::OnInventoryLoaded));
    }
}
