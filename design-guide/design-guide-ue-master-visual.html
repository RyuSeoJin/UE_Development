이 디자인이 지키는 다섯 가지

화려함이 아니라 읽기 쉬움과 일관성이 목표입니다. 다섯 원칙이 전부입니다.

원칙의미
다크 · 인디고 미니멀어두운 네이비 배경에 인디고 포인트 하나. 밝은 그라데이션·요란한 색을 쓰지 않습니다.
에디토리얼 단일 컬럼두꺼운 사이드바 대신 920px 중앙 컬럼 + 상단 sticky 내비. 글처럼 위에서 아래로 읽힙니다.
모노 라벨은 순서를 뜻함01 — 개념처럼 번호 라벨은 실제 읽기 순서가 있을 때만 씁니다. 장식용 번호는 금지.
색은 의미를 운반색을 예쁘라고 칠하지 않습니다. 특히 타입 컬러 맵은 트리·관계도·태그에서만 일관되게 적용합니다.
강조는 한 곳에한 화면에서 시선을 끄는 요소는 하나. 나머지는 조용하게 유지합니다.
🎯

시그니처 요소는 타입 컬러 맵입니다 — 폴더/클래스/파일을 코드 하이라이팅처럼 색으로 구분하는 것. 이 시스템이 기억되는 단 하나의 장치이므로, 다른 곳에서 색을 남발하지 않습니다.

05 — 일관성 규칙

Do & Don't

✓ 이렇게

  • 클래스·변수·경로·값은 항상 IBM Plex Mono로
  • 색은 의미가 있을 때만. 타입 색은 트리·관계도·태그·카드 보더에서만
  • 번호 라벨은 실제 읽기 순서가 있을 때만 사용
  • 강조 callout/통계는 섹션당 1~2개로 제한
  • 표·카드·코드 라운드는 8px(코드 10px)로 통일
  • 새 타입 색이 필요하면 1개 추가 후 이 색상표에 기록

✗ 이건 피하기

  • 밝은 그라데이션 헤더·두꺼운 좌측 사이드바
  • 본문에 모노 글꼴, 라벨에 본문 글꼴 섞어 쓰기
  • 한 화면에 4색 이상 포인트 컬러 난립
  • 이모지를 컬러 시스템 대신 의미 전달 수단으로 남용
  • 장식용 번호·아이콘으로 빈 공간 채우기
  • 같은 정보를 카드·표·다이어그램으로 중복 표현
📐

품질 기준선: 모바일까지 반응형, 키보드 포커스 표시, prefers-reduced-motion 존중. 이 세 가지는 모든 문서가 기본으로 갖춥니다.

07 — v1.4 데이터 설계 컴포넌트

파일-변수 카드 · 흐름 전도 · 변수 여정

데이터 설계 지도 템플릿(04 데이터 설계 지도) 전용 컴포넌트입니다. "변수가 어느 파일에 살고(정적 배치), 어떻게 흘러가는지(동적 연결)"를 시각화합니다. 색은 전부 기존 팔레트를 재사용합니다.

파일-변수 카드 (.var-card + .vbadge)

헤더/소스 파일 쌍이 소유한 변수의 전수 목록을 카드로 보여줍니다. 각 행은 변수명 · 타입 · 수명 배지 · 한 줄 설명으로 구성합니다. 수명 배지 5종: SAVE 디스크 저장 · RUNTIME 휘발 · CONFIG Settings 상수 · ROW DataTable 행 · DB SQLite 파일 저장 [v1.8].

블루프린트 에디터의 노드·핀·와이어를 UE 관례색 그대로 재현하는 컴포넌트입니다. 문서에서 "블루프린트로 이렇게 조립한다"를 보여줄 때 씁니다. .diagram과 동일 철학 — 생성 시점에 SVG로 구워넣고 런타임 렌더링 스크립트는 넣지 않습니다. UE 에디터의 의미색(--bp-*)은 이 컴포넌트 내부 전용이며 본문 UI에 유출하지 않습니다.

10 — v1.7 인터랙티브 체크리스트

따라 하며 완료를 체크하는 실습 컴포넌트 (.checklist)

플러그인 활성화·빌드처럼 순서대로 따라 하며 하나씩 완료를 확인하는 "따라하기" 문서용 컴포넌트입니다. 단계 헤더를 클릭(또는 Enter/Space)하면 완료로 표시되고, 상단 진행률 바가 즉시 갱신됩니다. 완료 상태는 로컬·비영속(새로고침 시 초기화)이며, 토글 JS는 탭/복사 JS처럼 결과물에 inline합니다. 색은 기존 팔레트만 재사용합니다(완료 체크 accent-3, 진행 바 accent).

⚠️

checklist 사용 규칙 — ① 완료 상태는 로컬·비영속(저장하지 않음), ② 토글/진행률 JS는 결과물에 inline(탭·복사 JS와 같은 최소 상호작용 범위), ③ 본문은 반드시 .check-body(헤더 .check-head 밖)에 넣어 codeblock 복사·탭 클릭이 완료를 토글하지 않게 함, ④ 색은 기존 팔레트만 재사용(신규 토큰 금지), ⑤ role="button"·tabindex="0"·aria-pressed로 키보드 접근성 유지.

토글 JS (결과물에 inline)
document.querySelectorAll('.checklist').forEach(function (list) {
  var steps = [].slice.call(list.querySelectorAll('.check-step'));
  var fill  = list.querySelector('.check-fill');
  var count = list.querySelector('.check-count b');
  function refresh() {
    var done = steps.filter(function (s) { return s.classList.contains('done'); }).length;
    if (count) count.textContent = done;
    if (fill)  fill.style.width = (steps.length ? done / steps.length * 100 : 0) + '%';
  }
  steps.forEach(function (step) {
    var head = step.querySelector('.check-head');
    function toggle() {
      step.classList.toggle('done');
      head.setAttribute('aria-pressed', String(step.classList.contains('done')));
      refresh();
    }
    head.addEventListener('click', toggle);
    head.addEventListener('keydown', function (e) {
      if (e.key === 'Enter' || e.key === ' ') { e.preventDefault(); toggle(); }
    });
  });
  refresh();
});
