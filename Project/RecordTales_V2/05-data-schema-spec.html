01 — 요구사항 / 범위

이 문서가 담는 것

Phase 2 게임 레이어의 데이터 스키마 전체입니다. 값(밸런스)은 넣지 않고 구조(컬럼)만 정의합니다 — 실제 수치는 DataTable/Settings에서 튜닝합니다.

🧭

세 계층: ROW DataTable 행 구조체(밸런스, FTableRowBase 상속) · SAVE SaveGame 구조(내 소유물) · CONFIG DeveloperSettings(전역 상수). 능력치는 현재 단계만 저장하고 값은 표에서 조회합니다.

03 — enum · 공용 타입

열거형

모든 등급·종류는 uint8 기반 UENUM(BlueprintType)으로(리플렉션·BP 노출 제약). 값 순서 = 등급 오름차순.

// 캐릭터 등급 — 뽑을 때 롤(인스턴스). 개화 슬롯 수·능력치 등급 분포를 결정
UENUM(BlueprintType)
enum class ECharacterRarity : uint8 { Normal, Rare, Epic, Legendary };

// 능력치 등급 — 각 능력치 자체의 등급. 효과 크기(계수 tier)를 결정
UENUM(BlueprintType)
enum class EAbilityRarity : uint8 { Normal, Rare, Epic, Legendary };

// 능력치 효과 종류 — 전부 골드 건물 배치 시 적용
UENUM(BlueprintType)
enum class EEffectType : uint8 { FlatGold, IdleMult, FocusMult };

// 기록 출처 — 아카이브 태그의 소스
UENUM(BlueprintType)
enum class ERecordSource : uint8 { Timer, Stopwatch, Quest };
04 — DataTable 행 구조체 8종

밸런스 표 (ROW)

모두 FTableRowBase를 상속해야 DataTable의 행 타입으로 지정할 수 있습니다. 값은 에디터/CSV에서 채웁니다.

💡

코드 상자 표기 규칙: 아래 8종은 실제로 단일 헤더 RTTables.h 하나에 다 들어갑니다 — 한 상자에 담기엔 너무 길어서 표시 편의상 3개 상자(능력치·캐릭터/가챠·건물)로 나눈 것일 뿐입니다. 탭 이름이 파일명.h — 부제처럼 대시(—)로 이어지면 "같은 파일의 일부 구간", 파일A.h / 파일B.h처럼 슬래시(/)로 이어지면 "실제로 다른 두 파일"이라는 뜻입니다(05 섹션의 RTSaveGame.h / RTSettings.h 상자 참고).

능력치 — Ability_Type · Ability_Step

DT_Ability_Type · RowName = 효과×등급 (예 FocusMult_Legendary)

컬럼타입의미
DisplayNameFTextUI 표시 이름
AbilityRarityEAbilityRarity이 능력치의 등급 = 효과 tier
EffectTypeEEffectTypeflat 골드 / 유휴 배율 / 집중 배율
SelectWeightint32같은 등급 안에서 뽑힐 가중치

DT_Ability_Step · RowName = AbilityId + 단계 (예 FocusMult_Legendary_1) · 등급 높을수록 행(단계) 많음

컬럼타입의미
AbilityIdFName어느 능력치 (그룹 키 — 필드로도 둬 조회·정렬)
StepIndexint32몇 단계 (정렬 키)
Valuefloat그 단계 효과값 (직접 지정, 자동 등분 아님)
RollWeightint32뽑기 시 이 단계로 시작할 가중치
ShardCostToNextint32다음 단계로 올리는 조각 비용 (최대 단계는 미사용)
// DT_Ability_Type — 능력치 종류(효과×등급). RowName 예: FocusMult_Legendary
USTRUCT(BlueprintType)
struct FAbilityTypeRow : public FTableRowBase
{
    GENERATED_BODY()
    UPROPERTY(EditAnywhere) FText DisplayName;                        // UI 표시 이름
    UPROPERTY(EditAnywhere) EAbilityRarity AbilityRarity = EAbilityRarity::Normal; // 이 능력치의 등급 = 효과 tier
    UPROPERTY(EditAnywhere) EEffectType EffectType = EEffectType::FlatGold;  // flat/유휴/집중 중 무엇
    UPROPERTY(EditAnywhere) int32 SelectWeight = 1;                     // 같은 등급 내에서 뽑힐 가중치
};

// DT_Ability_Step — 능력치 단계별 값·확률·비용. RowName 예: FocusMult_Legendary_1
// 단계마다 한 행. 등급 높을수록 행(단계) 수가 많다.
USTRUCT(BlueprintType)
struct FAbilityStepRow : public FTableRowBase
{
    GENERATED_BODY()
    UPROPERTY(EditAnywhere) FName AbilityId;              // 어느 능력치(그룹 키) — 필드로도 둬 조회·정렬
    UPROPERTY(EditAnywhere) int32 StepIndex = 0;        // 몇 단계(정렬 키)
    UPROPERTY(EditAnywhere) float Value = 0.f;           // 그 단계 효과값(직접 지정, 자동 등분 아님)
    UPROPERTY(EditAnywhere) int32 RollWeight = 0;       // 뽑기 시 이 단계로 시작할 가중치
    UPROPERTY(EditAnywhere) int32 ShardCostToNext = 0;  // 다음 단계로 올리는 조각 비용(최대 단계는 0/미사용)
};

05 — SaveGame · 런타임 struct

내 소유물 (SAVE)

💡

코드 상자 표기 규칙(04 참고): RTSaveTypes.h 상자 2개(인스턴스 struct / 길드·기록 struct)도 같은 파일 하나를 표시 편의상 나눈 것입니다. 반면 맨 아래 RTSaveGame.h / RTSettings.h 상자는 실제로 다른 두 파일(URecordTalesSaveGame은 RTSaveGame.h, URecordTalesSettings은 RTSettings.h)을 한 상자에 함께 보여준 것 — 슬래시(/)가 그 표시입니다.

// 개화된 능력치 하나 — 현재 단계만 저장(값·비용은 DT_Ability_Step에서 조회)
USTRUCT()
struct FOwnedAbility
{
    GENERATED_BODY()
    UPROPERTY() FName AbilityId;        // → DT_Ability_Type / _Step
    UPROPERTY() int32 CurrentStep = 0; // 현재 강화 단계
};

// 내가 뽑아 가진 캐릭터 하나 — 중복 허용이라 InstanceId로 구분
USTRUCT()
struct FOwnedCharacter
{
    GENERATED_BODY()
    UPROPERTY() int32 InstanceId = 0;                    // 고유 번호(건물 배치가 이걸 참조)
    UPROPERTY() FName CharacterId;                        // → DT_Character
    UPROPERTY() ECharacterRarity Rarity = ECharacterRarity::Normal; // 뽑을 때 롤된 등급
    UPROPERTY() TArray<FOwnedAbility> Abilities;        // 개화된 능력치(최대 4)
};

// 내 경영 건물 하나
USTRUCT()
struct FBuildingState
{

    UPROPERTY() TArray<int32> DeployedInstanceIds; // 배치된 캐릭터 InstanceId(열린 슬롯만큼)
    UPROPERTY() FDateTime LastCollectTime;           // 유휴 누적 계산 기준(오프라인 캡)
};

06 — 등록 · 검증

표 등록과 확인

1
DataTable 에셋 생성
각 F*Row를 행 타입으로 지정해 DataTable 에셋을 만든다(에디터: Miscellaneous ▸ Data Table ▸ 행 구조체 선택). 값은 여기서/CSV로 채운다.
2
로드 시 캐시
단계·업그레이드처럼 (키+인덱스) 행은 서브시스템 Initialize에서 TMap<FName, TArray<Row>>로 그룹·정렬해 캐시한다(RowName 조합에 의존하지 않음).
3
SaveVersion 마이그레이션
SaveGame 구조를 바꾸면 SaveVersion += 1. 로드 시 버전 분기로 구 데이터를 새 구조로 변환(예: v2→v3 기록에 GuildId=기본 길드 백필).
⚠️

확인: DataTable 행 구조체는 반드시 FTableRowBase 상속. 확률 컬럼들의 합이 100인지 검증. 능력치 효과값을 SaveGame에 복사하지 말 것(단계만 저장, 값은 표 조회). RowName은 조합(AbilityId_Step)이되 조회는 필드(AbilityId·StepIndex)로.
