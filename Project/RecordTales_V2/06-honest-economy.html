01 — 개념

정직한 레벨

레벨은 실제 집중 시간에서만 옵니다. 60분(3600초) 집중 = 1레벨. 캐릭터의 능력치나 Gold로는 절대 못 올립니다. 그래서 같은 100시간을 집중한 두 사람은 가챠 운과 무관하게 같은 레벨이 되고, 레벨이 노력을 정직하게 반영합니다.

💡

레벨은 저장하지 않습니다. GetLevel() = FocusSeconds / 3600으로 매번 계산합니다. 저장된 레벨 값이 시간과 어긋날 여지가 없어, "레벨 = 투자 시간"의 정직함이 코드 수준에서 보장됩니다.

02 — 자원 흐름

Gold · 조각 · XP

자원마다 나오는 곳(source)과 쓰이는 곳(sink)이 하나씩 분명합니다.

자원나오는 곳쓰이는 곳(sink)
Gold활동(집중/스톱워치/퀘스트) 기본 + 골드 건물(유휴 + 집중 N배)가챠 · 건물 업그레이드 · 해금
조각캐릭터 추출(기억 거래소)능력치 단계 강화
XP(=시간)집중/스톱워치 시간 (퀘스트는 XP 없음)길드 레벨(per-guild) + 계정 XP(가챠 확률)
🧭

세션 한 번의 시간은 두 곳으로. 집중 N초 완료 → 그 길드 레벨(FocusSeconds)에 + 계정 XP(AccountFocusSeconds)에 동시 적립. 퀘스트는 Gold만 주고 XP는 주지 않습니다(레벨을 부풀리지 않기 위해).

03 — 레벨 규칙

선형 · 무제한 · plateau

1
길드 레벨 = 시간 ÷ 3600
선형(60분=1레벨), 무제한(천장 없음). 레벨이 곧 그 도메인 투자 시간의 거울. 마일스톤 칭호를 얹을 수 있음.
2
계정 XP = 전 길드 시간 합
모든 길드의 집중 시간을 합한 값. 이 계정 레벨이 가챠 등급 출현 확률을 결정(DT_Gacha_Rate_ByLevel).
3
가챠 확률은 plateau
계정 레벨이 무한히 올라도 확률이 무한히 좋아지면 안 되므로, 어느 레벨부터 확률이 고정됩니다(표의 상한 이후 flat). 무한 시간이 무한 확률로 가지 않게.
⚠️

레벨을 Gold로 사지 못합니다. v1엔 "레벨업에 Gold 소비"가 있었지만 V2에선 제거 — Gold로 레벨을 사면 레벨이 지갑의 거울이 되어 정직함이 깨집니다. 레벨의 유일한 입력은 시간입니다.

04 — 해금 트리

Gold로 여는 것

Gold의 sink 중 하나는 해금입니다. 무엇을 여는지의 방향은 아래와 같습니다(구체 비용은 튜닝).

해금 대상내용
새 골드 건물 / 테마음식점·상점 등 새 경영 건물, 캐릭터 테마 매칭 니치 확장
기억 거래소 확장추출·조각 관련 확장
배경 / BGM 꾸미기영지·홈 꾸미기(몰입·개인화)

※ 해금 대상·비용은 계속 다듬는 항목입니다. 해금 여부는 세이브의 UnlockedIds(FName 집합)로 관리.

05 — 스켈레톤

뼈대 코드

경제 서브시스템의 뼈대입니다. 핵심은 시간을 두 곳에 적립하고 레벨은 파생으로 두는 부분입니다.

UCLASS()
class UEconomySubsystem : public UGameInstanceSubsystem
{
    GENERATED_BODY()
public:
    // 활동 완료 시: 그 길드 시간 + 계정 시간 양쪽에 적립 (레벨은 파생·저장 안 함)
    void AddFocusTime(int32 GuildId, int32 Seconds);

    // 활동/퀘스트 보상 Gold를 공유 지갑에 지급
    void AddGold(int64 Amount);

    // 계정 레벨 = 계정 총 시간 / 3600 (가챠 확률 입력)
    int32 GetAccountLevel() const;

    // Gold 지불 후 해금 id 등록 (부족하면 false)
    bool Unlock(FName UnlockId, int64 Cost);
};
// 세션 한 번의 시간을 두 곳으로 라우팅한다 — 이게 '정직한 레벨 + 계정 가챠'의 핵심
void UEconomySubsystem::AddFocusTime(int32 GuildId, int32 Seconds)
{
    // 1) 그 길드의 누적 시간 — 길드 레벨의 원천 (GetLevel이 여기서 파생)
    if (FGuildData* G = FindGuild(GuildId))
        G->FocusSeconds += Seconds;

    // 2) 계정 총 시간 — 가챠 확률의 입력 (전 길드 합)
    Save->AccountFocusSeconds += Seconds;

    // 레벨 값은 어디에도 저장하지 않는다 — 항상 시간에서 계산해 정직함을 보장
}

// 계정 레벨 = 총 시간 / 3600 (60분=1레벨). 무제한이지만 가챠 확률표는 상한에서 plateau.
int32 UEconomySubsystem::GetAccountLevel() const
{
    return Save->AccountFocusSeconds / 3600;
}
06 — 확장

더 얹을 때

✅

이렇게: 새 Gold 획득처를 더할 땐 AddGold로 공유 지갑에 넣는다. 새 해금 대상은 UnlockedIds에 FName으로 추가하고 비용은 밸런스로 둔다. 가챠 확률표는 계정 레벨 상한(plateau 지점)까지만 개선되게 짠다.

⚠️

이건 피하기: 레벨을 저장하거나 Gold·캐릭터로 올리지 말 것 — 정직함이 깨진다. 퀘스트에 XP 보상을 주지 말 것(Gold만). 가챠 확률을 무한 상승시키지 말 것 — plateau 필수.
