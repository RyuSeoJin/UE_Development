개화

캐릭터를 뽑아 능력치 슬롯을 개화시키고, 조각으로 강화하는 수집 시스템입니다. 중복 허용·보유 무제한이며, 스탯까지 보이는 3장 후보 중 1택으로 뽑습니다.

능력치 개화 (슬롯 ≤4) 3장 미리보기 1택 02-A 아키텍처 패턴
01 — 개념

능력치 개화란

캐릭터는 능력치 슬롯을 최대 4개 가질 수 있고, 뽑을 때 그중 몇 개가 "개화"됩니다. 개화된 슬롯마다 하나의 능력치가 자리잡습니다. 얼마나 개화되고 어떤 등급의 능력치가 뜨는지는 캐릭터 등급이 결정합니다.

⚠️

등급이 둘입니다 — 헷갈리지 마세요. 캐릭터 등급은 뽑을 때 롤되며, (a) 개화 슬롯 수와 (b) 뜨는 능력치의 등급 분포를 결정합니다. 능력치 등급은 각 능력치 자체의 등급으로, 그 능력치의 효과 크기(계수 tier)를 정합니다.

보유는 무제한·중복 허용입니다. 뽑기는 스탯까지 완성된 3장 후보 중 하나를 반드시 고르는 방식(스킵·리롤 없음)이라, 눈에 보이는 best 선택이 운의 완충 역할을 합니다. 그래서 별도 천장(pity)이 없습니다.

💡

엔드게임 = 무한 롤 체이스가 아니라 "등급 획득 + 상한까지". 롤은 능력치의 시작 단계를 정하고, 조각 강화가 상한까지 끌어올립니다. 좋게 뜨면 Best, 아니면 조각으로 도달 — 즉 롤은 "얼마나 적은 조각으로 도달하나"의 속도 차이입니다.

02 — 구성

등급 · 능력치 · 조각

요소내용
캐릭터 등급뽑을 때 롤(인스턴스). 계정 레벨이 출현 확률을 결정(높을수록 고등급↑, plateau). 개화 수·능력치 등급 분포를 정함.
능력치 종류3종: flat 골드 보너스 · 유휴 배율 · 집중 배율(전부 경영 골드 건물 배치용).
능력치 등급일반~레전더리. 같은 효과라도 등급이 계수 tier(효과 범위)를 정함(예: 집중배율 노말 0.5~1.5 → 레전더리 3~4).
능력치 단계능력치마다 여러 단계(고등급일수록 많음). 롤이 시작 단계, 조각으로 상한까지 강화.
조각캐릭터 추출(기억 거래소)로 획득. 능력치 단계를 올리는 데 씀.

※ 실제 확률·계수·비용 수치는 DataTable(03 문서 참조)에서 튜닝합니다.

03 — 뽑기 흐름

후보 1장이 만들어지는 순서

아래를 3번 반복해 완성된 3장을 보여주고, 플레이어가 하나를 고릅니다.

1
캐릭터 선택 + 등급 롤
풀에서 균등 랜덤으로 캐릭터 타입을 고르고, 계정 레벨로 캐릭터 등급을 롤합니다(같은 캐릭터가 노말로도 레전더리로도 나올 수 있음).
2
개화 슬롯 수 결정
캐릭터 등급으로 4슬롯 중 몇 개를 개화할지 정합니다(고등급일수록 많이 개화될 확률↑).
3
슬롯마다 능력치 확정
개화된 슬롯마다: 능력치 등급 롤 → 그 등급 능력치 중 하나 선택 → 시작 단계 롤(RollWeight). 능력치의 효과값이 정해집니다.
4
3장 제시 → 1택
이렇게 만든 완성 후보 3장을 스탯까지 보여주고, 플레이어가 하나를 선택합니다. 나머지는 버려집니다(스킵·리롤 없음).

04 — 강화

조각으로 단계 올리기

조각은 캐릭터의 개화된 능력치 하나를 다음 단계로 올립니다. 어떤 능력치가 오를지는 직접 고르지 않고 가중 랜덤입니다.

1
대상 = 가중 랜덤
아직 최대가 아닌 개화 능력치들 중, 가중치 = 남은 단계(상한 − 현재)로 하나를 뽑습니다. 덜 오른 능력치가 잘 뽑혀 고르게 성장합니다.
2
비용 지불 → +1단계
그 능력치 현재 단계의 조각 비용을 지불하고 단계를 +1합니다. 효과값은 저장하지 않고 단계에서 표로 조회하므로, 세이브는 단계만 바뀝니다.
🧭

롤과 강화의 관계. 롤은 시작 단계(운 좋으면 높게)를, 강화는 상한까지 올림을 담당합니다. 같은 능력치라도 상한은 그 능력치 등급이 정하므로, 더 좋은 캐릭터를 원하면 결국 고등급 능력치를 뽑아야 합니다(등급이 파워 게이트).

💡

코드 외 작업 — ① DT_Character(캐릭터 원형)·DT_Gacha(풀·확률)·DT_Ability(능력치 범위)에 실제 행 데이터 입력 — 뽑기가 "빈 결과"를 내면 십중팔구 테이블이 비어 있는 것 ② 확률·범위 튜닝은 코드 수정 없이 테이블 편집만으로. ③ 뽑기 연출 UI(3장 카드)는 별도 UMG 작업 — 이 문서는 로직까지만.

⚠️
컴파일 주의 — FOwnedCharacter 추가·시그니처 변경은 재빌드 코스(에디터 종료 → IDE 빌드), 확률 로직 등 .cpp 본문 반복 수정은 Live Coding으로. 검증: PIE 콘솔에서 뽑기 함수를 호출해 UE_LOG로 3장의 능력치가 매번 다르게 찍히는지 확인.

뽑기 생성과 강화의 뼈대입니다. 표 조회 함수(Roll*·Lookup*)의 실제 구현은 DataTable을 읽는 부분이라 생략합니다.

후보 1장 생성 (3번 호출)

// 후보 1장을 완성해 반환. 뽑기 UI는 이걸 3번 호출해 3장을 보여준다.
FOwnedCharacter UGachaSubsystem::GenerateCandidate()
{
    FOwnedCharacter C;
    C.CharacterId = PickRandomCharacter();               // 풀에서 균등 랜덤(등급 무관)
    C.Rarity      = RollRarity(GetAccountLevel());        // DT_Gacha_Rate_ByLevel[계정 레벨]

    int32 OpenCount = RollOpenCount(C.Rarity);            // DT_..._Open_Chance[등급] → 개화 슬롯 수
    for (int32 i = 0; i < OpenCount; ++i)           // 개화된 슬롯마다
    {
        EAbilityRarity AR = RollAbilityRarity(C.Rarity); // DT_..._Select_Chance[등급] → 능력치 등급
        FName AbilityId   = PickAbility(AR);           // DT_Ability_Type(등급 내 SelectWeight)
        int32 StartStep   = RollStartStep(AbilityId);   // DT_Ability_Step(RollWeight로 시작 단계)
        C.Abilities.Add({ AbilityId, StartStep });     // 개화된 능력치 추가(최대 4)
    }
    return C;
}

능력치 강화 (조각)

// 조각으로 한 캐릭터의 능력치를 한 단계 올린다. 대상은 가중 랜덤.
void UGachaSubsystem::UpgradeAbility(FOwnedCharacter& C)
{
    // 아직 최대가 아닌 개화 능력치들 중 가중 랜덤(가중치 = 남은 단계 = 상한 − 현재)
    int32 Pick = WeightedPickByRemainingSteps(C.Abilities);
    if (Pick == INDEX_NONE) return;                    // 전부 최대면 강화 불가

    FOwnedAbility& A = C.Abilities[Pick];
    // 비용은 능력치 등급으로 결정(단계 무관). AbilityId → DT_Ability_Type[등급] → DT_Ability_UpgradeCost[등급]
    int32 Cost = LookupUpgradeCost(A.AbilityId); // DT_Ability_UpgradeCost.Upgrade_ShardAmount
    if (!SpendShards(Cost)) return;                  // 조각 부족 시 중단

    A.CurrentStep += 1;                             // 다음 단계로 (효과값은 표에서 파생, 저장 안 함)
}
