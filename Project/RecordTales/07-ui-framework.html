01 — 개념 이해

UI 프레임워크란

UI 프레임워크는 RecordTales의 전체 UI를 관리하는 계층적 시스템입니다. RecordTales는 전통적인 3D 게임이 아니라 UI 중심 앱이므로, 거의 모든 사용자 상호작용이 UMG(Unreal Motion Graphics: 언리얼의 UI 제작 시스템) 위젯으로 구현됩니다. 핵심 관리자는 UUIManagerSubsystem(UGameInstanceSubsystem을 상속하여 프로그램 수명 동안 유지)이며, 패널 전환, 모달 스택, 토스트 알림을 통합 관리합니다.

핵심 용어

📚

패널 매니저(Panel Manager) — Focus/Quests/Archive 세 패널 중 하나만 열리도록 관리하는 시스템. 네비게이션 바 버튼으로 전환합니다.
모달 스택(Modal Stack) — 패널 위에 띄우는 대화 상자(길드 관리, 설정 등)를 스택(후입선출) 구조로 관리. Push로 쌓고 Pop으로 제거합니다.
토스트(Toast) — 화면 하단 중앙에 1~3초간 표시 후 자동으로 사라지는 짧은 알림 메시지.
캘린더 위젯(Calendar Widget) — 월별 날짜를 표시하고 선택할 수 있는 재사용 가능한 공통 위젯. 퀘스트(시작일/마감일 선택)와 아카이브(날짜 탐색)에서 사용됩니다.
HUD 레이아웃 — 화면 네 코너와 중앙에 위젯을 배치하는 구조. 각 코너는 독립 위젯으로 분리되어 재사용 가능합니다.

📌

RecordTales는 전통적인 3D 게임이 아닌 UI 중심 앱입니다. 거의 모든 로직이 UMG(Unreal Motion Graphics) 위젯으로 구현됩니다. 3D 월드는 배경 역할만 하며, 사용자 상호작용은 전부 위젯 버튼/슬라이더/텍스트를 통해 이루어집니다.

02 — HUD 레이아웃

4코너 + 중앙 구성

HUD(Head-Up Display: 게임 화면 위에 상시 표시되는 UI 레이어)는 화면을 4코너 + 중앙으로 나누고, 각 영역에 독립 위젯을 배치합니다. 이렇게 분리하면 특정 영역만 수정해도 다른 영역에 영향을 주지 않습니다.

레이아웃 구조

┌─────────────────┬─────────────────┐ │ Top-Left │ Top-Right │ │ 길드정보 │ 시계 · Nav · 미니 │ │ 오늘의퀘스트 │ │ ├─────────────────┴─────────────────┤ │ │ │ Center │ │ 시길 (문장+격언) │ │ │ ├─────────────────┬─────────────────┤ │ Bottom-Left │ Bottom-Right │ │ 앰비언트 │ Command Post │ │ 음악 플레이어 │ (모집/관리/해금/설정)│ └─────────────────┴─────────────────┘

영역별 위젯 클래스

영역위젯 클래스역할
Top-LeftUGuildInfoWidget길드 이름, 레벨, 칭호, Gold 잔액, XP 진행 바를 표시합니다.
Top-LeftUTodayQuestWidget오늘의 할 일 목록을 접기/펼치기 형태로 표시합니다.
Top-RightUClockWidget현재 시각, 날짜, 요일을 실시간으로 표시합니다.
Top-RightUSessionMiniWidget타이머/스톱워치가 실행 중일 때 남은 시간과 현재 단계를 축소 표시합니다.
Top-RightUNavBarWidgetFocus/Quests/Archive 버튼 3개. 클릭 시 해당 패널을 엽니다.
Bottom-LeftUAmbientWidget5종 앰비언트 볼륨 슬라이더와 글로벌 토글 버튼.
Bottom-LeftUMusicPlayerWidget앨범아트, 재생 바, Play/Pause/이전/다음 버튼, 볼륨.
Bottom-RightUCommandPostWidget모집/관리/해금/설정 버튼 4개. 클릭 시 해당 모달을 엽니다.
CenterUSigilWidget길드 문장(이미지)과 격언 텍스트를 화면 중앙에 표시합니다.
💡

각 영역을 독립 위젯으로 분리하면, HUD 레이아웃을 변경해도 개별 위젯은 그대로 재사용 가능합니다. 예를 들어 UMusicPlayerWidget은 Bottom-Left에서 떼어 다른 위치에 놓아도 동작이 바뀌지 않습니다.

03 — 패널 매니저

Focus / Quests / Archive 패널

패널 매니저(앞서 설명한 단일 패널 전환 시스템)는 3개 패널을 관리하며, 한 번에 하나만 열리는 Single Overlay 방식을 사용합니다. 패널은 HUD 위에 전체 화면으로 덮이며, 닫으면 HUD로 돌아갑니다.

패널 전환 흐름

1
Nav 버튼 클릭
Top-Right의 UNavBarWidget에서 Focus/Quests/Archive 버튼 중 하나를 누르면 UUIManagerSubsystem::OpenPanel(EPanel)이 호출됩니다. EPanel은 Focus, Quests, Archive 세 값을 가진 열거형(UENUM)입니다.
2
현재 패널 닫기
이미 열려 있는 패널이 있으면 RemoveFromParent()(위젯을 화면에서 제거하는 UMG 함수)를 호출하여 먼저 닫습니다. 패널 위젯의 참조를 nullptr로 초기화합니다.
3
새 패널 위젯 생성
CreateWidget<T>(GetWorld(), PanelClass)로 요청된 패널의 위젯 인스턴스를 생성합니다. PanelClass는 EPanel 값에 매핑된 TSubclassOf<UUserWidget>입니다.
4
Viewport에 추가
AddToViewport(ZOrder)로 화면에 표시합니다. 패널의 ZOrder(렌더링 우선순위)는 10으로 설정하여 HUD 위에 표시되게 합니다.
5
배경(Backdrop) 표시
패널 뒤에 반투명 배경을 깔아 HUD가 흐리게 보이도록 합니다. 배경 클릭 시 패널이 닫힙니다.
6
현재 패널 상태 갱신
CurrentPanel 변수를 새 패널의 EPanel 값으로 갱신합니다. 닫힐 때는 None으로 설정됩니다.

패널 닫기는 ESC 키 또는 배경(Backdrop) 클릭으로 트리거됩니다. 닫을 때는 위젯 제거 + 배경 숨김 + CurrentPanel = None 순서입니다.

패널별 위젯 클래스

패널위젯 클래스역할
FocusUFocusPanelWidget포모도로 타이머와 스톱워치의 시간/반복 횟수 설정 + 실행/일시정지/중지 버튼. UFocusSubsystem과 바인딩됩니다.
QuestsUQuestPanelWidget할 일 목록 표시, 새 퀘스트 추가, 편집, 상세 보기, 반복 규칙 설정/해제. UQuestSubsystem과 바인딩됩니다.
ArchiveUArchivePanelWidget캘린더 위젯 + 선택한 날짜의 집중 기록과 완료된 퀘스트 표시. URecordSubsystem과 바인딩됩니다.
⚠️

패널을 열 때 기존 패널을 먼저 닫지 않으면 위젯이 중첩됩니다. 반드시 Close → Open 순서를 지키세요. OpenPanel() 함수 내부에서 자동으로 기존 패널을 닫는 로직을 넣는 것이 안전합니다.

04 — 모달 스택

Push / Pop 모달 관리

모달(Modal: 현재 화면 위에 떠서 특정 작업을 수행하는 대화 상자)은 패널 위에 표시됩니다. 스택(Stack) 구조로 관리하며, Push(쌓기)로 모달을 추가하고 Pop(빼기)으로 제거합니다. 최상단 모달만 사용자 입력을 수신하고, 그 아래 모달/패널은 입력이 차단됩니다.

모달 목록

모달위젯 클래스트리거
길드원 모집URecruitmentModalWidgetCommand Post의 "모집" 버튼
길드원 관리UManagementModalWidgetCommand Post의 "관리" 버튼
길드원 상세UMemberDetailModalWidget관리 모달에서 멤버 클릭
길드원 선택UMemberPickerModalWidget관리 모달에서 슬롯 클릭
해금 샵UUnlockShopModalWidgetCommand Post의 "해금" 버튼
설정USettingsModalWidgetCommand Post의 "설정" 버튼
기록 저장URecordModalWidget포커스 세션 종료 시 자동
카테고리 추가UCategoryAddModalWidget기록/퀘스트에서 카테고리 "+" 버튼
뮤직 라이브러리UMusicLibraryModalWidget음악 플레이어 라이브러리 버튼

Push / Pop 흐름

1
Push — 모달 열기
CreateWidget<T>()로 모달 위젯을 생성합니다. → ModalStack.Push(Widget)으로 스택에 추가합니다. → AddToViewport(100 + StackIndex)로 화면에 표시합니다. ZOrder는 100부터 시작하여 스택 깊이만큼 증가하므로 항상 최상단이 위에 보입니다. → 이전 모달(있다면)의 입력을 차단합니다(SetVisibility 또는 SetIsEnabled(false)).
2
Pop — 모달 닫기
ModalStack.Pop()으로 최상단 위젯을 꺼냅니다. → RemoveFromParent()로 화면에서 제거합니다. → 이전 모달(있다면)의 입력을 복원합니다(SetIsEnabled(true)). 스택이 비면 패널 또는 HUD로 포커스가 돌아갑니다.
3
PopAll — 전체 비우기
스택의 모든 모달을 순서대로 Pop합니다. 패널을 닫을 때나 특수 상황(앱 백그라운드 진입 등)에서 사용합니다.
📌

모달 위의 모달이 가능합니다. 예: Command Post → "관리" 모달 Push → 멤버 클릭 → "상세" 모달 Push. 뒤로가기(ESC)를 누르면 "상세"가 Pop되고 "관리"가 다시 활성화됩니다.

05 — 토스트 알림

자동 사라지는 알림

토스트(앞서 설명한 짧은 알림 메시지)는 화면 하단 중앙에 표시되며, 설정된 시간(기본 2초) 후 자동으로 사라집니다. 여러 토스트가 동시에 요청되면 큐(Queue) 방식으로 하나씩 순차 표시합니다.

항목타입설명
MessageFString표시할 텍스트. 예: "Gold가 부족합니다", "배경이 변경되었습니다".
Durationfloat표시 시간(초). 기본값 2.0. 짧은 알림은 1.5, 중요 알림은 3.0 등 조절 가능.

ShowToast 코드 골격

UIManagerSubsystem.cpp — ShowToast 핵심 흐름 
void UUIManagerSubsystem::ShowToast(const FString& Message, float Duration)
{
    // 1. 토스트 위젯 생성
    UToastWidget* Toast = CreateWidget<UToastWidget>(GetWorld(), ToastClass);

    // 2. 메시지 설정
    Toast->SetMessage(Message);

    // 3. 화면에 추가 (ZOrder 200 = 모달보다 위)
    Toast->AddToViewport(200);

    // 4. Duration 후 자동 제거 (FTimerHandle 사용)
    FTimerHandle Handle;
    GetWorld()->GetTimerManager().SetTimer(
        Handle, [Toast]()
        {
            if (IsValid(Toast))
                Toast->RemoveFromParent();
        },
        Duration, false
    );
}
06 — 캘린더 위젯

재사용 가능한 날짜 선택기

캘린더 위젯(앞서 설명한 월별 날짜 선택 컴포넌트)은 범용 위젯으로 설계되어, 퀘스트 패널(시작일/마감일/반복종료일 선택)과 아카이브 패널(날짜 탐색)에서 재사용됩니다. 위젯은 자신이 어떤 목적으로 쓰이는지 알 필요가 없으며, 날짜가 선택되면 델리게이트(콜백 시스템)로 부모에게 알려줍니다.

위젯 파라미터

파라미터타입설명
Yearint32현재 표시 중인 연도. 초기값은 오늘의 연도.
Monthint32현재 표시 중인 월(1~12). 초기값은 오늘의 월.
SelectedDateFString사용자가 선택한 날짜 문자열(YYYY.MM.DD 형식). 선택 전에는 빈 문자열.
bShowDotsbool날짜 셀에 도트(점)를 표시할지 여부. 아카이브 용도에서 기록이 있는 날짜에 점을 찍어 한눈에 알 수 있게 합니다.
OnDateSelectedFOnDateSelected
(델리게이트)		날짜를 클릭했을 때 호출되는 콜백. 부모 위젯이 이 델리게이트에 함수를 바인딩하여 선택 결과를 받습니다.

동작 방식

월 네비게이션: "이전 월"/"다음 월" 버튼을 누르면 Year/Month를 갱신하고 달력을 재렌더링합니다. 1월에서 이전을 누르면 전년도 12월로, 12월에서 다음을 누르면 다음 해 1월로 이동합니다.

날짜 셀 배치: 7열(일~토) × 6행 그리드로 배치합니다. 해당 월에 속하지 않는 날짜(이전/다음 월)는 흐리게(SetColorAndOpacity으로 투명도 조절) 표시하여 구분합니다.

날짜 선택: 셀을 클릭하면 SelectedDate를 갱신하고 OnDateSelected 델리게이트를 Broadcast합니다. 부모 위젯은 이 신호를 받아 자신의 로직(퀘스트 마감일 설정, 아카이브 필터링 등)을 수행합니다.

💡

델리게이트로 날짜 선택 결과를 부모에게 전달하면, 캘린더가 어떤 목적으로 쓰이는지 알 필요 없이 재사용 가능합니다. 퀘스트에서 쓸 때는 퀘스트 패널이 콜백을 등록하고, 아카이브에서 쓸 때는 아카이브 패널이 콜백을 등록합니다. 캘린더 위젯 코드는 동일합니다.

07 — 구현 스켈레톤

UUIManagerSubsystem 최소 코드

아래는 UUIManagerSubsystem의 최소 동작 뼈대입니다. 헤더(.h)에 인터페이스를 선언하고, 소스(.cpp)에 PushModal/PopModal 스택 로직의 구현 골격을 보여줍니다.

#pragma once

#include "CoreMinimal.h"
#include "Subsystems/GameInstanceSubsystem.h"
#include "Blueprint/UserWidget.h"
#include "UIManagerSubsystem.generated.h"

// 패널 종류
// UENUM(BlueprintType): 블루프린트에서도 이 열거형을 쓸 수 있게 노출 (uint8 기반 필수)
UENUM(BlueprintType)
enum class EPanel : uint8
{
    None,      // 아무 패널도 안 열린 상태 (기본값)
    Focus,
    Quests,
    Archive
};

// UGameInstanceSubsystem 상속: 게임 실행 내내(레벨 전환에도) 유지되는 전역 UI 관리자
UCLASS()
class RECORDTALES_API UUIManagerSubsystem : public UGameInstanceSubsystem
{
    GENERATED_BODY()

public:
    // 엔진이 서브시스템 생성 직후 자동 호출 (직접 부르지 않음)
    virtual void Initialize(FSubsystemCollectionBase& Collection) override;

    // ── 패널 ──
    // BlueprintCallable: 블루프린트 그래프에서 노드로 호출 가능. Category: 노드 팔레트 분류 경로
    UFUNCTION(BlueprintCallable, Category = "UI|Panel")
    void OpenPanel(EPanel Panel);

    UFUNCTION(BlueprintCallable, Category = "UI|Panel")
    void ClosePanel();

    // ── 모달 ──
    UFUNCTION(BlueprintCallable, Category = "UI|Modal")
    void PushModal(TSubclassOf<UUserWidget> ModalClass);

    UFUNCTION(BlueprintCallable, Category = "UI|Modal")
    void PopModal();

    UFUNCTION(BlueprintCallable, Category = "UI|Modal")
    void PopAllModals();

    // ── 토스트 ──
    // Duration 기본값 2.0f초 — 호출부에서 생략하면 2초 뒤 자동 사라짐
    UFUNCTION(BlueprintCallable, Category = "UI|Toast")
    void ShowToast(const FString& Message, float Duration = 2.0f);

private:
    // 패널
    EPanel CurrentPanel = EPanel::None;   // 현재 열린 패널. None으로 시작 (아무것도 안 열림)
    // UPROPERTY(): GC가 이 위젯 참조를 추적하게 해 재사용 중 회수되지 않도록 보호 (없으면 크래시)
    UPROPERTY()
    UUserWidget* ActivePanelWidget = nullptr;   // nullptr 초기화 필수 — 열린 패널 없음을 의미

    // 모달 스택
    // UPROPERTY(): 배열 안 모든 위젯을 GC 추적 대상으로. 마지막 원소(Last)가 최상단 모달
    UPROPERTY()
    TArray<UUserWidget*> ModalStack;

    // 토스트 위젯 클래스 (에디터에서 설정)
    // EditDefaultsOnly: 인스턴스가 아닌 "클래스 기본값"에서만 편집 가능 → BP 에디터에서 토스트 위젯 지정
    UPROPERTY(EditDefaultsOnly, Category = "UI")
    TSubclassOf<UUserWidget> ToastClass;

    // ZOrder = 화면 겹침 순서(클수록 위). 패널·모달·토스트 구간을 분리해 입력 충돌 방지
    static constexpr int32 PanelZOrder = 10;        // 패널: HUD(0) 위
    static constexpr int32 ModalBaseZOrder = 100;   // 모달: 100부터 스택 깊이만큼 가산
};
#include "UIManagerSubsystem.h"
#include "Blueprint/UserWidget.h"

void UUIManagerSubsystem::Initialize(FSubsystemCollectionBase& Collection)
{
    Super::Initialize(Collection);
    // HUD 위젯 초기 생성은 GameMode 또는 PlayerController에서 수행
}

void UUIManagerSubsystem::PushModal(TSubclassOf<UUserWidget> ModalClass)
{
    if (!ModalClass) return;   // 클래스가 비었으면 CreateWidget이 실패하므로 조기 반환

    // 1) 이전 최상단 모달의 입력을 차단 (뒤 모달이 클릭을 가로채지 않도록 → 진짜 모달 동작)
    if (ModalStack.Num() > 0)
    {
        UUserWidget* Top = ModalStack.Last();   // Last = 스택 최상단(가장 최근에 연 모달)
        if (IsValid(Top))   // GC 회수·파괴된 위젯 접근 방지
            Top->SetIsEnabled(false);
    }

    // 2) 새 모달 위젯 생성 (GetWorld의 컨텍스트에 소속시킴)
    UUserWidget* Modal = CreateWidget<UUserWidget>(
        GetGameInstance()->GetWorld(), ModalClass);

    if (!Modal) return;   // 생성 실패 시 스택 오염 방지

    // 3) 스택에 추가 후 화면에 표시
    ModalStack.Push(Modal);
    // ZOrder = 100 + 스택 개수 → 나중에 쌓일수록 큰 값 = 항상 위에 그려짐
    int32 ZOrder = ModalBaseZOrder + ModalStack.Num();
    Modal->AddToViewport(ZOrder);
}

void UUIManagerSubsystem::PopModal()
{
    // 빈 스택에서 Pop 시도 방지 — 가드 없으면 TArray::Pop이 빈 배열 접근으로 크래시
    if (ModalStack.Num() == 0) return;

    // 1) 최상단 모달을 스택에서 꺼내 화면에서 제거
    UUserWidget* Top = ModalStack.Pop();
    if (IsValid(Top))   // 이미 GC/파괴됐을 수 있어 확인 후 제거
        Top->RemoveFromParent();

    // 2) 아래 모달이 남았으면 그 입력을 복원 (Push에서 꺼둔 것을 되살림)
    if (ModalStack.Num() > 0)
    {
        UUserWidget* NewTop = ModalStack.Last();
        if (IsValid(NewTop))
            NewTop->SetIsEnabled(true);
    }
}

void UUIManagerSubsystem::PopAllModals()
{
    while (ModalStack.Num() > 0)
    {
        PopModal();
    }
}

Pitfall

⚠️
모달 Pop 시 nullptr 체크 필수 — 빈 스택에서 Pop()을 호출하면 크래시가 발생합니다. PopModal() 함수 시작부에 if (ModalStack.Num() == 0) return; 가드를 반드시 넣으세요. 또한 Pop된 위젯이 이미 GC(가비지 컬렉션)에 의해 해제된 경우를 대비하여 IsValid(Top) 체크도 필요합니다.
⚠️
패널과 모달의 ZOrder 충돌 주의 — 패널(ZOrder=10)과 모달(ZOrder=100+)이 겹치면 입력 이벤트가 의도치 않은 위젯으로 전달될 수 있습니다. 패널은 10, 모달은 100부터 시작, 토스트는 200으로 명확히 구간을 분리하세요.
⚠️
토스트 위젯 제거 시 IsValid 체크 — FTimerHandle의 콜백이 실행될 시점에 위젯이 이미 GC되었을 수 있습니다(레벨 전환 등). 콜백 내부에서 반드시 if (IsValid(Toast))를 확인한 후 RemoveFromParent()를 호출하세요.

Do & Don't

✓ 이렇게

  • 패널은 Open 전에 반드시 기존 패널 Close (Close → Open 순서)
  • 모달 ZOrder를 100+StackIndex로 자동 계산 (순서 보장)
  • 스택 Pop 전 Num() == 0 가드 + IsValid 체크
  • 토스트 타이머 콜백에서 IsValid 체크 후 RemoveFromParent
  • 캘린더 위젯은 델리게이트로 날짜 선택 결과 전달 (재사용성)

✗ 이건 피하기

  • 패널과 모달을 같은 ZOrder에 배치 (입력 충돌)
  • 모달 스택을 무시하고 직접 AddToViewport (순서 관리 불가)
  • 빈 스택에서 Pop 호출 (크래시)
  • 토스트 위젯을 수동으로 제거하지 않기 (메모리 누수)
  • 캘린더가 부모의 로직을 직접 호출 (결합도 증가)
