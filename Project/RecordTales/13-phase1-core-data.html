02 — 설계 결정

어떤 데이터를 어느 파일에, 왜 두는가

02-B의 심장인 책임 배치 결정표입니다. 12번 지도의 소유 결정을 Phase 1 산출물에 맞게 확정합니다.

데이터/타입배치근거
공용 열거형 · FArchiveRecord · 행 구조체 7종Data/RecordTalesTypes.h여러 도메인이 공유 — 한 곳에 모아 순환 include 차단 (12 지도의 Data/ 원칙)
퀘스트 구조체 2종 / 길드 구조체 3종Quests/QuestTypes.h · Guild/GuildTypes.h도메인 소속 타입은 도메인 폴더에(12 지도) — SaveGame(Core)이 이 헤더를 include하는 것은 단방향이라 순환이 없음(타입 헤더는 Core를 모름)
레거시 EFocusMode·FFocusRecordData/RecordTalesTypes.h (deprecated 표기)v1 세이브 마이그레이션(11 문서)이 읽어야 하므로 삭제 불가 — 이전 완료 후 별도 승인으로 제거
밸런스 상수 21종RecordTalesSettings기획 조정 값 — 코드 재컴파일 없이 에디터에서 (첫 실행 기본값의 원천)
플레이어 상태 29필드RecordTalesSaveGame전부 "사본" — 런타임 주인은 Phase 2~4의 각 서브시스템 (12 지도)
저장/로드 왕복 · 첫 실행 주입RecordTalesGameInstance앱과 수명이 같은 유일한 객체 — 디스크 왕복 창구 단일화
💡
마이그레이션 코드가 여기 없는 이유 — 새 프로젝트의 세이브는 SaveVersion=2로 시작하므로 v1→v2 이전 코드는 Phase 1에 필요 없습니다. 그 코드의 자리는 11 문서가 정한 대로 RecordSubsystem(Phase 3)입니다 — GameInstance는 버전을 확인하고 로그만 남깁니다.
💡
신뢰 시간(치트 방지) 필드는 예약만 — 위키 신뢰 시간 문서를 채택하면 SaveGame에 LastApprovedTrustedUtc·LastLocalDateKey 2필드가 추가됩니다(스팀 연동 시점, 08 문서 개정과 함께). 필드 추가는 기본값으로 안전하므로 SaveVersion을 올릴 필요 없습니다 — 지금은 넣지 않습니다.
03 — 파일 구성

7개 파일과 include 방향

Source/RecordTales/ ├── Core/ │ ├── RecordTalesSaveGame.h ← Types + QuestTypes + GuildTypes를 include (단방향) │ └── RecordTalesGameInstance.h/.cpp ← SaveGame + Settings를 include ├── Quests/ │ └── QuestTypes.h ← Types만 include (Core를 모름) ├── Guild/ │ └── GuildTypes.h ← Types만 include └── Data/ ├── RecordTalesTypes.h ← 아무도 include하지 않음 (의존 사슬의 뿌리) └── RecordTalesSettings.h ← .cpp 불필요 (UHT가 구현 생성)

include 화살표는 항상 Core → 도메인 타입 → Data 한 방향입니다. 거꾸로 가는 include가 하나라도 생기면(예: Types가 SaveGame을 include) 순환이 시작되니, 컴파일 에러가 나면 이 방향부터 확인하세요.

04 — 파일별 코드

복사해서 시작하는 완성본

각 파일을 그대로 만들면 됩니다. 모든 수치·이름은 08의 실측 스키마와 동일하며, 임의로 바꾼 것이 없습니다.

1) Data/RecordTalesTypes.h — 공용 타입의 뿌리

// Data/RecordTalesTypes.h — 여러 도메인이 공유하는 공용 타입. 변수는 없다 (12 지도).
#pragma once
#include "CoreMinimal.h"
#include "Engine/DataTable.h"
#include "RecordTalesTypes.generated.h"

class UTexture2D;
class USoundBase;

// ── 공용 열거형 5종 (08 §03) ──
// BlueprintType = 블루프린트에서도 이 enum을 변수·핀으로 쓸 수 있게 노출.
// : uint8 = 8비트 정수 기반 — UENUM(BlueprintType)의 필수 조건
UENUM(BlueprintType)
enum class ERecordSource : uint8 { Timer, Stopwatch, Quest, Manual };

UENUM(BlueprintType)
enum class EQuestRecurrence : uint8 { None, Daily, Weekly, Monthly };

UENUM(BlueprintType)
enum class EGuildRarity : uint8 { Normal, Rare, Epic, Legendary };

// 능력치 9종 — 프로토타입 STAT_POOL 순서. 값은 전부 % 단위.
UENUM(BlueprintType)
enum class EMemberStat : uint8
{
    FocusTime, GoldGain, QuestReward, StopwatchReward, AccuracyBonus,
    XpBonus, StreakBonus, AllReward, XpBoost
};

UENUM(BlueprintType)
enum class EUnlockCategory : uint8 { Preset, Squad, Stopwatch, Timer, Quest };

// 스탯 1개 = (종류, 수치%). 캐릭터 스탯·시너지 보너스 공용.
USTRUCT(BlueprintType)
struct FStatBonus
{
    GENERATED_BODY()
    // EditAnywhere = 에디터에서 값 편집 허용, BlueprintReadOnly = BP에서 읽기만(쓰기 금지)
    UPROPERTY(EditAnywhere, BlueprintReadOnly) EMemberStat Stat = EMemberStat::FocusTime;
    UPROPERTY(EditAnywhere, BlueprintReadOnly) float Value = 0.f;
};

// ── 통합 아카이브 기록 1건 (08 §05-1 · 11) ──
// 인자 없는 UPROPERTY() = 에디터 노출은 안 하지만 직렬화(SaveGame 저장)·GC 추적은 됨.
// 저장 전용 데이터라 편집 지정자가 필요 없다.
USTRUCT(BlueprintType)
struct FArchiveRecord
{
    GENERATED_BODY()
    UPROPERTY() FString Id;                    // "YYYYMMDD"+5자리 — 불변
    UPROPERTY() ERecordSource Source = ERecordSource::Timer;
    UPROPERTY() FString Category;
    UPROPERTY() FDateTime CreatedAt;
    UPROPERTY() int32 DurationSeconds = 0;
    UPROPERTY() FString Title;
    UPROPERTY() FString Notes;
    UPROPERTY() FString SourceRefId;           // Source==Quest: 원본 마스터 ID
    UPROPERTY() FDateTime DueDate;
};

// ── [레거시 · v1 마이그레이션 전용] 삭제 금지 — 11 문서의 이전 코드가 읽는다 ──
UENUM()
enum class EFocusMode : uint8 { Timer, Stopwatch };

USTRUCT()
struct FFocusRecord
{
    GENERATED_BODY()
    UPROPERTY() FDateTime Timestamp;
    UPROPERTY() EFocusMode Mode = EFocusMode::Timer;
    UPROPERTY() FString Category;
    UPROPERTY() int32 DurationSeconds = 0;
    UPROPERTY() FString Notes;
};

// ── DataTable 행 구조체 7종 (08 §04 — 값은 CSV/에디터에서) ──
// FTableRowBase를 상속해야 이 구조체를 DataTable의 "행 타입"으로 지정할 수 있다.

// Core/RecordTalesSaveGame.h — 디스크에 저장되는 유일한 지점 (08 §05-4).
// 모든 필드는 "사본" — 런타임 주인은 각 서브시스템 (12 지도). 슬롯 "RecordTalesSave"/0.

#include "GameFramework/SaveGame.h"
#include "Data/RecordTalesTypes.h"

// USaveGame 상속 = 이 클래스가 SaveGameToSlot/LoadGameFromSlot의 대상이 된다.
// 아래 필드는 전부 UPROPERTY()라야 직렬화 대상에 포함(빠뜨리면 저장 후 값이 사라짐).
UCLASS()
class RECORDTALES_API URecordTalesSaveGame : public USaveGame
{
    GENERATED_BODY()
public:

5) Core/RecordTalesGameInstance — 왕복의 창구

#pragma once
#include "Engine/GameInstance.h"
#include "RecordTalesGameInstance.generated.h"

class URecordTalesSaveGame;

UCLASS()
class RECORDTALES_API URecordTalesGameInstance : public UGameInstance
{
    GENERATED_BODY()
public:
    // 엔진이 GameInstance 생성 직후 1회 호출 — 앱 시작점. 여기서 세이브를 준비
    virtual void Init() override;

    // 디스크 왕복은 이 두 함수로만 — 서브시스템은 요청만 한다 (12 지도)
    // BlueprintCallable = BP에서 호출 가능(부수효과 있는 함수)
    UFUNCTION(BlueprintCallable, Category = "Save")
    void SaveNow();

    // BlueprintPure = 상태를 안 바꾸는 순수 조회 함수(BP에서 실행핀 없이 값만 반환)
    UFUNCTION(BlueprintPure, Category = "Save")
    URecordTalesSaveGame* GetSave() const { return CurrentSave; }

private:
    void LoadOrCreate();
    void InitNewSaveDefaults(URecordTalesSaveGame* Save);

    UPROPERTY()   // GC로부터 세이브 객체 보호 — 빠뜨리면 로드 후 임의 시점에 소멸
    TObjectPtr<URecordTalesSaveGame> CurrentSave;
};
#include "Core/RecordTalesGameInstance.h"
#include "Core/RecordTalesSaveGame.h"
#include "Data/RecordTalesSettings.h"
#include "Kismet/GameplayStatics.h"

static const TCHAR* kSlotName = TEXT("RecordTalesSave");   // 슬롯 이름의 단일 정의처
static constexpr int32 kUserIndex = 0;

void URecordTalesGameInstance::Init()
{
    Super::Init();
    LoadOrCreate();   // 어떤 서브시스템보다 먼저 세이브를 준비한다
}

void URecordTalesGameInstance::LoadOrCreate()
{
    if (UGameplayStatics::DoesSaveGameExist(kSlotName, kUserIndex))
    {
        CurrentSave = Cast<URecordTalesSaveGame>(
            UGameplayStatics::LoadGameFromSlot(kSlotName, kUserIndex));

        if (CurrentSave && CurrentSave->SaveVersion < 2)
        {
            // v1 세이브 감지 — 실제 이전(FocusRecords→ArchiveRecords)은
            // RecordSubsystem::MigrateV1ToV2가 담당 (11 문서, Phase 3).
            // Phase 1에서는 감지 로그만 남긴다.
            UE_LOG(LogTemp, Warning,
                TEXT("[Save] v%d 세이브 감지 — Phase 3 마이그레이션 대기"),
                CurrentSave->SaveVersion);
        }
    }

    if (!CurrentSave)   // 파일 없음 or 캐스팅 실패(손상) — 새로 시작
    {
        CurrentSave = Cast<URecordTalesSaveGame>(
            UGameplayStatics::CreateSaveGameObject(
                URecordTalesSaveGame::StaticClass()));
        InitNewSaveDefaults(CurrentSave);
        SaveNow();   // 첫 실행 즉시 파일 생성 — 이후 로직이 항상 파일 존재를 전제 가능
    }
}

void URecordTalesGameInstance::InitNewSaveDefaults(URecordTalesSaveGame* Save)
{
    // 기본값의 원천은 Settings — 코드에 수치를 중복하지 않는다
    const URecordTalesSettings* S = GetDefault<URecordTalesSettings>();

    Save->Gold              = S->StartingGold;
    Save->FocusMinutes      = S->DefaultFocusMinutes;
    Save->RestMinutes       = S->DefaultRestMinutes;
    Save->LastRestMinutes   = S->DefaultLastRestMinutes;
    Save->SingleRestMinutes = S->DefaultSingleRestMinutes;
    Save->RepeatCount       = S->DefaultRepeatCount;
    Save->Categories        = S->DefaultCategories;

    for (const FString& Cat : Save->Categories)
    {
        // 색은 UI Phase에서 팔레트 확정 — 우선 흰색 시드
        Save->CategoryColors.Add(Cat, FLinearColor::White);
    }
    // LastRolloverDate·QuestGoldDate는 기본값(0틱) 유지 —
    // 첫 롤오버 체크가 "과거"로 인식해 오늘 기준으로 초기화한다 (02 · 위키 FDateTime)
}

void URecordTalesGameInstance::SaveNow()
{
    if (!CurrentSave) return;
    const bool bOk =
        UGameplayStatics::SaveGameToSlot(CurrentSave, kSlotName, kUserIndex);
    if (!bOk)
    {
        UE_LOG(LogTemp, Error, TEXT("[Save] SaveGameToSlot 실패 — 디스크/권한 확인"));
    }
}
