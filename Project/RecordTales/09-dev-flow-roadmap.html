프로젝트 생성 절차

1
Epic Games Launcher 실행
Unreal Engine 탭에서 사용할 엔진 버전(5.4 이상 권장)의 Launch 버튼을 클릭합니다.
2
템플릿 선택: Blank — C++
Games 카테고리 → Blank 템플릿 → 하단의 C++ 라디오 버튼을 선택합니다. Blueprint가 아닌 C++을 선택해야 Visual Studio 솔루션이 자동 생성됩니다.
3
프로젝트 설정
Project Name: RecordTales
Location: 원하는 작업 경로 (예: D:/Projects/)
Target Platform: Desktop (Windows)
Quality Preset: Maximum (개발 중 최적화는 Phase 6에서)
4
IDE 설정 확인
Edit → Editor Preferences → Source Code → Source Code Editor에서 Visual Studio 2022 또는 Rider를 선택합니다. IDE가 정상 연동되면 File → Open Visual Studio로 솔루션이 열립니다.
5
필수 플러그인 확인
RecordTales는 순수 C++ + UMG로 구현합니다. 추가 플러그인은 필요 없습니다. Edit → Plugins에서 불필요한 플러그인(VR, AR, Niagara Extras 등)을 비활성화하면 컴파일 시간이 줄어듭니다.
6
.uproject 파일 확인
RecordTales.uproject를 텍스트 에디터로 열어 "Modules" 배열에 "RecordTales" 모듈이 등록되어 있는지 확인합니다. "Type": "Runtime", "LoadingPhase": "Default"가 기본값입니다.
⚠️
Hot Reload vs Live Coding — UE5의 Live Coding(Ctrl+Alt+F11)은 편리하지만, 헤더 변경이나 UPROPERTY 추가 시 에디터가 불안정해질 수 있습니다. 권장 워크플로우: 에디터를 닫고 → IDE에서 빌드(Ctrl+B) → 에디터 재실행. 특히 UCLASS/USTRUCT 구조를 변경할 때는 반드시 이 방식을 사용하세요.

03 — UE 빌드 시스템 기초

Build.cs — 모듈 의존성의 핵심

언리얼 빌드 시스템(UBT)은 .Build.cs 파일로 모듈 간 의존성을 관리합니다. RecordTales가 사용하는 모듈을 여기에 등록해야 #include가 정상 작동합니다.

RecordTales.Build.cs

// RecordTales.Build.cs
using UnrealBuildTool;

public class RecordTales : ModuleRules
{
    public RecordTales(ReadOnlyTargetRules Target) : base(Target)
    {
        PCHUsage = PCHUsageMode.UseExplicitOrSharedPCHs;

        PublicDependencyModuleNames.AddRange(new string[]
        {
            "Core",
            "CoreUObject",
            "Engine",
            "UMG",
            "Slate",
            "SlateCore",
            "InputCore"
        });

        PrivateDependencyModuleNames.AddRange(new string[] { });

        PublicIncludePaths.AddRange(new string[]
        {
            "RecordTales/Core",
            "RecordTales/Data",
            "RecordTales/Focus",
            "RecordTales/Quests",
            "RecordTales/Archive",
            "RecordTales/Guild",
            "RecordTales/Economy",
            "RecordTales/Audio",
            "RecordTales/UI"
        });
    }
}
⚠️
모듈 의존성 빠뜨리면 include해도 링커 에러 — 예를 들어 UMG 모듈을 Build.cs에 추가하지 않고 #include "Blueprint/UserWidget.h"를 쓰면 LNK2019: unresolved external symbol 에러가 발생합니다. 컴파일 에러가 아닌 링커 에러라서 원인을 찾기 어렵습니다.
⚠️
GENERATED_BODY() 매크로 빠뜨리면 컴파일 실패 — UCLASS(), USTRUCT(), UENUM()을 선언한 클래스/구조체 본문의 첫 줄에 반드시 GENERATED_BODY()를 넣어야 합니다. 누락 시 UHT(Unreal Header Tool)가 에러를 발생시킵니다.
🧭

Build.cs에 모듈 추가 후 프로젝트를 Regenerate해야 IntelliSense가 갱신됩니다. Visual Studio: RecordTales.uproject 우클릭 → Generate Visual Studio project files. Rider: 자동 갱신되지만, 안 되면 File → Invalidate Caches.

브레이크포인트 사용법 (Visual Studio)

F9로 코드 줄에 중단점을 설정합니다. F5로 디버그 모드 실행(에디터가 디버거에 연결된 상태로 뜹니다). 중단점에 걸리면 변수 위에 마우스를 올려 현재 값을 확인하고, F10(Step Over)이나 F11(Step Into)으로 한 줄씩 실행합니다.

PIE vs Standalone 차이

PIE(Play In Editor)는 에디터 프로세스 안에서 게임을 실행합니다. 에디터의 GC(가비지 컬렉션)와 공유하기 때문에, 간혹 에디터 전용 오브젝트가 섞여 들어올 수 있습니다. Standalone(독립 프로세스)은 별도 창에서 게임을 실행하므로, 최종 빌드와 더 가까운 환경에서 테스트할 수 있습니다. 저장/불러오기, 초기화 순서 등을 검증할 때는 Standalone을 사용하세요.

Output Log 열기: Window → Developer Tools → Output Log. 필터에 카테고리명(예: LogFocus)을 입력하면 해당 로그만 표시됩니다.

💡

서브시스템별로 로그 카테고리를 나누면 필터링이 쉽습니다. LogFocus, LogQuest, LogEconomy 등으로 분리하면 Output Log에서 원하는 서브시스템의 로그만 골라 볼 수 있습니다.

06 — 자주 발생하는 실수 & 트러블슈팅

개발 중 반드시 만나는 문제들

아래는 UE C++ 개발에서 거의 모든 개발자가 한 번은 겪는 실수들입니다. 미리 알아두면 디버깅 시간을 크게 절약할 수 있습니다.

⚠️
GENERATED_BODY() 누락 — UCLASS(), USTRUCT()를 선언한 클래스/구조체의 본문 첫 줄에 반드시 GENERATED_BODY()를 넣어야 합니다. 빠뜨리면 UHT가 "Expected GENERATED_BODY()" 에러를 발생시킵니다.
⚠️
순환 include — A.h가 B.h를 include하고, B.h가 다시 A.h를 include하면 컴파일이 무한 루프에 빠집니다. 해결: 헤더에서는 전방 선언(class UFocusSubsystem;)만 쓰고, 실제 #include는 .cpp 파일에서 합니다.
⚠️
Hot Reload 꼬임 — Live Coding 후 에디터가 이상하게 동작할 때(프로퍼티가 안 보이거나, 크래시). 해결: 에디터를 닫고 → IDE에서 빌드(Ctrl+B) → 에디터 재실행이 가장 안전합니다.
⚠️
리플렉션 매크로 오타 — UPROPERTY(EditAnywehre)처럼 지정자에 오타가 있으면 컴파일은 되지만 에디터에서 해당 프로퍼티가 안 보입니다. 에러 메시지가 없어서 원인을 찾기 어렵습니다.
⚠️
Build.cs 모듈 미등록 — 다른 모듈의 클래스를 #include했는데 LNK2019(unresolved external symbol) 에러가 나면, Build.cs의 PublicDependencyModuleNames에 해당 모듈을 추가했는지 확인하세요.
⚠️
SaveGame 직렬화 실패 — UPROPERTY() 매크로가 없는 필드는 SaveGameToSlot/LoadGameFromSlot에서 저장되지 않습니다. 영속 데이터는 반드시 UPROPERTY()를 붙여야 합니다.
⚠️
에디터에서 클래스 안 보임 — 새로 만든 C++ 클래스가 에디터의 클래스 목록이나 콘텐츠 브라우저에 나타나지 않을 때. 해결: 에디터 재시작, 또는 Ctrl+Shift+B로 전체 리빌드.

09 — PHASE 1: 프로젝트 기반 & 데이터 계층

토대를 세우고 저장을 검증한다

⚠️

선행 조건 — 기반 지식 블록(섹션 1~6)을 모두 읽었을 것. 프로젝트가 생성되고 빈 컴파일이 성공할 것.

구현 순서

1
공유 타입 정의 — Data/RecordTalesDataTypes.h
모든 USTRUCT(FFocusRecord, FQuestData, FGuildMember 등)와 UENUM(EFocusState, EQuestState, EAmbientType 등)을 이 파일 하나에 선언합니다. 이렇게 하면 서브시스템 간 순환 include를 원천적으로 방지할 수 있습니다.
2
URecordTalesGameInstance — Core/
UGameInstance를 상속. Gold, XP, GuildLevel을 UPROPERTY로 보유. Init()에서 SaveGame 로드, Shutdown()에서 저장.
3
URecordTalesSaveGame — Core/
USaveGame을 상속. 모든 영속 데이터를 UPROPERTY로 선언: FocusSettings, ActiveQuests, AllRecords, OwnedMembers, CurrentSquad, GuildLevel, XP, Gold, UnlockedIds, CurrentBgId.
4
URecordTalesSettings — Core/
UDeveloperSettings을 상속 (Config=Game). 밸런스 상수를 에디터에서 조정 가능하게 노출: TimerDuration(25분), RestDuration(5분), GoldTickInterval(6초), XPPerCycle(15), GachaCost(1000) 등.
5
DataTable 생성 (6개)
Content/RecordTales/DataTables/ 에 생성:
• DT_GuildLevels (20행: Lv, XPThreshold, UnlocksPerLevel)
• DT_MusicTracks (6행: Id, Title, Genre, Duration, SoundWave soft ref, bFree)
• DT_Backgrounds (9행: Id, DisplayName, Texture soft ref, Cost)
• DT_GuildCharacters (12행: Id, Name, Rarity, BaseStats)
• DT_UnlockTree (17행: UnlockId, Type, Cost, RequiredLevel)
• DT_QuestMasterData (퀘스트 템플릿: Id, Title, Type, RepeatRule, BaseReward)
6
저장/불러오기 테스트
GameInstance.Init()에서 UGameplayStatics::LoadGameFromSlot() 호출. 슬롯이 없으면 CreateSaveGameObject()로 새로 생성 후 SaveGameToSlot(). Output Log에서 로그 확인.

GameInstance 코드 예시

#pragma once

#include "CoreMinimal.h"
#include "Engine/GameInstance.h"
#include "RecordTalesGameInstance.generated.h"

class URecordTalesSaveGame;

UCLASS()
class RECORDTALES_API URecordTalesGameInstance : public UGameInstance
{
    GENERATED_BODY()

public:
    virtual void Init() override;
    virtual void Shutdown() override;

    /** SaveGame 인스턴스 (런타임 영속 데이터) */
    UPROPERTY()
    URecordTalesSaveGame* SaveGameRef;

    /** 현재 Gold */
    UPROPERTY(BlueprintReadOnly, Category = "Economy")
    int32 Gold;

    /** 현재 XP */
    UPROPERTY(BlueprintReadOnly, Category = "Economy")
    int32 XP;

    /** 현재 길드 레벨 */
    UPROPERTY(BlueprintReadOnly, Category = "Guild")
    int32 GuildLevel;

    /** 저장 슬롯 이름 */
    static const FString SaveSlotName;

private:
    void LoadSaveGame();
    void WriteSaveGame();
};
#include "RecordTalesGameInstance.h"
#include "RecordTalesSaveGame.h"
#include "RecordTalesLog.h"
#include "Kismet/GameplayStatics.h"

const FString URecordTalesGameInstance::SaveSlotName = TEXT("RecordTalesSlot");

void URecordTalesGameInstance::Init()
{
    Super::Init();
    LoadSaveGame();
    UE_LOG(LogRecordTales, Display,
        TEXT("GameInstance Init: Gold=%d XP=%d GuildLevel=%d"),
        Gold, XP, GuildLevel);
}

void URecordTalesGameInstance::Shutdown()
{
    WriteSaveGame();
    UE_LOG(LogRecordTales, Display, TEXT("GameInstance Shutdown: SaveGame written."));
    Super::Shutdown();
}

void URecordTalesGameInstance::LoadSaveGame()
{
    USaveGame* Loaded = UGameplayStatics::LoadGameFromSlot(SaveSlotName, 0);
    if (Loaded)
    {
        SaveGameRef = Cast<URecordTalesSaveGame>(Loaded);
        Gold       = SaveGameRef->Gold;
        XP         = SaveGameRef->XP;
        GuildLevel = SaveGameRef->GuildLevel;
        UE_LOG(LogRecordTales, Display, TEXT("SaveGame loaded successfully."));
    }
    else
    {
        SaveGameRef = Cast<URecordTalesSaveGame>(
            UGameplayStatics::CreateSaveGameObject(
                URecordTalesSaveGame::StaticClass()));
        Gold = 0; XP = 0; GuildLevel = 1;
        UE_LOG(LogRecordTales, Warning, TEXT("No SaveGame found. Created new."));
    }
}

void URecordTalesGameInstance::WriteSaveGame()
{
    if (!SaveGameRef) return;
    SaveGameRef->Gold       = Gold;
    SaveGameRef->XP         = XP;
    SaveGameRef->GuildLevel = GuildLevel;
    UGameplayStatics::SaveGameToSlot(SaveGameRef, SaveSlotName, 0);
}

10 — PHASE 2: 핵심 로직 서브시스템

비즈니스 로직의 심장을 만든다

⚠️

선행 조건 — Phase 1의 중간 검증을 모두 통과. GameInstance/SaveGame/DataTable이 동작할 것.

구현 순서

1
UEconomySubsystem — Economy/
Gold/XP 관리의 중심. 다른 서브시스템들이 가장 많이 참조하므로 가장 먼저 만듭니다.
핵심 함수: AddXP(int32), AddGold(int32), SpendGold(int32):bool, GetGuildLevel():int32, PurchaseUnlock(FString):bool, IsUnlocked(FString):bool
델리게이트: OnGoldChanged, OnXPChanged, OnLevelUp, OnUnlockChanged
DT_GuildLevels를 참조하여 XP 누적 → 레벨업 판정.
2
URecordSubsystem — Archive/
집중 기록 저장/조회 전담.
핵심 함수: LogFocusRecord(FFocusRecord), QueryByDate(FDateTime, FDateTime):TArray<FFocusRecord>, GetCalendarData(FDateTime):TArray<FFocusRecord>
SaveGame의 AllRecords 배열에 push하고, 날짜 범위 쿼리를 지원합니다.
3
UFocusSubsystem — Focus/
타이머/스톱워치 상태 머신. RecordTales의 핵심 기능입니다.
상태: EFocusState — Idle → Focusing → Resting → Complete
핵심 함수: StartFocus(), StopFocus(), PauseTimer(), ResumeTimer(), CompleteSession()
FTimerHandle로 카운트다운을 구현합니다.
Gold 틱: 6초마다 EconomySubsystem.AddGold(10) 호출.
세션 완료 시: EconomySubsystem.AddXP(15) + RecordSubsystem.LogFocusRecord().
4
UQuestSubsystem — Quests/
퀘스트 CRUD + 반복 규칙 + 자정 롤오버.
핵심 함수: CreateQuest(), UpdateQuestState(), CompleteQuest(), RolloverDaily()
마스터-인스턴스 패턴: DT_QuestMasterData에서 퀘스트 템플릿을 로드하고, 인스턴스(FQuestData)를 생성하여 ActiveQuests에 추가합니다.
자정 체크: FTimerHandle로 1분마다 현재 시각을 확인, 자정을 넘기면 RolloverDaily()를 호출하여 반복 퀘스트 인스턴스를 갱신합니다.
퀘스트 완료 시 EconomySubsystem.AddGold() 호출.

FocusSubsystem 코드 예시

#pragma once

#include "CoreMinimal.h"
#include "Subsystems/GameInstanceSubsystem.h"
#include "RecordTalesDataTypes.h"
#include "FocusSubsystem.generated.h"

DECLARE_DYNAMIC_MULTICAST_DELEGATE_OneParam(FOnFocusStateChanged, EFocusState, NewState);
DECLARE_DYNAMIC_MULTICAST_DELEGATE_OneParam(FOnTimerTick, float, RemainingSeconds);

UCLASS()
class RECORDTALES_API UFocusSubsystem : public UGameInstanceSubsystem
{
    GENERATED_BODY()

public:
    virtual void Initialize(FSubsystemCollectionBase& Collection) override;
    virtual void Deinitialize() override;

    UFUNCTION(BlueprintCallable) void StartFocus();
    UFUNCTION(BlueprintCallable) void StopFocus();
    UFUNCTION(BlueprintCallable) void PauseTimer();
    UFUNCTION(BlueprintCallable) void ResumeTimer();

    UPROPERTY(BlueprintReadOnly)
    EFocusState CurrentState = EFocusState::Idle;

    UPROPERTY(BlueprintAssignable) FOnFocusStateChanged OnFocusStateChanged;
    UPROPERTY(BlueprintAssignable) FOnTimerTick OnTimerTick;

private:
    FTimerHandle CountdownHandle;
    FTimerHandle GoldTickHandle;
    float RemainingTime;

    void OnCountdownTick();
    void OnGoldTick();
    void CompleteSession();
};
#include "FocusSubsystem.h"
#include "EconomySubsystem.h"
#include "RecordSubsystem.h"
#include "RecordTalesSettings.h"
#include "RecordTalesLog.h"

void UFocusSubsystem::StartFocus()
{
    const auto* Settings = GetDefault<URecordTalesSettings>();
    RemainingTime = Settings->TimerDuration;
    CurrentState = EFocusState::Focusing;
    OnFocusStateChanged.Broadcast(CurrentState);

    // 1초마다 카운트다운 틱
    GetGameInstance()->GetTimerManager().SetTimer(
        CountdownHandle, this, &UFocusSubsystem::OnCountdownTick,
        1.0f, true);

    // Gold 틱 (기본 6초 간격)
    GetGameInstance()->GetTimerManager().SetTimer(
        GoldTickHandle, this, &UFocusSubsystem::OnGoldTick,
        Settings->GoldTickInterval, true);

    UE_LOG(LogFocus, Display, TEXT("Focus started: %.0f seconds"), RemainingTime);
}

void UFocusSubsystem::OnCountdownTick()
{
    RemainingTime -= 1.0f;
    OnTimerTick.Broadcast(RemainingTime);

    if (RemainingTime <= 0.0f)
    {
        CompleteSession();
    }
}

void UFocusSubsystem::OnGoldTick()
{
    auto* EcoSub = GetGameInstance()->GetSubsystem<UEconomySubsystem>();
    if (EcoSub)
    {
        const auto* Settings = GetDefault<URecordTalesSettings>();
        EcoSub->AddGold(Settings->GoldPerTick);
    }
}

void UFocusSubsystem::CompleteSession()
{
    GetGameInstance()->GetTimerManager().ClearTimer(CountdownHandle);
    GetGameInstance()->GetTimerManager().ClearTimer(GoldTickHandle);

    // XP 지급
    auto* EcoSub = GetGameInstance()->GetSubsystem<UEconomySubsystem>();
    if (EcoSub)
    {
        const auto* Settings = GetDefault<URecordTalesSettings>();
        EcoSub->AddXP(Settings->XPPerCycle);
    }

    // 기록 저장
    auto* RecSub = GetGameInstance()->GetSubsystem<URecordSubsystem>();
    if (RecSub)
    {
        FFocusRecord Record;
        Record.Timestamp = FDateTime::Now();
        Record.DurationSeconds = GetDefault<URecordTalesSettings>()->TimerDuration;
        Record.bCompleted = true;
        RecSub->LogFocusRecord(Record);
    }

    CurrentState = EFocusState::Resting;
    OnFocusStateChanged.Broadcast(CurrentState);
    UE_LOG(LogFocus, Display, TEXT("Focus session completed. Entering rest phase."));
}

void UFocusSubsystem::Deinitialize()
{
    // 반드시 타이머 정리
    if (UGameInstance* GI = GetGameInstance())
    {
        GI->GetTimerManager().ClearTimer(CountdownHandle);
        GI->GetTimerManager().ClearTimer(GoldTickHandle);
    }
    Super::Deinitialize();
}
⚠️
서브시스템 간 직접 #include 대신 GetSubsystem으로 접근 — GetGameInstance()->GetSubsystem<UEconomySubsystem>()를 사용합니다. 헤더에서는 전방 선언만 하고, .cpp에서 include합니다.
⚠️
FTimerHandle은 서브시스템 Deinitialize()에서 반드시 ClearTimer — 정리하지 않으면 이미 파괴된 오브젝트를 참조하여 크래시가 발생할 수 있습니다.
