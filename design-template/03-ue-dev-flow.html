01 — 구성 개요

4블록 · 17섹션 구조

개발 플로우 로드맵 문서는 기반 지식 → 설계 이해 → 구현(Phase 1~6) → 마무리 순서로 독자를 안내합니다. Phase 진입 전에 컨벤션·빌드·Git 등 기반을 완료하고, 설계 맵으로 전체 그림을 잡은 뒤, 각 Phase를 선행 조건 → 구현 → 중간 검증 패턴으로 반복하며, 마무리 블록에서 리스크와 총괄 마일스톤을 점검합니다.

02 — Phase 파이프라인

개발 페이즈 총괄 로드맵 패턴

섹션 8에서 사용하는 패턴입니다. 6개 Phase를 .pipe-step으로 나열하여 전체 개발 순서를 한눈에 보여줍니다. 각 Phase의 .pipe-desc에는 핵심 산출물과 의존 관계를 요약합니다.

Live — 개발 페이즈 총괄
P1
프로젝트 기반 & 데이터 계층
GameInstance, SaveGame, DataTable, 에셋 매니저 설정. 선행 없음
P2
핵심 로직 서브시스템
GameMode, PlayerController, Character, 입력 매핑. P1 완료 후

게임화 서브시스템
퀘스트, 인벤토리, 스탯, 레벨링 등 게임 고유 로직. P2 완료 후
P4
비주얼 에셋 제작 & 임포트
3D 모델, 머티리얼, 애니메이션, VFX 임포트·세팅. P1 이후 병렬 가능
P5
UI 프레임워크 & 위젯
HUD, 메뉴, 인벤토리 UI 위젯 구현. P3 + P4 완료 후
P6
통합·빌드·패키징·배포
전체 통합 테스트, 패키징 설정, 배포 파이프라인. P5 완료 후
03 — Phase 블록 패턴

선행 조건 → 구현 내용 → 중간 검증

섹션 9~14(구현 블록)에서 각 Phase는 동일한 3파트 구조를 반복합니다: 선행 조건 확인 → 구현 내용 상세 → 중간 검증 체크리스트. 아래는 Phase 2를 예시로 보여줍니다.

Live — Phase 블록 예시 (Phase 2)

Phase 2: 핵심 로직 서브시스템

⚠️

선행 조건 — Phase 1(프로젝트 기반 & 데이터 계층)이 완료되어야 합니다. GameInstance·DataTable이 정상 작동하는지 확인하세요.

2-1
GameMode 생성
AMyGameMode 클래스를 만들고 Project Settings → Maps & Modes에 등록한다.
2-2
PlayerController 생성
AMyPlayerController를 만들고 Enhanced Input으로 입력 매핑을 설정한다.
2-3
Character 생성
AMyCharacter를 만들고 이동·카메라를 구성한다. GameMode의 DefaultPawnClass에 지정한다.
✅

Phase 2 중간 검증

  • PIE 실행 시 캐릭터가 스폰되고 WASD 이동이 동작하는가?
  • GameMode 로그(UE_LOG)에서 InitGame → Login → Possess 순서가 확인되는가?
  • PlayerController에서 Enhanced Input Action이 정상 바인딩되는가?
🧭

모든 Phase는 이 선행 조건(.callout.warn) → 구현 단계(.pipe-step) → 중간 검증(.callout.tip) 패턴을 따릅니다. 검증을 통과해야 다음 Phase로 진행합니다.

04 — 시스템 의존성 맵

의존성 맵 (.rel-map) 패턴

섹션 7에서 사용하는 패턴입니다. 시스템 간 의존 방향을 .rel-map으로 표현하여, 어떤 시스템이 다른 시스템을 전제하는지 시각화합니다.

의존성 맵은 Phase 간 선후 관계의 근거가 됩니다. 화살표 방향은 "사용하는 쪽 → 제공하는 쪽"입니다. 실제 문서에서는 프로젝트의 모든 핵심 서브시스템을 빠짐없이 나열합니다.

마일스톤 총괄 검증표 패턴

섹션 16에서 사용하는 패턴입니다. 전체 Phase의 완료 기준을 하나의 표로 모아 프로젝트 진행률을 추적합니다.
