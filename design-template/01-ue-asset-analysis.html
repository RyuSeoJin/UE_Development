05 — 에셋 분석 문서 템플릿

새 에셋을 분석할 때의 표준 구성

언리얼 MCP로 에셋을 받으면 구조를 파악한 뒤, 아래 7개 섹션 순서를 그대로 따릅니다. 이 순서가 "개념 → 구조 → 사용 → 응용 → 팁"이라는 학습 흐름을 보장합니다.

01
개념 이해
이 에셋이 무엇이고 왜 쓰는지. 한 문장 정의 + 핵심 용어 callout + 구성요소 스탯 + 종류 카드 그리드.
02
구조 이해
에셋 타입 컬러 트리 + 범례. 폴더를 3~4개 레이어로 요약하는 표. "어디만 만지면 되는지"를 분명히.
03
핵심 구성요소
주요 오브젝트의 변수/속성 표(타입 태그 포함). 대표 1개는 전부, 나머지는 차이만. 내부 컴포넌트 표.
04
데이터 타입 & 공통 요소
Enum은 필(pill)로, 동작 흐름은 Arch 다이어그램으로. 공통 변수의 의미를 짧게.
05
사용 방법
번호 파이프라인으로 단계별. 블루프린트 노드 흐름은 코드 블록(의사 코드)으로. 전제 조건 callout.
06
응용 방법
기본을 넘어선 확장 아이디어 카드 그리드. 커스터마이징·조합·재사용·검증 등 에셋에 근거한 실전 시나리오.
07
팁 & 주의사항
Pitfall 리스트(⚠️ 주의 / 💡 팁) + FAQ 표(증상 → 해결).
🧭

섹션 수는 에셋 복잡도에 따라 줄여도 됩니다 — 단 개념 · 구조 · 사용 · 응용 · 팁 다섯 기둥은 항상 포함합니다. 데이터 타입이 없는 단순 에셋이면 04를 03에 흡수해도 좋습니다.
